# dynamic styles with js

- JS может менять CSS напрямую
- это называется dynamic styles - динамическое изменение стилей во время работы страницы

## свойство style

- у каждого DOM-элемента есть объект `style`, через который можно менять inline-стили

```html
<div id="banner"></div>
```

```js
const bannerElement = document.getElementById('banner');
bannerElement.style.backgroundColor = 'red';
```

- браузер автоматически добавит :

```html
<div id="banner" style="background-color: red;"></div>
```

### css и js используют разные названия

- в css - kebab-case
- в js - camelCase

### единицы измерения нужно писать вручную

```js
bannerElement.style = '300px'; // не просто 300
```

### несколько стилей подряд

```js
bannerElement.style.width = '300px';
bannerElement.style.height = '100px';
bannerElement.style.backgroundColor = 'orange';
bannerElement.style.borderRadius = '10px';
```

### удаление стиля

```js
bannerElement.style.backgroundColor = '';
```

- браузер удалит этот inline-стиль

### что возвращает `element.style`

- специальный объект со всеми inline-стилями элемента

## cssText

- можно задать много стилей одной строкой:

```js
bannerElement.style.cssText = `
  width: 300px;
  height: 100px;
  background: orange;
`;
```

- получится:

```html
<div
  style="
    width:300px;
    height:100px;
    background:orange;
  "
></div>
```

### недостаток cssText

- он перезаписывает все содержимое `style`

## setProperty()

- иногда имя css-свойства неудобно записывать через camelCase
- можно использовать:

```js
bannerElement.style.setProperty('background-color', 'green');
```

## removeProperty()

- удаление свойства:

```js
bannerElement.style.removeProperty('background-color');
```

## частая практика - менять классы, а не стили

- вместо:

```js
bannerElement.style.backgroundColor = 'red';
bannerElement.style.color = 'white';
bannerElement.style.padding = '20px';
```

- обычно делают:

```css
.error-banner {
  background: red;
  color: white;
  padding: 20px;
}
```

```js
bannerElement.classList.add('error-banner');
```

## почему классы обычно лучше

- логика остается в js
- стили остаются в css
- код легче поддерживать
- проще менять дизайн

## когда используют style

- обычно, когда значение заранее неизвестно
- например:

```js
progressElement.style.width = '75%';

// или

tooltipElement.style.left = '120px';

// или
modalElement.style.top = `${mouseY}px`;
```

- то есть стиль вычисляется во время выполнения программы
