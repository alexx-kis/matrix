# computed styles

- `element.style` не показывает все стили элемента, он показывает только inline-стили
- именно для получения итоговых (финальных) стилей существует понятие computed styles

```html
<div id="card"></div>
```

```css
#card {
  width: 300px;
  color: blue;
}
```

```js
const cardElement = document.getElementById('card');
```

- у элемента есть стили в css-файле, не в атрибуте `style`, поэтому

```js
console.log(cardElement.style.width); // ''
```

## почему так происходит

- потому что `element.style` содержит только inline-стили

## получение computed styles

- для этого используется

```js
getComputedStyle(element);

const cardStyles = getComputedStyle(cardElement);

console.log(cardStyles.width); // '300px'
```

- `getComputedStyle` возвращает специальный объект `CSSStyleDeclaration`, который содержит итоговые значения всех css-свойств

- браузер собирает стили из всех источников (html, css, inline-style (js), наследование), после чего вычисляет финальный результат

## важно: значения уже вычислены

```css
width: 50%;
```

- например, родитель имеет ширину 800px

```js
const panelStyles = getComputedStyle(panelElement);

console.log(panelStyles.width); // '400px'
```

- цвета преобразуются в `rgb(, , )`

## доступ через camelCase

```js
const cardStyles = getComputedStyle(cardElement);

console.log(cardStyles.backgroundColor);
```

## или через getPropertyValue()

```js
cardStyles.getPropertyValue('background-color');
```

## computed styles нельзя изменить
