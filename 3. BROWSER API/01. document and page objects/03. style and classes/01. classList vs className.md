# classList vs className

## зачем нужны className и classList

```html
<div id="notification" class="success visible"></div>
```

```css
.success {
  color: green;
}

.visible {
  display: block;
}
```

- из JS часто нужно:
  - добавить класс
  - удалить класс
  - проверить наличие класса
  - заменить класс

```js
// для этого существует 2 основных способа:
element.className;

// и
element.classList;
```

## className

- это строка со всеми классами элемента

```html
<div id="status" class="success visible"></div>
```

```js
const statusElement = document.getElementById('status');

console.log(statusElement.className); // "success visible"
```

- то есть браузер просто возвращает содержимое атрибута `class`

## изменение через className

- можно заменить все классы сразу

```js
statusElement.className = 'error';
```

- классы success и visible полностью исчезли

## добавление класса через className

- раньше частно делали так:

```js
statusElement.className += ' active';
```

```html
// было: class="success visible" // стало: class="success visible active"
```

- работает, но есть проблемы:

1. можно случайно удалить классы

```js
statusElement.className = 'active';
```

- разработчик хотел добавить класс, но фактически он затер остальные классы

```
class="success visible" -> class="active"
```

2. появляются дубликаты

```js
statusElement.className += ' active';
statusElement.className += ' active';
```

- получим:

```html
class="success visible active active"
```

- класс появился дважды

3. неудобно проверять наличие класса

- приходится работать со строкой:

```js
statusElement.className.includes('active');
```

- но тут есть ошибка, например:

```html
class="inactive"
```

- тогда

```js
element.className.includes('active'); // true
```

## classList

- для решения этих проблем появился специальный объект `element.classList`
- он предоставляет методы для работы с классами

```js
console.log(statusElement.classList); // DOMTokenList
```

- это специальная коллекция классов элемента

### добавление класса

```js
statusElement.classList.add('active');
```

### повторное добавление

- дубликатов не будет

### удаление класса

```js
statusElement.classList.remove('visible');
```

### проверка наличия класса

```js
statusElement.classList.contains('active'); // true/false
```

### переключение класса

```js
statusElement.classList.toggle('active');
```

- если класс есть, то он удалится; если класса нет, то он добавится

### замена класса

```js
statusElement.classList.replace('success', 'error');
```

- класс `success` заменится на `error`

### добавление нескольких классов

```js
statusElement.classList.add('active', 'large', 'rounded');
```

### перебор классов

```js
for (const cssClass of statusElement.classList) {
  console.log(cssClass);
}
```

## когда использовать classList

- практически всегда

- для:
  - добавления классов
  - удаление классов
  - проверки классов
  - переключения классов
  - замены классов
