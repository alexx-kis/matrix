# events delegation

- представим список из 100 кнопок

```html
<ul>
  <li><button>Item 1</button></li>
  <li><button>Item 2</button></li>
  <li><button>Item 3</button></li>
</ul>
```

- первое, что приходит в голову:

```js
const itemButtonElements = document.querySelectorAll('button');

itemButtonElements.forEach((itemButtonElement) => {
  itemButtonElement.addEventListener('click', () => {
    console.log('clicked');
  });
});
```

- то есть 100 кнопок -> 100 обработчиков

## проблема динамических элементов

```js
const productsListElement = document.querySelector('#productsList');

productsListElement.insertAdjacentHTML(
  'beforeend',
  `
    <li>
      <button>
        New item
      </button>
    </li>
  `,
);
```

- появилась новая кнопка, но обработчик не появился, потому что `document.querySelectorAll('button');` выполнился раньше, новая кнопка не участвовала в подписке

## идея делегирования

- вместо подписки на каждую кнопку отдельно мы подписываемся на общего родителя

```html
<ul id="productsList">
  <li><button>Item 1</button></li>
  <li><button>Item 2</button></li>
  <li><button>Item 3</button></li>
</ul>
```

```js
const productsListElement = document.querySelector('#productsList');

productsListElement.addEventListener('click', (event) => {
  console.log(event.target);
});
```

## почему это работает

- вспоминаем bubbling

- при клике по кнопке событие поднимается до элемента списка
- обработчик списка получает событие

### event.target приходит на помощь

- допустим, пользователь нажал на кнопку

```html
<button>Item 2</button>
```

- в обработчике списка:

```js
productsListElement.addEventListener('click', (event) => {
  console.log(event.target);
});
```

- получим:

```
<button>
  Item 2
</button>
```

- то есть мы можем понять, по какой кнопке кликнули

## самая частая проблема

- пользователь нажал не кнопку, а текст внутри нее, тогда `console.log(event.target)` вернет `<span>`, а не кнопку

```html
<button>
  <span>Buy</span>
</button>
```

```js
console.log(event.target);
```

```
<span>
  Buy
</span>
```

### решение - closest()

- очень часто используется вместе с делегированием

```js
catalogContainerElement.addEventListener('click', (event) => {
  const buyButtonElement = event.target.closest('.buy-button');

  if (!buyButtonElement) {
    return;
  }

  console.log('buy');
});
```

### что делает closest()

- поднимается вверх по DOM

```html
<button class="buy-button">
  <span>Buy</span>
</button>
```

- если кликнули по span, то

```js
event.target.closest('.buy-button');
```

- найдет `<button class="buy-button">`

## типичный паттерн делегирования

```js
tableBodyElement.addEventListener('click', (event) => {
  const rowElement = event.target.closest('.row');

  if (!rowElement) {
    return;
  }

  console.log(rowElement);
});
```

## польза делегирования

1. меньше обработчиков

2. работает с динамическими элементами

3. проще поддерживать код

## когда делегирование не подходит

- иногда событие не всплывает (focus, blur)

## event.target vs currentTarget в делегировании

```html
<ul id="categoriesList">
  <li>Phones</li>
</ul>
```

```js
categoriesListElement.addEventListener('click', (event) => {
  console.log(event.target);
  console.log(event.currentTarget);
});
```

- клик по `<li>`:

- event.target -> li (потому что событие возникло на нем)

- event.currentTarget -> ul (потому что обработчик висит на нем)
