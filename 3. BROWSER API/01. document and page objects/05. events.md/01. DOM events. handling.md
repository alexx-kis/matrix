# DOM events. handling

- событие - это сигнал о том, что что-то произошло
- например:
  - пользователь нажал кнопку
  - пользователь ввел текст в поле
  - пользователь навел мышь
  - страница загрузилась
  - пользователь нажал клавишу на клавиатуре

- браузер постоянно отслеживает такие действия и генерирует события

## из чего состоит работа с событиями

- обычно есть 3 участника:
  - источник события (event target)
  - событие (event)
  - обработчик события (event handler / listener)

- пример:

```html
<button id="submitButton">Send</button>
```

```js
const submitButtonElement = document.querySelector('#submitButton');

submitButtonElement.addEventListener('click', () => {
  console.log('Button clicked');
});
```

- здесь:
  - кнопка -> источник события
  - click -> событие
  - функция -> обработчик события

```
Пользователь нажал кнопку
            ↓
Браузер создал событие click
            ↓
Браузер вызвал обработчик
            ↓
Выполнился код
```

## event handler

- обработчик события - это функция, которую браузер вызывает при возникновении события

```js
function handleButtonClick() {
  console.log('Clicked');
}
```

- сама по себе функция ничего не делает, она начнет вызываться только после регистрации

```js
submitButtonElement.addEventListener('click', handleButtonClick);
```

## addEventListener()

- основной современный способ подписаться на событие

```js
element.addEventListener(eventType, listener);
```

### eventType - тип события

```js
'click';
'mouseover';
'keydown';
'input';
'submit';
```

### параметр listener - функция-обработчик

```js
function handleSaveClick() {
  console.log('Saved');
}

saveButtonElement.addEventListener('click', handleSaveClick);
```

## несколько обработчиков одного события

- можно подписать несколько функций на одно событие

```js
function trackClick() {
  console.log('Analytics');
}

function showNotification() {
  console.log('Notification');
}

buyButtonElement.addEventListener('click', trackClick);

buyButtonElement.addEventListener('click', showNotification);
```

- при клике выполнятся обе функции

## анонимная функция

- часто обработчик пишут прямо внутри addEventListener

```js
logoutButtonElement.addEventListener('click', () => {
  console.log('Logout');
});
```

## event object

- при возникновении события браузер создает специальный объект события, который автоматически передается первым аргументом в функцию-обработчик

```js
profileButtonElement.addEventListener('click', (event) => {
  console.log(event);
  console.log(event.type); // click
});
```

## removeEventListener

- удаляет обработчик события

```js
element.removeEventListener(eventType, listener);
```

### важное правило

- для удаления нужен тот же объект функции, который использовался при добавлении

```js
function handleCloseClick() {
  console.log('Close');
}

closeButtonElement.addEventListener('click', handleCloseClick);

closeButtonElement.removeEventListener('click', handleCloseClick);
```

- удалить анонимную функцию нельзя, потому что каждая стрелочная функция создает новый объект функции
- для JS это 2 разные функции, поэтому удалить обработчик не получится

## addEventListener vs onclick

- старый способ:

```js
searchButtonElement.onclick = () => {
  console.log('Search');
};
```

1. можно использовать только один обработчик

```js
searchButtonElement.onclick = () => {
  console.log('First');
};

searchButtonElement.onclick = () => {
  console.log('Second');
};
```

- результат: 'Second' - первый обработчик затерт
- addEventListener позволяет несколько обработчиков

2. хуже управляется

- удаление через `onclick` обычно выглядит так:

```js
searchButtonElement.onclick = null;
// это менее гибко, чем:

searchButtonElement.removeEventListener('click', handleSearchClick);
```

## объект Event

- практически все обработчики получают объект Event

```js
messageButtonElement.addEventListener('click', (event) => {
  console.log(event);
});
```

- самые часто используемые свойства:
  - тип события `event.type`
  - элемент, на котором произошло событие `event.target`
  - элемент, на котором сейчас выполняется обработчик `event.currentTarget`
