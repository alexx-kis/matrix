# events propagation (bubbling, capturing)

- когда событие происходит на элементе, оно не обязательно остается только на этом элементе
- оно может проходить через несколько элементов DOM-дерева

```html
<body>
  <section id="catalogSection">
    <button id="orderButton">Order</button>
  </section>
</body>
```

- DOM-дерево:

```
body
└── section
    └── button
```

- пользователь нажал кнопку -> в событии могут участвовать button, section, body, document, window

## фазы события:

1. capturing phase
2. target phase
3. bubbling phase

## схема

- пользователь нажал кнопку

```
window
↓
document
↓
body
↓
section
↓
button
```

- это Capturing
- после достижения кнопки

```
button
```

- это Target

- потом событие поднимается обратно:

```
button
↑
section
↑
body
↑
document
↑
window
```

- это Bubbling

## Capturing phase

- также называется Capture phase, Event capturing

- событие идет сверху вниз

```
window
↓
document
↓
html
↓
body
↓
section
↓
button
```

- по умолчанию эту фазу почти никто не использует
- большинство обработчиков работают на следующей фазе

## Target phase

- событие достигло элемента, на котором произошло действие
- например, кликнули по button, значит target-фаза происходит на button

## Bubbling phase

- самая важная фаза
- после target событие начинает подниматься вверх

```
button
↑
section
↑
body
↑
document
↑
window
```

- это называется Event bubbling - всплытие события

## пример

```html
<section id="productSection">
  <button id="buyNowButton">Buy</button>
</section>
```

```js
const productSectionElement = document.querySelector('#productSection');

const buyNowButtonElement = document.querySelector('#buyNowButton');

productSectionElement.addEventListener('click', () => {
  console.log('Section');
});

buyNowButtonElement.addEventListener('click', () => {
  console.log('Button');
});
```

- клик по кнопке:

```
Button
Section
```

- почему?:

```
Событие произошло на button
↓
Выполнился обработчик button
↓
Событие всплыло вверх
↓
Выполнился обработчик section
```

## Bubbling происходит автоматически

- большинство DOM-событий всплывают автоматически
- например: click, input, change, keydown, keyup

## event.target

- показывает элемент, на котором событие возникло

```js
productSectionElement.addEventListener('click', (event) => {
  console.log(event.target); // <button id="buyNowButton">
});
```

- важно: target не меняется, во время всплытия он всегда указывает на исходный элемент

## event.currentTarget

- показывает элемент, обработчик которого выполняется сейчас

```js
productSectionElement.addEventListener('click', (event) => {
  console.log(event.currentTarget); // <section id="productSection">
});
```

## остановка всплытия

- для этого используется

```js
event.stopPropagation();
```

```js
productSectionElement.addEventListener('click', () => {
  console.log('Section');
});

buyNowButtonElement.addEventListener('click', (event) => {
  event.stopPropagation();

  console.log('Button');
});
```

- клик по кнопке -> Button

- что произошло:

```
button
↑
section
↑
body
```

- после вызова `event.stopPropagation()` цепочка оборвалась на кнопке

### stopImmediatePropagation()

- более сильная версия

```js
function handlePrimaryClick(event) {
  event.stopImmediatePropagation();

  console.log('Primary');
}

function handleSecondaryClick() {
  console.log('Secondary');
}

actionPanelButtonElement.addEventListener('click', handlePrimaryClick);

actionPanelButtonElement.addEventListener('click', handleSecondaryClick);
```

- результат: Primary
- без `stopImmediatePropagation()` было бы Primary Secondary

### разница

```
stopPropagation()
↓
останавливает движение вверх

stopImmediatePropagation()
↓
останавливает движение вверх
+
останавливает остальные обработчики элемента
```

## Capturing handlers

- по умолчанию обработчики работают во время всплытия
- то есть:

```js
catalogElement.addEventListener('click', handleCatalogClick);

// равносильно:

catalogElement.addEventListener('click', handleCatalogClick, false);
```

- но можно подписаться на фазу захвата:

```js
catalogElement.addEventListener('click', handleCatalogClick, true);

// или современный вариант:

catalogElement.addEventListener('click', handleCatalogClick, { capture: true });
```

### что изменится?

```html
<section id="catalogSection">
  <button id="detailsButton">Details</button>
</section>
```

```js
catalogSectionElement.addEventListener(
  'click',
  () => {
    console.log('Section');
  },
  { capture: true },
);

detailsButtonElement.addEventListener('click', () => {
  console.log('Button');
});
```

- клик по кнопке:

```
Section
Button
```

- почему:

```
Capture phase
↓
Section

Target phase
↓
Button
```
