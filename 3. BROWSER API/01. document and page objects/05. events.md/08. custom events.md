# custom events

- пользовательские события
- можно создать свое собственное событие, например:
  - userLoggedIn
  - cartUpdated
  - themeChanged
  - productAdded
  - modalOpened

- таких событий в браузере нет
- для этого существуют Custom Events

## зачем нужны custom events

- представим интернет-магазин
- пользователь добавил товар в корзину
- после этого нужно:
  - обновить корзину
  - обновить итоговую цену
  - отправить аналитику
  - показать уведомление

- можно вызывать все функции напрямую:

```js
updateCartCounter();
updateCartTotal();
sendAnalytics();
showNotification();
```

- но компоненты становятся сильно связаны друг с другом

- вместо этого можно создать событие `cartUpdated`, а остальные части приложения сами подпишутся на него

## создание события

- для этого используется конструктор:

```js
new Event(eventName);
```

### пример

```js
const cartUpdatedEvent = new Event('cartUpdated');
```

- но событие еще не произошло

## dispatchEvent()

- чтобы запустить событие используется

```js
element.dispatchEvent(event);
```

### пример

```js
document.dispatchEvent(cardUpdatedEvent);
```

## подписка на событие

```js
document.addEventListener('cartUpdated', () => {
  console.log('Cart Updated');
});
```

## почему `Event` используется редко

- обычный `Event` не позволяет передавать данные

- например:

```js
const productAddedEvent = new Event('productAdded');
```

- мы можем сообщить: "Событие произошло", но не можем нормально передать: какой товар, какое количество, какая цена

- поэтому чаще используют `CustomEvent`

## CustomEvent

```js
new CustomEvent(eventName, options);
```

### пример

```js
const themeChangedEvent = new CustomEvent('themeChanged');
```

## передача данных через detail

```js
const productAddedEvent = newCustomEvent('productAdded', {
  detail: {
    productId: 42,
    quantity: 3,
  },
});
```

- теперь событие содержит данные

## получение данных

- подписываемся

```js
document.addEventListener('productAdded', (event) => {
  console.log(event.detail);
});
```

- запускаем:

```js
document.dispatchEvent(productAddedEvent);
```

- результат:

```js
{
  productId: 42,
  quantity: 3,
}
```

## часто используемый паттерн

- создание:

```js
const userLoggedInEvent = new CustomEvent('userLoggedIn', {
  detail: {
    userId: 15,
  },
});
```

- подписка:

```js
document.addEventListener('userLoggedIn', (event) => {
  console.log(event.detail.userId);
});
```

- отправка:

```js
document.dispatchEvent(userLoggedInEvent);
```

## где отправлять событие

- можно отправлять не только на document

```js
profileCardElement.dispatchEvent(profileUpdateEvent);
```

- тогда событие будет связано именно с этим элементом

## всплытие пользовательских событий

- по умолчанию CustomEvent не всплывает

- то есть:

```js
new CustomEvent('profileUpdated'); // не будет подниматься по DOM
```

- чтобы включить всплытие:

```js
const profileUpdateEvent = new CustomEvent('profileUpdated', {
  bubbles: true,
});
```

- теперь работает обычный bubbling

## отмена пользовательских событий

- можно сделать событие отменяемым

- создание:

```js
const checkoutStartEvent = new CustomEvent('checkoutStarted', {
  cancelable: true,
});
```

- подписчик:

```js
document.addEventListener('checkoutStarted', (event) => {
  event.preventDefault();
});
```

- отправитель:

```js
const dispatchResult = document.dispatchEvent(checkoutStartedEvent);
```

```js
dispatchResult === false; // если событие отменили
dispatchResult === true; // если событие не отменили
```

## dispatchEvent работает синхронно

- многие думают

```js
document.dispatchEvent(customNotificationEvent);
```

- запускает событие позже – это неверно

- на самом деле:

```
dispatchEvent -> все обработчики выполняются сразу -> dispatchEvent завершается
```

### пример

```js
console.log('A');

document.dispatchEvent(syncEvent);

console.log('B');
```

- результат:

```
A
обработчики события
B
```

## когда используются Custom Events

- чаще всего:
  - связь между компонентами
  - виджеты
  - Web Components
  - большие SPA
  - плагины и библиотеки

- особенно часто Custom Events встречаются в теме Web Components, потому что компонент может сообщать наружу:
  - modalOpened
  - modalClosed
  - tabChanged
  - ratingChanged
