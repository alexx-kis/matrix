# content, document, window load events

- это события, связанные с жизненным циклом страницы
- они позволяют понять:
  - когда построился DOM
  - когда загрузились стили
  - когда загрузились картинки
  - когда пользователь уходит со страницы
  - когда изменилось состояние документа

## главная проблема

```html
<body>
  <button id="saveButton">Save</button>
</body>
```

```js
const saveButtonElement = document.querySelector('#saveButton');

console.log(saveButtonElement);
```

- если скрипт выполнится раньше, чем браузер построит DOM -> null

- поэтому исторически появилось событие загрузки страницы

## DOMContentLoaded

- срабатывает, когда HTML загружен -> DOM построен

```js
document.addEvenListener('DOMContentLoaded', () => {
  console.log('DOM ready');
});
```

- после этого можно безопасно работать с DOM

## что не ждет DOMContentLoaded

- изображения
- видео
- аудио
- iframe

## load

- срабатывает, когда загружено всё:
  - HTML
  - CSS
  - JS
  - изображения
  - шрифты
  - iframe
  - видео

```
Начало загрузки
↓
HTML получен
↓
DOM построен
↓
DOMContentLoaded
↓
Загрузка картинок
↓
Загрузка шрифтов
↓
load
```

## что используется чаще

- практически всегда DOMContentLoaded

- потому что обычно не нужно ждать картинки, видео, шрифты, нужен только DOM
