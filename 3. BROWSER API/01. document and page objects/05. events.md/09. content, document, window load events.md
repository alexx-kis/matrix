# content, document, window load events

- это события, связанные с жизненным циклом страницы
- они позволяют понять:
  - когда построился DOM
  - когда загрузились стили
  - когда загрузились картинки
  - когда пользователь уходит со страницы
  - когда изменилось состояние документа

## главная проблема

```html
<body>
  <button id="saveButton">Save</button>
</body>
```

```js
const saveButtonElement = document.querySelector('#saveButton');

console.log(saveButtonElement);
```

- если скрипт выполнится раньше, чем браузер построит DOM -> null

- поэтому исторически появилось событие загрузки страницы

## DOMContentLoaded

- срабатывает, когда HTML загружен -> DOM построен

```js
document.addEvenListener('DOMContentLoaded', () => {
  console.log('DOM ready');
});
```

- после этого можно безопасно работать с DOM

## что не ждет DOMContentLoaded

- изображения
- видео
- аудио
- iframe

## load

- срабатывает, когда загружено всё:
  - HTML
  - CSS
  - JS
  - изображения
  - шрифты
  - iframe
  - видео

```
Начало загрузки
↓
HTML получен
↓
DOM построен
↓
DOMContentLoaded
↓
Загрузка картинок
↓
Загрузка шрифтов
↓
load
```

## что используется чаще

- практически всегда DOMContentLoaded

- потому что обычно не нужно ждать картинки, видео, шрифты, нужен только DOM

## современная реальность

- сегодня часто пишут

```html
<script defer src="main.js"></script>
```

### что делает defer

- HTML продолжает парситься -> DOM строится -> DOM готов -> выполняется скрипт

- поэтому во многих проектах `DOMContentLoaded` вообще не используется

- можно сразу писать:

```js
const saveButtonElement = document.querySelector('#saveButton');
```

## readystatechange

- менее известное событие

- документ имеет состояние:
  - loading
  - interactive
  - complete

- можно отслеживать изменения

```js
document.addEventListener('readystatechange', () => {
  console.log(document.readyState);
});
```

## состояния

### loading

- документ еще загружается
- HTML еще читается

### interactive

- DOM уже построен
- почти соответствует DOMContentLoaded

### complete

- все загружено
- почти соответствует load

### beforeunload

- срабатывает перед уходом пользователя со страницы
- например:
  - закрытие вкладки
  - обновление страницы
  - переход на другой URL

```js
window.addEventListener('beforeunload', () => {
  console.log('Leaving page');
});
```

#### для чего используется

- самый частый кейс – есть несохраненные данные
- например, форма заполнена, но пользователь пытается закрыть страницу
- браузер может показать предупреждение

```js
window.addEventListener('beforeunload', (event) => {
  event.preventDefault();
  event.returnValue = '';
});
```

- современные браузеры сами решают:
  - показывать диалог
  - не показывать
  - какой текст отображать

### unload

- срабатывает, когда документ выгружается
- исторически использовался для:
  - аналитики
  - очистки ресурсов
  - логирования

```js
window.addEventListener('unload', () => {
  console.log('unloaded');
});
```

- на сегодня `unload` считается устаревшим, потому что:
  - не всегда вызывается
  - плохо работает на мобильных устройствах
  - может ломать производительность

### pagehide

- современная альтернатива `unload`

```js
window.addEventListener('pagehide', () => {
  console.log('hidden');
});
```

- используется браузерами гораздо надежнее

### visibilitychange

- очень полезное современное событие
- срабатывает, когда вкладка:
  - стала активной
  - стала неактивной

```js
document.addEventListener('visibilitychange', () => {
  console.log(document.visibilityState);
});
```

- возможные значения:
  - `visible`
  - `hidden`

- применение:
  - остановка анимаций
  - пауза видео
  - пауза запросов
  - экономия батареи

## порядок событий загрузки

```
HTML начал загружаться
↓
readyState = loading
↓
DOM построен
↓
DOMContentLoaded
↓
readyState = interactive
↓
Все ресурсы загружены
↓
load
↓
readyState = complete
```
