# client vs offset

- когда работаешь с DOM, очень часто нужно знать:
  - ширину и высоту элемента
  - размеры контента
  - размеры вместе с рамками
  - координаты элемента

- для этого браузер предоставляет несколько групп свойств:

```js
element.clientWidth;
element.clientHeight;

element.offsetWidth;
element.offsetHeight;

element.scrollWidth;
element.scrollHeight;
```

## clientWidth и clientHeight

- возвращают размеры видимой внутренней области элемента
- включают:
  - content
  - padding

- не включают:
  - border
  - margin
  - scrollbar (и вертикальный, и горизонтальный)

```html
<div class="panel"></div>
```

```css
.panel {
  width: 300px;
  height: 200px;

  padding: 20px;
  border: 10px solid black;
}
```

```
┌──────────────────────────────┐
│ border 10px                  │
│  ┌────────────────────────┐  │
│  │ padding 20px           │  │
│  │                        │  │
│  │ content                │  │
│  │                        │  │
│  └────────────────────────┘  │
└──────────────────────────────┘
```

- если `box-sizing: content-box;`, то `panel.clientWidth` вернет 300 + 20 + 20 === 340
- потому что учитываются content + left padding + right padding

## offsetWidth и offsetHeight

- возвращают размеры элемента, как объекта на странице
- включают:
  - content
  - padding
  - border

- не включают:
  - margin

- для того же элемента получим 300 + 20 + 20 + 10 + 10 === 360

## главное отличие

- для большинства элементов offsetWidth = clientWidth + leftBorder + rightBorder

```
offsetWidth
┌────────────────────────────┐
│ border                     │
│ ┌────────────────────────┐ │
│ │ clientWidth            │ │
│ │                        │ │
│ └────────────────────────┘ │
└────────────────────────────┘
```

- то есть offsetWidth шире clientWidth на величину рамок

## что происходит со scrollbar

```css
overflow-y: scroll;
```

- появилась вертикальная полоса прокрутки

### clientWidth:

- ширина scrollbar вычитается

### offsetWidth:

- ширина scrollbar входит в расчет

## как узнать ширину scrollbar?

```js
const scrollbarWidth = element.offsetWidth - element.clientWidth; // если рамок нет

const scrollbarWidth = element.offsetWidth - element.clientWidth - borderLeft - borderRight; // если рамки есть
```

## почему значения – целые числа

- для clientWidth и offsetWidth браузер возвращает `125`, а не `125.37`, то есть результат округляется до целого числа

- если нужны дробные значения, используют:

```js
element.getBoundingClientRect();
```

## offsetTop и offsetLeft

- показывают смещение элемента относительно его offsetParent

```html
<div class="wrapper">
  <div class="box"></div>
</div>
```

```css
.wrapper {
  position: relative;
}

.box {
  position: absolute;
  top: 100px;
  left: 50px;
}
```

```js
console.log(boxElement.offsetTop); // 100
console.log(boxElement.offsetLeft); // 50
```

## clientTop и clientLeft

- возвращают толщину рамки
- на практике используются редко
