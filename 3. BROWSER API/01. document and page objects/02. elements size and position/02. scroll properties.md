# scroll properties

- client* и offset* свойства измеряют видимую часть элемента

- но очень часто содержимого внутри элемента больше, чем помещается на экран или в контейнер

```html
<div class="article">Очень длинный текст...</div>
```

```css
.article {
  width: 300px;
  height: 200px;
  overflow: auto;
}
```

- если текста много, то появится прокрутка

- возникают вопросы:
  - как узнать полный размер содержимого?
  - как узнать, насколько пользователь уже прокрутил элемент?
  - как программно прокрутить элемент?

- для этого существует группа свойств `scroll*`

```js
element.scrollWidth;
element.scrollHeight;

element.scrollLeft;
element.scrollTop;
```

## scrollWidth

- показывает полную ширину содержимого элемента
- даже ту часть, которая сейчас скрыта

- scrollWidth включает:
  - content
  - padding

- не включает:
  - border
  - margin

- очень похоже на `clientWidth`
- но есть разница – clientWidth -> только видимая область, scrollWidth -> весь контент

## scrollHeight

- полный размер содержимого по высоте, даже если часть скрыта прокруткой

## как понять, есть ли прокрутка?

```js
if (element.scrollHeight > element.clientHeight) {
  console.log('есть вертикальная прокрутка');
}
```

## scrollTop

- показывает, насколько элемент уже прокручен по вертикали

```
Начало страницы
|
|
|
|
Текущая позиция
|
|
|
Конец страницы
```

```js
console.log(element.scrollTop);
```

- вернет количество пикселей от верхней границы

## scrollLeft

- показывает, насколько элемент прокручен по горизонтали

## можно ли менять `scrollTop` ?

- да, свойства scrollTop и scrollLeft не только читаются, но и изменяются

## прокрутка вниз

```js
element.scrollTop = 500;
```

- браузер сразу прокрутит элемент

### вернуться в начало

```js
element.scrollTop = 0;
```

### пример кнопки "наверх"

```js
buttonToTop.addEventListener('click', () => {
  document.documentElement.scrollTop = 0;
});
```

## прокрутка до конца

- используется в:
  - чатах
  - логах
  - комментариях
  - сообщениях

```js
element.scrollTop = element.scrollHeight;
```

- пример:

```js
messageContainer.scrollTop = messageContainer.scrollHeight;
```

- после добавления нового сообщения контейнер автоматически прокрутится вниз

## как понять, что пользователь докрутил до конца

```js
const reachedBottom = element.scrollTop + element.clientHeight >= element.scrollHeight;
```

- используется для:
  - бесконечной подгрузки данных
  - активации кнопки "принять условия"
  - аналитика

## получение позиции страницы

- у документа тоже есть прокрутка

- для всей страницы обычно используют:

```js
document.documentElement.scrollTop; // насколько пикселей прокручена страница
```

## window.scrollY и window.scrollX

- это современный вариант для `document.documentElement.scrollTop` и `document.documentElement.scrollLeft`
