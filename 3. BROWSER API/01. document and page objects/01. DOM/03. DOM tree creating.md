# DOM tree creating

- после того, как браузер получает HTML-документ, он начинает строить DOM Tree
- это один из самых важных процессов в работе браузера

## зачем браузер создаёт DOM Tree

- HTML — это просто текст:

```html
<button>Click</button>
```

- браузеру неудобно работать с текстом, поэтому он создаёт объектную структуру:

```js
{
  tagName: 'BUTTON',
  textContent: 'Click'
}
```

- теперь JS может обращаться к элементу:

```js
document.querySelector('button');
```

- изменять его:

```js
buttonElement.textContent = 'Send';
```

- удалять:

```js
buttonElement.remove();
```

- всё это возможно благодаря DOM-дереву

## Как происходит создание DOM

- упрощённо процесс выглядит так:

```
HTML
 ↓
Parsing
 ↓
DOM Tree
```

### щаг 1. браузер читает HTML

- например:

```html
<body>
  <h1>Hello</h1>
</body>
```

- чтение происходит сверху вниз

### шаг 2. токенизация (tokenization)

- HTML разбивается на токены:

```html
<body>
  <h1>Hello</h1>
</body>
```

- токен — это отдельная смысловая единица

### шаг 3. создание узлов

- для каждого тега создаётся DOM-узел

```
<body>
↓
Body Node

<h1>
↓
H1 Node

Hello
↓
Text Node
```

### Шаг 4. Построение дерева

- браузер связывает узлы между собой

- получается:

```
Document
│
└── body
    │
    └── h1
        │
        └── "Hello"
```

## корневой узел

- У любого DOM-дерева есть корень: `Document`

- это объект: `document`

- через него начинается вся работа с DOM

## автоматическое исправление HTML

- браузер умеет исправлять ошибки в HTML.

```html
<html>
  <body>
    <p>Hello</p>
  </body>
</html>
```

- закрывающего </p> нет, браузер автоматически достроит DOM:

```html
<p>Hello</p>
```

- поэтому DOM может отличаться от исходного HTML

## пропущенные html, head и body

- даже если написать:

```
<h1>Hello</h1>
```

- браузер создаст полноценное дерево:

```
Document
│
└── html
    │
    ├── head
    │
    └── body
         │
         └── h1
```

- то есть некоторые узлы браузер создаёт автоматически

## элементы и текст — разные узлы

- рассмотрим:

```html
<h1>Hello</h1>
```

- многие думают, что есть только один узел, на самом деле два:

```
h1
└── "Hello"
```

- `h1` — Element Node
- `"Hello"` — Text Node

- текст внутри элемента тоже является узлом DOM

## когда DOM Tree готов

- после завершения построения дерева браузер генерирует событие: `DOMContentLoaded`

- пример:

```js
document.addEventListener('DOMContentLoaded', () => {
  console.log('DOM готов');
});
```

- это означает: "HTML разобран DOM построен можно безопасно искать элементы через document

## что важно помнить для собеседования

### что такое DOM Tree?

- DOM Tree — это объектное древовидное представление HTML-документа, которое браузер создаёт в памяти.

### из чего строится DOM Tree?

- из HTML-разметки после её парсинга.

### что является корнем дерева?

- `document`

### являются ли текстовые данные узлами?

- да

```html
<h1>Hello</h1>
```

- создаёт:

Element Node (h1)

Text Node ("Hello")

### Может ли DOM отличаться от исходного HTML?

- да

- Браузер может:

- автоматически закрывать теги
- добавлять `html`, `head`, `body`
- исправлять некоторые ошибки разметки

- поэтому DOM Tree — это результат работы HTML-парсера, а не просто копия исходного HTML-кода
