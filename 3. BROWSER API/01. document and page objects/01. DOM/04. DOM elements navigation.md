# DOM elements navigation

- после того как браузер построил DOM-дерево, JS может перемещаться по нему: получать родительские элементы, дочерние элементы, соседние элементы итд

- это называется DOM navigation

```html
<body>
  <div id="container">
    <h1>Title</h1>
    <p>Text</p>
    <button>Click</button>
  </div>
</body>
```

- DOM-дерево будет выглядеть примерно так:

```
body
└── div#container
    ├── h1
    ├── p
    └── button
```

- навигация позволяет перемещаться между этими узлами

## почему это важно

- очень часто спрашивают:
  - как получить родителя элемента
  - как получить детей элемента
  - как перейти к соседнему элементу
  - чем отличается `children` и `childNodes`
  - чем отличаются `nextSibling` и `nextElementSibling`

- это базовые знания DOM

## два типа навигации

- в DOM есть два набора свойств:

### навигация по всем узлам

- работает со всеми типами узлов:
  - элементами
  - текстом
  - комментариями

```js
node.parentNode;
node.childNode;
node.firstChild;
node.lastChild;
node.nextSibling;
node.previousSibling;
```

### навигация только по элементам

- игнорирует текст и комментарии

```js
element.parentElement;
element.children;
element.firstElementChild;
element.lastElementChild;
element.nextElementSibling;
element.previousElementSibling;
```

- на практике чаще используют именно этот набор

## parentNode

- возвращает родительский узел

```html
<div id="wrapper">
  <button id="submit-button">Send</button>
</div>
```

```js
const submitButton = document.getElementById('submit-button');

console.log(submitButton.parentNode);
```

- результат: `<div id="wrapper">`

## parentElement

- возвращает родительский элемент

```js
const submitButton = document.getElementById('submit-button');

console.log(submitButton.parentElement);
```

- результат тот же: `<div id="wrapper">`

## разница между parentNode и parentElement

- обычно одинаково
- но `parentNode` может вернуть любой тип узла, например:

```js
console.log(document.documentElement.parentNode); // Document (потому что родитель <html> - объект `document`)

console.log(document.documentElement.parentElement); // null (потому что `document` не является HTML-элементом)
```

## childNodes

- возвращает все дочерние узлы

```html
<ul id="menu">
  <li>Home</li>
  <li>About</li>
</ul>
```

- фактически браузер видит

```html
<ul id="menu">
  [перенос строки]
  <li>Home</li>
  [перенос строки]
  <li>About</li>
  [перенос строки]
</ul>
```

- поэтому:

```js
const menu = document.getElementById('menu');

console.log(menu.childNodes);
```

### самая частая ошибка тут

- ожидать, что `menu.childNodes[0]` вернет первый `<li>`
- на самом деле `menu.childNodes[0]` часто оказывается текстовым узлом с переносом строки
- поэтому обычно используют `menu.children`, а не `childNodes`

## children

- возвращает коллекцию дочерних элементов

```html
<ul id="menu">
  <li>Home</li>
  <li>About</li>
  <li>Contacts</li>
</ul>
```

```js
const menu = document.getElementById('menu');

console.log(menu.children);
```

- результат: HTMLCollection(3)

```
0 → li
1 → li
2 → li
```

## firstChild и lastChild

- первый и последний дочерний узел

```js
const menu = document.getElementById('menu');

console.log(menu.firstChild);
console.log(menu.lastChild);
```

- это могут быть текстовые узлы

## firstElementChild и lastElementChild

- первый и последний дочерний элемент

```js
const menu = document.getElementById('menu');

console.log(menu.firstElementChild);
console.log(menu.lastElementChild);
```

- результат:

```html
<li>Home</li>
<li>Contacts</li>
```

## nextSibling и previousSibling

- следующий и предыдущий соседний узел

```html
<div id="card">
  <h2 id="title">Title</h2>
  <p>Description</p>
</div>
```

```js
const title = document.getElementById('title');

console.log(title.nextSibling);

const paragraph = document.querySelector('p');

console.log(paragraph.previousSibling);
```

- очень часто результат - Text Node, потому что между тегами есть перенос строки

## nextElementSibling и previousElementSibling

- следующий соседний элемент

```js
const title = document.getElementById('title');

console.log(title.nextElementSibling); // <p>Description</p>

const paragraph = document.querySelector('p');

console.log(paragraph.previousElementSibling); // <h2>Title</h2>
```

## children - это не array

- многие думают:

```js
const menuItems = menu.children;

menuItems.map(...) // menuItems.map is not a function
```

- потому что `menu.children` возвращает HTMLCollection, а не массив

### преобразование в массив

```js
const menuItems = [...menu.children];
// или
const menuItems = Array.from(menu.children);

// теперь можно:
menuItems.map((item) => item.textContent);
```
