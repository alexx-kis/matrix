# DOM inspecting

- когда говорят DOM inspecting, обычно имеют в виду использование инструментов разработчика браузера (DevTools) для просмотра и исследования текущего DOM-дерева страницы в реальном времени

- это очень важный навык для фронтенд-разработчика, потому что позволяет увидеть не то, что написано в HTML-файле, а то, что браузер реально построил после выполнения JS

## что такое Live DOM Viewer

- пусть есть такой код:

```html
<body>
  <div id="root"></div>

  <script>
    const rootElement = document.getElementById('root');

    rootElement.innerHTML = '<h1>Hello</h1>';
  </script>
</body>
```

- в исходном HTML элемента `<h1>` нет
- но после выполнения JS браузер изменяет DOM:

```html
<body>
  <div id="root">
    <h1>Hello</h1>
  </div>
</body>
```

- DevTools показывает именно это актуальное состояние
- поэтому его называют live DOM viewer - просмотрщик живого DOM

- крч тут и так все понятно

## частый вопрос на собеседовании

- что показывает вкладка Elements в DevTools: исходный HTML или DOM? - вкладка Elements показывает текущее состояние DOM-дерева, которое существует в памяти браузера после обработки HTML и выполнения JS - это не обязательно исходный HTML-код страницы
