# console.log vs console.dir

## console.log

- предназначен для вывода значения в консоль

```js
const pageTitle = document.querySelector('h1');

console.log(pageTitle); // <h1>Hello World</h1>
```

## console.dir

- предназначен для просмотра объекта как JS-объекта со всеми его свойствами и методами

```js
const pageTitle = document.querySelector('h1');

console.dir(pageTitle);
```

- вывод:

```
HTMLHeadingElement
  id: ""
  className: ""
  innerText: "Hello World"
  ...
```

## главное различие

```html
<button id="save-btn">Save</button>
```

```js
const saveButton = document.querySelector('#save-btn');

console.log(saveButton); //

// <button id="save-btn">
//   Save
// </button>

// то есть отображение похоже на вкладку Elements в DevTools

console.dir(saveButton); //

// HTMLButtonElement
//    disabled: false
//    innerText: "Save"
//    id: "save-btn"
//    click()
//    focus()
//    blur()
//    ...
```

## почему это важно

- когда работаешь с DOM, часто нужно понять
  - какие свойства есть у элемента
  - какие методы доступны
  - как называется нужное поле

```js
const profileLink = document.querySelector('.profile-link');

console.dir(profileLink);
```

```
href
target
rel
click()
focus()
```
