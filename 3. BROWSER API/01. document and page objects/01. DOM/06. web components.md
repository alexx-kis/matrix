# web components

- позволяют создавать собственные HTML-теги со своим поведением, стилями и разметкой

```html
// например, вместо:

<div class="user-card">...</div>

// можно написать:

<user-card></user-card>

// и браузер будет понимать этот тег так же, как понимает

<button></button>
<input />
<select></select>
```

## зачем появились web-components

- до появления Web Components разработчики создавали UI-компоненты через библиотеки: React, Vue, Angular, Svelte

- но проблема была в том, что компонент React работает только внутри React

- например, `<UserCard />` невозможно просто вставить в обычную HTML-страницу
- Web Components решают эту проблему
- компонент создается на уровне браузера и может использоваться где угодно:

```html
<user-card></user-card>
```

## из чего состоят Web Components

- обычно под Web Components подразумевают 3 технологии:
  - Custom Elements
  - shadow DOM
  - HTML Templates

## 1. Custom Elements

- позволяют создавать собственные HTML-теги

```html
<user-card></user-card>
<app-header></app-header>
<custom-button></custom-button>
```

- создание своего элемента

```js
// создаем класс

class UserCard extends HTMLElement {
  constructor() {
    super();

    this.innerHTML = `
      <h2>Alex</h2>
    `;
  }
}

// затем регистрируем:
customElements.define('user-card', UserCard);
```

- теперь браузер знает новый тег `<user-card></user-card>`
- результат:

```html
<user-card>
  <h2>Alex</h2>
</user-card>
```

### почему имя должно содержать дефис

- это обязательное правило
- браузер резервирует обычные имена под свои будущие теги
- чтобы избежать конфликтов, пользовательские элементы обязаны содержать дефис

```html
// можно

<user-card></user-card>
<app-header></app-header>
<custom-button></custom-button>

// нельзя

<usercard></usercard>
<header></header>
<button2></button2>
```

### жизненный цикл Custom Element

- у элемента есть специальные методы
- они похожи на lifecycle hooks во фреймворках

```js
// constructor
// вызывается при создании объекта

class UserCard extends HTMLElement {
  constructor() {
    super();

    console.log('constructor');
  }
}

// connectedCallback()
// вызывается когда элемента попадает в DOM

class UserCard extends HTMLElement {
  connectedCallback() {
    console.log('added to DOM');
  }
}

// например
document.body.append(userCardElement); // 'added to DOM'

// disconnectedCallback()
// вызывается когда элемент удаляется из DOM

class UserCard extends HTMLElement {
  disconnectedCallback() {
    console.log('removed');
  }
}

userCardElement.remove(); // 'removed'

// attributeChangedCallback()
// срабатывает при изменении отслеживаемых атрибутов
// сначала нужно указать список наблюдаемых атрибутов

class UserCard extends HTMLElement {
  static observedAttributes = ['name'];

  attributeChangedCallback(attributeName, oldValue, newValue) {
    console.log(attributeName);
    console.log(oldValue);
    console.log(newValue);
  }
}

// теперь:

<user-card name='Alex'></user-card>;

// и

document.querySelector('user-card').setAttribute('name', 'John');

// дадут name Alex John
```

## 2. Shadow DOM

- самая важная часть Web Components

- позволяет изолировать:
  - HTML
  - CSS
  - JS
- от остальной страницы

### без Shadow DOM

- есть страница

```html
<h2>Hello</h2>
```

```css
h2 {
  color: red;
}
```

- любой `<h2>` станет красным

### проблема больших приложений

- представим компонент

```html
<user-card></user-card>
```

- внутри:

```html
<h2>Alex</h2>
```

- но где-то на сайте есть:

```css
h2 {
  color: red;
}
```

- стили случайно изменят компонент

### создание Shadow DOM

```js
class UserCard extends HTMLElement {
  constructor() {
    super();

    const shadowRoot = this.attachShadow({
      mode: 'open',
    });

    shadowRoot.innerHTML = `
      <h2>Alex</h2>
    `;
  }
}
```

- теперь внутри компонента появляется отдельное DOM-дерево
- схематично:

```
document
 └─ user-card
     └─ #shadow-root
         └─ h2
```

- обычные стили страницы туда не проникают
- например:

```css
h2 {
  color: red;
}
```

- не повлияет на

```
#shadow-root
    h2
```

### собственные стили внутри Shadow DOM

- можно писать стили прямо внутри компонента:

```js
shadowRoot.innerHTML = `
  <style>
    h2 {
      color: green;
    }
  </style>

  <h2>Alex</h2>
`;
```

- стили будут работать только внутри этого компонента, наружу они не утекут

### mode: open

```js
// чаще всего используют
this.attachShadow({
  mode: 'open',
});

// тогда можно получить доступ:

const componentElement = document.querySelector('user-card');

console.log(componentElement.shadowRoot);
```

### mode: closed

```js
this.attachShadow({
  mode: 'closed',
});

console.log(componentElement.shadowRoot); // null
```

- внутренности становятся скрытыми

## 3. Template

- позволяет хранить HTML-шаблон

```html
<template id="user-template">
  <h2>User</h2>
  <p>Description</p>
</template>
```

- браузер его не отображает

```js
// можно получить его содержимое:
const templateElement = document.querySelector('#user-template');

// скопировать:
const templateContent = templateElement.content.cloneNode(true);

// и вставить:
document.body.append(templateContent);
```

- теперь шаблон появится на странице

## комбинация Template + Shadow DOM

- самый популярный вариант

```html
<template id="profile-template">
  <style>
    h2 {
      color: blue;
    }
  </style>

  <h2>User profile</h2>
</template>
```

```js
class ProfileCard extends HTMLElement {
  constructor() {
    super();

    const shadowRoot = this.attachShadow({ mode: 'open' });

    const templateElement = document.querySelector('#profile-template');

    const templateContent = templateElement.content.cloneNode(true);

    shadowRoot.append(templateContent);
  }
}

customElements.define('profile-card', ProfileCard);
```

- использование:

```html
<profile-card></profile-card>
```

## Web Components не заменяют React

- потому что React решает гораздо больше задач:
  - управляет состоянием
  - виртуальный DOM
  - роутинг (через экосистему)
  - экосистему компонентов
