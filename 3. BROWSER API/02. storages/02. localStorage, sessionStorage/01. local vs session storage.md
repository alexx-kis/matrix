# local vs session storage

- и то и то - часть Web Storage API
- они позволяют хранить данные прямо в браузере в виде пар ключ-значение
- например:

```
theme → dark
token → abc123
language → en
```

- в отличие от cookie, эти данные:
  - не отправляются автоматически на сервер
  - доступны только JS
  - имеют значительно больший размер хранения

## зачем вообще нужны storage

- пусть есть сайт
- пользователь:
  - выбрал темную тему
  - сменил язык
  - свернул меню
  - ввел что-то в форму

- если обновить страницу, то без сохранения все исчезнет

- storage позволяет сохранить эти данные между действиями пользователя

## есть два типа хранения

```
Web Storage
│
├── localStorage
└── sessionStorage
```

- они имеют одинаковый API
- главное различие - время жизни данных

## localStorage

- данные сохраняются надолго

- даже если пользователь:
  - обновил страницу
  - закрыл вкладку
  - закрыл браузер
  - выключил компьютер
- после открытия сайта снова данные останутся

- данные удаляются только если:
  - пользователь очистил данные сайта
  - JS сам удалил их
  - браузер удалил их при очистке хранилища

## sessionStorage

- работает почти так же, но живет только пока открыта конкретная вкладка

```
открыли вкладку
↓
создали данные
↓
обновили страницу
↓
данные остались
↓
закрыли вкладку
↓
данные исчезли
```

## отличия в таблице

```
| Свойство                           | localStorage      | sessionStorage            |
| ---------------------------------- | ----------------- | ------------------------- |
| Хранится после обновления страницы | ✅                 | ✅                         |
| Хранится после закрытия вкладки    | ✅                 | ❌                         |
| Хранится после закрытия браузера   | ✅                 | ❌                         |
| Доступно между вкладками           | ✅ (одного origin) | ❌                         |
| Удаляется автоматически            | Нет               | Да (при закрытии вкладки) |
```

## что значит "между вкладками"

- допустим открыли сайт `example.com`

### localStorage

- вкладка 1: theme=dark
- открыли вторую вкладку этого же сайта
- там тоже можно прочитать theme=dark
- потому что localStorage общий для данного origin

### sessionStorage

- вкладка 1: cart = 5 товаров
- открыли вторую вкладку: cart = empty
- потому что каждая вкладка получает собственный sessionStorage

## что такое origin

- Storage работает не для всего браузера, а для конкретного origin

- origin - это комбинация protocol+domain+port

- например: `http://example.com` и `https://example.com` - это разные origin, следовательно, у них разные storage

## что обычно хранят в localStorage

- обычно туда кладут данные, которые должны пережить закрытие браузера
- например:
  - тему оформления
  - язык сайта
  - настройки интерфейса
  - недавно просмотренные элементы
  - состояние бокового меню
  - предпочтения пользователя

## что обычно кладут в sessionStorage

- то, что должно жить только во время текущей вкладки
- например:
  - шаг мастера регистрации
  - временные фильтры
  - состояние поиска
  - черновик формы
  - данные многошаговой формы

## токены

- хранить токены в localStorage и sessionStorage можно технически, но это менее безопасно из-за риска XSS
- для чувствительных токенов обычно предпочитают `HttpOnly` cookies

## производительность

- и localStorage и sessionStorage работают синхронно
- из-за этого не рекомендуется:
  - хранить очень большие объемы данных
  - часто выполнять запись в циклах
  - использовать storage как базу данных

- для больших объемов данных обычно используют `IndexedDB`
