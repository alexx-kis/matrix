# setting and getting of cookies

- существует 2 способа работы с cookie:
  1. через HTTP (сервер)
  2. через JS (браузер)

## 1. установка cookie через сервер

- это основной и самый правильный способ
- сервер отправляет специальный заголовок:

```http
Set-Cookie: sessionId=abc123
```

- например:

```http
HTTP/1.1 200 OK
Set-Cookie: sessionId=abc123
```

- после получения ответа браузер автоматически сохранит cookie

- при следующем запросе браузер автоматически добавит:

```http
Cookie: sessionId=abc123
```

- например:

```http
GET /profile HTTP/1.1
Cookie: sessionId=abc123
```

- поэтому сервер сможет узнать пользователя

## 2. установка cookie через JS

- JS работает через объект `document.cookie`

### создание cookie

```js
document.cookie = 'language=en';
```

- теперь будет существовать 2 cookie:
  - theme=dark
  - language=en

## важный нюанс

- cookie не работают как обычные переменные
- каждое присваивание добавляет или обновляет конкретную cookie

```js
document.cookie = 'theme=dark';

document.cookie = 'language=en'; // эта вторая строка не перезапишет первую
```

## получение cookie

- для чтения используется тот же объект `document.cookie`

- например:

```js
console.log(document.cookie); // theme=dark; language=en
```

## что возвращает document.cookie

- многие ожидают объект, но возвращается строка

```js
'theme=dark; language=en';
```

- то есть браузер не парсит cookie в объект

## разбор строки cookie

```js
const cookiesString = document.cookie;

console.log(cookiesString); // theme=dark; language=en; currency=eur

// чтобы получить отдельные значения:

const cookieParts = cookiesString.split('; ');

console.log(cookieParts);
// результат:

['theme=dark', 'language=en', 'currency=eur'];
```

## получение конкретной cookie

- например нужно получить 'language=en'

- обычно создают вспомогательную функцию

```js
function getCookieValue(cookieName) {
  const cookieItems = document.cookie.split('; ');

  for (const cookieItem of cookieItems) {
    const [currentName, currentValue] = cookieItem.split('=');

    if (currentName === cookieName) {
      return currentValue;
    }
  }

  return null;
}
```

- использование:

```js
const selectedLanguage = getCookieValue('language');

console.log(selectedLanguage); // en
```

## обновление cookie

- если cookie с таким именем уже существует: `document.cookie = 'theme=dark';`, то можно изменить значение: `document.cookie = 'theme=light';`

- старое значение будет заменено

## проверка cookie в браузере

- в chrome: DevTools → Application → Cookies

- там можно увидеть:
  - имя
  - значение
  - домен
  - путь
  - срок жизни
  - защитные флаги

- это самый удобный способ отладки

## почему `document.cookie` считается неудобным

- у него есть несколько проблем

### чтение

- возвращается строка, а не объект, приходится самостоятельно парсить данные

### запись

```js
// нельзя сделать так:
document.cookie.theme = 'dark';

// или так:
document.cookie.language = 'en';
```

- потому что `document.cookie` не объект

### изменение

- чтобы изменить одну cookie, приходится снова записывать всю строку

```js
document.cookie = 'theme=light';
```

## что используют в реальных проектах

- в чистом JS можно работать напрямую через `document.cookie`, но часто используют библиотеки:
  - js-cookie
  - universal-cookie

- они дают удобный API:

```js
Cookies.set('theme', 'dark');

Cookies.get('theme');

Cookies.remove('theme');
```

## что важно для React и Next

- нужно понимать, что cookie бывают 2 типов:
  1. доступные JS (можно через `document.cookie`)
  2. недоступные JS

  ```http
  Set-Cookie: sessionId=abc123; HttpOnly
  ```

  - такую cookie `document.cookie` не увидит

- это сделано для безопасности
