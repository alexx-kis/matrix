# third-party cookies

## first-party cookie

- это cookie, которую устанавливает сайт, который пользователь сейчас посещает
- например, пользователь открыл `https://shop.com`, сервер отвечает:

```http
Set-Cookie: sessionId=abc123
```

- эта cookie принадлежит shop.com

- она называется first-party cookie

## third-party cookie

- теперь представим, что страница содержит сторонний сервис
- например:
  - рекламный баннер
  - кнопку социальной сети
  - виджет
  - аналитический скрипт

- например, shop.com загружает рекламу с ads-network.com
- во время загрузки рекламы сервер может отправить

```http
Set-Cookie: adUserId=987654
```

- теперь браузер сохранит cookie для ads-network.com, хотя пользователь никогда напрямую не открывал этот сайт

## визуальная схема:

```
Пользователь
      │
      ▼
 shop.com
      │
      ├────────────► CSS
      │
      ├────────────► JS
      │
      ├────────────► image
      │
      └────────────► ads-network.com
                         │
                         ▼
              Set-Cookie: adUserId=987654
```

## зачем появились third-party cookies?

- главная причина - отслеживание пользователей между разными сайтами
- например, пользователь открывает news.com, на сайте загружается реклама от ads-network.com, рекламная сеть устанавливает `adUserId=12345`
- через час пользователь открывает sports.com, там используется та же рекламная сеть
- браузер отправляет:

```http
Cookie: adUserId=12345
```

- теперь рекламная сеть понимает: это тот же пользователь, которого мы уже видели на другом сайте

- затем пользователь открывает music.com, снова:

```http
Cookie: adUserId=12345
```

- получается, что рекламная сеть может собирать информацию о посещении множества сайтов

## для чего использовались third-party cookies?

- самые распространенные сценарии:
  - персонализированная реклама
  - отслеживание пользователей между сайтами
  - аналитика
  - измерение эффективности рекламных компаний

## почему возникли проблемы?

- с точки зрения приватности это оказалось очень спорным
- пользователь может даже не знать, что shop.com передает информацию сторонним сервисам
- в результате одна рекламная сеть способна составить подробный профиль интересов пользователя
  - какие сайты он посещает
  - чем интересуется
  - какие товары смотрит
  - что покупает

## что изменилось в последние годы?

- современные браузеры постепенно отказываются от поддержки third-party cookies

- например:
  - safari блокирует их по умолчанию уже несколько лет
  - firefox также блокирует большинство third-party cookies по умолчанию
  - chrome долгое время поддерживал их, но активно внедряет альтернативные механизмы защиты приватности и постепенно ограничивает их использование

- поэтому сегодня third-party cookies используются значительно реже, чем несколько лет назад

## как это влияет на frontend-разработку?

- во многих современных проектах вообще можно не работать с third-party cookies напрямую
- если проект использует:
  - рекламные платформы
  - системы аналитики
  - внешние виджеты
  - сервисы авторизации
- то часть работы может выполняться через third-party cookies или их современные аналоги

## first-party vs third-party

```
| First-Party                             | Third-Party                                |
| --------------------------------------- | ------------------------------------------ |
| Создаётся текущим сайтом                | Создаётся сторонним доменом                |
| Используется самим сайтом               | Используется внешним сервисом              |
| Обычно нужна для работы сайта           | Часто используется для рекламы и аналитики |
| Не позволяет автоматически отслеживать  | Может использоваться для отслеживания      |
| пользователя между разными сайтами      | пользователя на разных сайтах              |
```

## это означает, что third-party cookies исчезли?

- нет
- некоторые браузеры все еще поддерживают их в определенных сценариях, а многие сервисы перешли на другие технологии:
  - server-side tracking
  - first-party cookies
  - специальные API браузеров
  - другие способы измерения эффективности рекламы и аналитики
- то есть отрасль не отказалась от аналитики и рекламы - изменились механизмы их реализации
