# what is cookie

- это маленький текстовый фрагмент данных, который сайт сохраняет в браузере пользователя

- проще говоря:
  - сервер отправляет браузеру cookie
  - браузер сохраняет ее
  - при следующих запросах на тот же сайт браузер автоматически отправляет эту cookie обратно

- за счет этого сайт может "узнавать" пользователя между запросами

## зачем вообще появились cookie?

- HTTP - протокол без состояния (stateless)

- это означает, что каждый запрос независим от предыдущего

- например: `GET /profile`

- сервер получает запрос и не знает:
  - кто его отправил
  - был ли этот пользователь на сайте минуту назад
  - авторизован ли он

- для сервер это просто новый запрос
- cookie решают эту проблему

## пример без cookie

- представим интернет-магазин

- пользователь открыл сайт, добавил товар в корзину, перешел на другую страницу

- без cookie сервер не сможет понять, что это тот же самый пользователь -> козина потеряется

## пример с cookie

- после первого захода сервер говорит браузеру:

```http
Set-Cookie: sessionId=abc123
```

- браузер сохраняет `sessionId=abc123`
- теперь при каждом запросе браузер автоматически отправляет `Cookie: sessionId=abc123`
- сервер видит `sessionId=abc123` и понимает: это тот же пользователь, что был раньше

## где физически хранятся cookie

- в браузере
- каждый браузер имеет собственное хранилище cookies
- у разных браузеров набор cookies разный

## что может храниться в cookie

- технически это просто строка
- например:
  - `theme=dark`
  - `language=en`
  - `sessionId=abc123`
  - `cartId=789`

## обычно в cookie хранят

- идентификатор сессии (самый частый вариант)
- тему сайта (темная/светлая)
- язык
- токен авторизации (но здесь есть нюанс безопасности)

## cookie ≠ данные пользователя

- обычно хранится только идентификатор, а не весь профиль пользователя
- а уже по этому идентификатору сервер находит данные в базе

## как cookie участвует в запросах

1. пользователь открывает сайт: `GET /login`
2. сервер отвечает: `Set-Cookie: sessionId=abc123`
3. браузер сохраняет cookie
4. пользователь открывает другую страницу: `GET /profile`, браузер автоматически добавляет `Cookie: sessionId=abc123`
5. сервер получает

```
GET /profile
Cookie: sessionId=abc123
```

- и понимает, кто сделал запрос

## Cookie и JS

- важно понимать, что cookie появились намного раньше JS
- изначально они были механизмом взаимодействие `Сервер ←→ Браузер`
- только потом JS получил возможность работать с ними
- то есть cookie это не API JS, это часть веб-протокола HTTP

## Cookie vs Local Storage

```
| Cookie                               | Local Storage         |
| ------------------------------------ | --------------------- |
| Автоматически отправляется на сервер | Не отправляется       |
| Создана для HTTP                     | Создан для JS         |
| Маленький объем                      | Больший объем         |
| Может иметь срок жизни               | Живет пока не удалить |
| Может быть защищена флагами          | Нет HttpOnly          |
```

- главное отличие:
  - Cookie участвует в сетевых запросах
  - Local Storage существует только внутри браузера

## почему cookie до сих пор используются?

- хотя есть:
  - LocalStorage
  - SessionStorage
  - IndexedDB

- cookie все равно остаются основным способом хранения:
  - сессий
  - авторизации
  - серверного состояния

- причина: браузер сам прикладывает cookie к HTTP-запросам, ничего дополнительно делать не нужно
