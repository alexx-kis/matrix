# size restrictions

- cookie специально создавались для хранения небольших данных
- они никогда не предназначались для хранения больших объемов информации

- это одна из причин, почему существуют:
  - Local Storage
  - Session Storage
  - IndexedDB

## почему размер ограничен?

- важная особенность cookie: при каждом подходящем HTTP-запросе браузер автоматически отправляет их серверу

```http
GET /profile
Cookie: sessionId=abc123; theme=dark; language=en
```

- это происходит:
  - при загрузке страницы
  - при запросах fetch()
  - при загрузке изображений
  - при загрузке CSS
  - при загрузке JS
  - и для других запросов к тому же домену (если условия отправки cookie соблюдены)

- если бы cookie могли быть очень большими, каждый запрос содержал бы много лишних данных, что увеличивало бы сетевой трафик и замедляло работу сайта

## максимальный размер одной cookie

- стандарт рекомендует поддержку cookie размером не более 4096 байт (4 КБ)
- то есть безопасно считать примерно 4 КБ на одну cookie

- сюда входят:
  - имя (theme)
  - значение (dark)
  - атрибуте (Path, Secure, SameSite и др)

## ограничение на количество cookie

- кроме ограничения по размеру одной cookie, браузеры ограничивают и их количество
- точные значения зависят от браузера, но обычно можно ориентироваться на следующие цифры:
  - около 180-300 cookie на один домен
  - общее количество cookie в браузере также ограничено

- когда лимит достигается, браузер может автоматически удалить старые cookie, чтобы освободить место

### важно

- эти значения не стандартизированы, разные браузеры могут использовать разные лимиты и алгоритмы удаления

## что происходит при превышении лимита?

- если попытаться записать слишком большую cookie:

```js
document.cookie = 'veryLargeCookie=...';
```

- браузер может
  - не сохранить ее
  - обрезать значение (в некоторых реализациях)
  - проигнорировать установку

- поэтому нельзя рассчитывать, что огромная cookie всегда сохранится

## почему нельзя хранить большие данные в cookie?

- так как с каждым запросом передаются одни и те же данные
- если данные большие в cookie, то будет лишний сетевой трафик

## что обычно хранят в cookie

- небольшие значения:
  - тема
  - id сессии
  - язык интерфейса
  - запоминание пользователя

## что не стоит хранить

- большой JSON пользователя
- список товаров корзины целиком
- фотографии
- документы
- длинные массивы данных

- вместо этого лучше хранить идентификатор

## Cookie vs Local Storage по объему

```
| cookie                             | Local Storage                                       |
| ---------------------------------- | --------------------------------------------------- |
| ≈ 4 КБ на одну cookie              | обычно около 5–10 МБ на домен (зависит от браузера) |
| отправляется на сервер             | не отправляется                                     |
| предназначена для небольших данных | подходит для более крупных клиентских данных        |

```

## Как это связано с производительностью?

- представим два варианта

1. Cookie:

```
sessionId=abc123
```

- размер — несколько десятков байт
- практически не влияет на запросы

2. Cookie:

```
userPreferences={...3500 байт...}
```

- эти 3,5 КБ будут добавляться к каждому подходящему запросу
- если за одну страницу выполняется 50 запросов, это уже: 3500 × 50 = 175000 байт ≈ 171 КБ дополнительного трафика только из-за одной cookie
