# глобальный и независимый от регистра поиск

## 1. флаг `g` (global)

- означает - искать все совпадения, а не только первое

### без `g`

```js
const textGlobal = 'test test test';

const patternNoGlobal = /test/;

const resultNoGlobal = textGlobal.match(patternNoGlobal);

console.log(resultNoGlobal);
```

- результат:

```
["test"] // только первое совпадение
```

### С `g`

```js
const patternGlobal = /test/g;

const resultGlobal = textGlobal.match(patternGlobal);

console.log(resultGlobal);
```

- результат:

```
["test", "test", "test"]
```

### важно понимать

- без `g` → остановка на первом совпадении
- с `g` → поиск по всей строке

### особенность .exec() с `g`

```js
const patternExecGlobal = /test/g;

console.log(patternExecGlobal.exec('test test'));
console.log(patternExecGlobal.exec('test test'));
```

- результат:

```
"test" (первое)
"test" (второе)
```

### почему так?

- RegExp запоминает позицию (lastIndex)
- каждый вызов `.exec()` идёт дальше

## 2. флаг i (case-insensitive)

- игнорировать регистр (большие/маленькие буквы)

### без i

```js
const patternCase = /hello/;

console.log(patternCase.test('Hello')); // false
```

### с i

```js
const patternIgnoreCase = /hello/i;

console.log(patternIgnoreCase.test('Hello')); // true
```

### что происходит

```
hello = Hello = HELLO
```

- регистр больше не важен

## 3. Комбинирование флагов

- флаги можно объединять:

```js
const patternComboFlags = /hello/gi;
```

- означает:

```
g → найти все
i → игнор регистра
```

### пример

```js
const textCombo = 'Hello hello HELLO';

const resultCombo = textCombo.match(/hello/gi);

console.log(resultCombo);
```

- результат:

```
["Hello", "hello", "HELLO"]
```

### частая ошибка

```
"Hello hello".match(/hello/i);
```

- результат:

```
["Hello"]
```

- почему не два? - потому что нет g
