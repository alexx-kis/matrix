# экранирование символов

- проблема: в regexp есть специальные символы, которые имеют особое значение

- если нужно искать их, как обычные символы, то их нужно экранировать

## специальные символы в regexp

```
. ^ * + ? ( ) [ ] { } | \ /
```

- они не означают сами себя, у них есть особое поведение

## как экранировать

- для этого используется обратный слэш `\`

### пример: точка `.` означает любой символ

```js
const patternDot = /./;
```

- но если нужна именно точка:

```js
const patternRealDot = /\./;
```

### пример

```js
console.log('a.b'.match(/\./)); // "."
```

### пример: знак вопроса `?`

```js
const patternQuestion = /\?/; // ищем символ ?, а не "0 или 1 раз"
```

### пример: плюс `+`

```js
const patternPlus = /\+/; // ищем +, а не "один или больше"
```

### пример: скобки `()`

```js
const patternBrackets = /\(test\)/; // ищем буквально "(test)"
```

### Особый случай — \ (обратный слэш)

- чтобы найти `\`, нужно:

```js
const patternBackslash = /\\/;
```

- потому что: первый `\` — экранирует, второй — сам символ

## 🔴 ВАЖНО: двойное экранирование в `RegExp()`

- если использовать конструктор, нужно экранировать дважды:

```js
const patternConstructor = new RegExp('\\.');
```

- JS строка: `"\\."` → превращается в `\.`
- regexp потом понимает это как "экранированная точка"

### пример реальной ошибки

```js
const wrongPattern = new RegExp('\d+'); // ❌ ошибка
```

- `\d` внутри строки → интерпретируется неправильно

```js
const correctPattern = new RegExp('\\d+'); // правильно
```

## когда не нужно экранировать

- внутри `[]` некоторые символы теряют спец. смысл:

```js
const patternInside = /[.]/; // точка здесь - просто точка
```

- но есть исключения: `^` `-` `]`
