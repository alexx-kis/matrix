# fake multithreading and concurrency

## 1. multithreading (многопоточность)

- это когда программа выполняет несколько задач одновременно в разных потоках (threads)
- пример (не JS): один поток скачивает файлы, второй рисует UI, третий - считает данные

## 2. JS другой:

- однопоточный
- один call stack
- одна операция за раз

#### как JS делает вид, что он "одновременно всё умеет"?

## 3. Fake multithreading (псевдомногопоточность)

- это иллюзия многопоточности
- JS не выполняет задачи параллельно, но быстро переключается между ними за счёт:
  - Event Loop
  - очередей задач
  - async API (Web APIs)

### простая аналогия

- не несколько человек одновременно делают каждый свою задачу, а один человек делает несколько задач, возвращаясь то к одной, то к другой, и кажется, что он делает несколько дел параллельно

## 4. concurrency (конкурентность)

- это способность управлять несколькими задачами одновременно по времени, но не обязательно одновременно по выполнению

- важно различать:

```
Concurrency:	задачи "перекрываются" по времени
Parallelism:  задачи реально выполняются одновременно
```

- в JS есть concurrency, и нет настоящего parallelism

## 5. как JS реализует concurrency:

1. Web APIs
   - setTimeout
   - fetch
   - DOM events

2. Event Loop

3. Очереди
   - microtasks
   - macrotasks

### пример

```js
console.log('Start');

setTimeout(() => console.log('Timeout'), 1000);

fetch('https://example.com').then(() => console.log('Fetch done'));

console.log('End');
```

- что происходит:
  1. 'Start' -> сразу
  2. setTimeout -> уходит в Web API
  3. fetch -> уходит в Web API
  4. 'End' -> сразу

- пока таймер тикает, запрос выполняется, JS может делать другие вещи

- это и есть concurrency

## почему это fake

- потому что JS не делает 2 вещи одновременно, он делегирует работу наружу (Web APIs), потом обрабатывает результат

## где есть настоящий multithreading в JS

- Web Workers

```js
const workerExample = new Worker('worker.js');
```

- запускает код в другом потоке
- не блокирует основной поток

- но нет доступа к DOM
- общение через сообщения (postMessage)

## 8. связь с blocking code
- blocking code -> ломает concurrency
- Event Loop -> управляет очередями
- Microtasks -> приоритет
- Macrotasks -> обычные задачи

## 9. частый вопрос на собеседовании

"JS - многопоточный?"

- JS - однопоточный, но поддерживает concurrency через Event Loop и Web APIs. настоящая многопоточность возможна через Web Workers
