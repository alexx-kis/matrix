# копирование/клонирование объектов. указатели объектов

### пример

```js
let a = { value: 5 };
let b = a;
b.value = 10;

console.log(a.value); //10
```

- объекты в JS копируются по ссылке (reference), а не по значению, то есть переменная не хранит сам объект, она хранит ссылку (указатель, pointer) на место в памяти, где лежит объект

```
переменная a -> { value: 5 }
переменная b  ↗
```

- после `let b = a` происходит не копирование объекта, а копирование ссылки
- поэтому `b.value = 10` меняет тот же самый объект

## проверка, что две переменные указывают на один и тот же объект

```js
let a = { x: 1 };
let b = { x: 1 };

console.log(a === b); // false, потому что создаются два разных объекта, выглядящих одинаково, ссылки у них разные, поэтому a !== b
```

```js
let a = { x: 1 };
let b = a;

console.log(a === b); // true, потому что в переменной b ссылка на тот же объект { x: 1 }
```

## копирование объекта

### 1. через `Object.assign()`

```js
let copy = Object.assign({}, user);
```

### 2. через spread оператор

```js
let copy = { ...user };
```

- делают shallow copy (поверхностное копирование):
  - сами примитивные свойства копируются полностью
  - если в объекте есть вложенные объекты, то копируются ссылки, а не копии вложенных объектов

```js
let user3 = {
  name: 'Alex',
  address: {
    city: 'NY',
  },
};

let copy = { ...user };

copy.address.city = 'LA';

console.log(user.address.city); // LA, потому что при поверхностном копировании копируются значения примитивных свойств, а вложенные объекты копируются по ссылке. то есть свойство address в обоих объектах ссылается на один и тот же объект { city: "NY" }
```

### 3. через structuredCLone (глубокое копирование)

```js
let copy = structuredClone(user);
```

- создаёт копию объекта вместе с вложенными объектами, если они есть

```js
let user = {
  name: 'Alex',
  address: { city: 'NY' },
};

let copy = structuredClone(user);

copy.address.city = 'LA';

console.log(user.address.city); // NY
```

### 4. JSON hack

```js
let copy = JSON.parse(JSON.stringify(user));
```

⚠️ ограничения JSON hack:

1. работает для объектов без функций, `undefined`, `Symbol`

- JSON не умеет хранить:
  - функции, он их просто игнорирует
  - undefined игнорируется, превращается в `null` в массивах
  - `Symbol` не сериализуется, игнорируется

```js
let obj = {
  x: 1,
  y: undefined,
  z: function () {},
  s: Symbol('s'),
};

console.log(JSON.stringify(obj)); // {"x":1}
```

⭐️ _сериализация - процесс преобразования объекта в строку, чтобы её можно было сохранить, передать или отправить по сети, десериализация - наоборот_

```js
let jsonString = JSON.stringify(user); // сериализация
let obj = JSON.parse(jsonString); // десериализация
```

2. не копирует специальные объекты типа `Date`, `Map`, `Set`

- JSON не знает, что такое `Date`, `Map`, `Set` - это специальные объекты JS
- при сериализации `Date` превращается в строку, а `Map` и `Set` полностью игнорируются

```js
let obj = {
  d: new Date(),
  m: new Map([['a', 1]]),
  s: new Set([1, 2]),
};

console.log(JSON.stringify(obj)); //{"d":"2026-03-09T12:00:00.000Z"}
```

### 5. рекурсивное копирование вручную

- можно написать функцию, которая ходит по объекту и создаёт новые объекты для каждого вложенного свойства
- это гибко, но больше кода
