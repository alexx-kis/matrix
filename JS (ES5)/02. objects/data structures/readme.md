# структуры данных

## 1. array (массив)

- это последовательность элементов, доступ к которым можно получить по индексу

```js
[10, 20, 30];
```

➕:

- быстрый доступ по индексу
- простая структура

➖:

- вставка/удаление в середине медленнее
- элементы хранятся подряд в памяти

✅ используется для:

- списков, очередей, стэков, таблиц

## 2. linked list (связанный список)

- это узлы, которые ссылаются друг на друга через `next` (или `prev`)

➕:

- быстро вставлять и удалять узлы в середине списка

➖:

- медленный доступ к элементу по индексу, нужно идти по ссылкам

✅ используется для:

- очередей, реализации стэков, некоторых алгоритмов, где важны быстрые вставки/удаления

## 3. stack (стэк)

- это структура данных, где элементы добавляются и удаляются с одного конца
- принцип работы - last in, first out - последний добавленный элемент выходит первым

```
bottom
  ↓
 [10] - добавили первым
 [20] - добавили вторым
 [30] - добавили третьим (верхний)
  |
 top
```

- если мы делаем `pop()`, удалится 30, потому что он сверху

### основные операции стэка

- `push` - добавляет элемент на вершину
- `pop` - удаляет верхний элемент (и возвращает)
- `peek`/`top` - показывает верхний элемент без удаления
- `isEmpty` - проверяет, пуст ли стэк

### пример стэка на массиве

```js
const stack = [];

stack.push(10);
stack.push(20);
stack.push(30);

console.log(stack.pop()); // 30
console.log(stack); // [10, 20]
console.log(stack[stack.length - 1]); // peek -> 20
```

### применение стэка

- undo/redo в редакторах (последнее действие отменяется первым)
- вызовы функций (когда функция вызывает другую, она кладётся в стэк, а после завершения убирается)
- обход графа/дерева (например DFS - Depth First Search - можно реализовать через стэк)

### важная ассоциация

- стэк - стопка тарелок: положил тарелку сверху -> верхняя, забрал тарелку -> верхнюю

## 4. queue (очередь)

- это структура данных, которая работает по принципу first in -> first out, то есть первый пришёл -> первый ушёл

- пример из жизни - очередь в магазине

### основные операции

- `enqueue` - добавить элемент в конец (в rear)
- `dequeue` - удалить первый элемент (из front)
- `peek` - посмотреть первый элемент
- `isEmpty` - проверить, пустая ли очередь

### визуализация

```
enqueue(5) // [5];
enqueue(10) // [5, 10]
enqueue(15) // [5, 10, 15]
dequeue() // [10, 15]
```

### пример очереди на массиве

```js
const queue = [];
queue.push(10); // enqueue
queue.push(20);
queue.push(30);

console.log(queue.shift()); // dequeue -> 10
console.log(queue); // [20, 30]
```

### где используется

1. очередь задач, например, обработка событий. система обрабатывает их в порядке поступления:

```
event1 -> event2 -> event3
```

2. BFS (обход графа) - алгоритм Breadth First Search использует очередь, он проходит граф по уровням

3. очередь печати - первый документ печатается первым

```
document1
    ↓
document2
    ↓
document3
```

4. серверные очереди - когда сервер получает много запросов, они обрабатываются по очереди

```
request1
    ↓
request2
    ↓
request3
```

## 5. tree (дерево)

- это структура данных, где каждый элемент называется узлом (node) и может иметь потомков (children)
- основные понятия:
  - node (узел) - элемент дерева
  - root (корень) - верхний узел, от которого всё дерево начинается
  - parent (родитель) - узел, у которого есть узлы-потомки
  - child (потомок) - узел, у которого есть родитель
  - leaf (лист) - узел без потомков

```
        Root
       /    \
    Node1   Node2
    /   \
Node3   Node4
```

### структура узла в JS

- простейший узел дерева можно записать так

```js
const node = {
  value: 'A',
  children: [],
};
```

- пример дерева

```js
const tree = {
  value: 'root',
  children: [
    {
      value: 'node1',
      children: [
        {
          value: 'node3',
          children: [],
        },
        {
          value: 'node4',
          children: [],
        },
      ],
    },
    {
      value: 'node2',
      children: [],
    },
  ],
};
```

### визуализация

```
Root
├─ Node1
│  ├─ Node3
│  └─ Node4
└─ Node2
```

### основные операции с деревом

1. traversal (обход дерева)
   - существует 2 популярных способа:
     1. DFS (Deep First Search) - глубинный обход
        - идём вниз по одной ветке, пока не дойдём до листа
        - потом поднимаемся и идём по другой ветке

          ```js
          function traverseDFS(node) {
            console.log(node.value); // посещаем узел
            node.children.forEach((child) => traverseDFS(child));
          }

          traverseDFS(tree);

          // результат: node1, node3, node4, node2
          ```

     2. BFS (Breadth First Search)
        - идём по уровням, сначала все узлы на одном уровне, потом на следующем

          ```js
          function traverseBFS(root) {
            const queue = [root];
            while (queue.length > 0) {
              const current = queue.shift();
              console.log(current.value);
              current.children.forEach((child) => queue.push(child));
            }
          }

          traverseBFS(tree);

          // результат: node1, node2, node3, node4
          ```

### почему дерево важно

- DOM в браузере - дерево узлов
- меню и вложенные структуры - дерево
- файловая система - дерево папок
- алгоритмы: поиск, сортировка, обход итд

## 6. graph (граф)

- это структура данных, состоящая из
  - вершин (nodes, vertices) - элементы графа
  - рёбер (edges) - связи между вершинами

### визуальный пример

```
A --- B
|     |
C --- D
```

- вершины: A, B, C, D
- рёбра: AB, AC, BD, CD

### основные типы графов

1. направленный граф (directed graph)
   - рёбра имеют направления
   - пример A -> B

2. ненаправленный граф (undirected graph)
   - рёбра не имеют направления
   - пример: A - B

3. взвешенный граф (weighted graph)
   - рёбра имеют "стоимость" или "вес"
   - пример: A -(5)- B

4. невзвешенный граф (unweighted graph)
   - все рёбра равны

### представление графа в JS

1. через список смежности (adjacent list) - самый популярный

```js
const graph = {
  A: ['B', 'C'],
  B: ['A', 'D'],
  C: ['A', 'D'],
  D: ['B', 'C'],
};
```

- ключи объекта - вершина
- массив - соседние вершины

2. через матрицу смежности (adjacency matrix)

```js
// A = 0, B = 1, C = 2, D = 3

const adjacencyMatrix = [
  [0, 1, 1, 0], // A
  [1, 0, 0, 1], // B
  [1, 0, 0, 1], // C
  [0, 1, 1, 0], // D
];
```

- строки/столбцы - вершины
- `1` - есть ребро, `0` - нет ребра

### основные операции графа

- добавить вершину
- добавить ребро
- удалить вершину
- удалить ребро
- обход графа

### обход графа

1. DFS (Depth First Search) - глубинный поиск
   - идём по одному пути, пока на упрёмся в конец
   - потом возвращаемся и идём по следующему пути

```js
const dfs = (graph, start, visited = new Set()) => {
  console.log(start);
  visited.add(start);

  graph[start].forEach((neighbor) => {
    if (!visited.has(neighbor)) {
      dfs(graph, neighbor, visited);
    }
  });
};

dfs(graph, 'A');
```

2. BFS (Breadth First Search) - поиск в ширину
   - идём уровень за уровнем, используя очередь

```js
const bfs = (graph, start) => {
  const visited = new Set();
  const queue = [start];

  while (queue.length > 0) {
    const node = queue.shift();
    if (!visited.has(node)) {
      console.log(node);
      visited.add(node);
      graph[node].forEach((neighbor) => {
        queue.push(neighbor);
      });
    }
  }
};

bfs(graph, 'A');
```

### применение графов

- соц. сети - друзья, подписчики
- карты и маршруты - города и дороги
- задачи на зависимости - сборка проектов
- игры - карты, пути

## 7. hash table

- это структура данных, которая хранит пары ключ-значение и позволяет:
  - быстро добавлять значения по ключу
  - быстро искать значения по ключу
  - быстро удалять значения

### пример через объект

```js
const userAge = {
  Alice: 25,
  Bob: 30,
  Charlie: 28,
};

console.log(userAge['Alice']); // 25
userAge['David'] = 32;
delete userAge['Bob'];
```

### пример через Map

```js
const userMap = new Map();

userMap.set('Alice', 25);
userMap.set('Bob', 30);

console.log(userMap.get('Alice')); // 25
userMap.delete('Bob'); // удаляем
```

### основные операции hash table

- добавить
- получить
- проверить наличие
- удалить

### отличия object и Map

```
object                        Map

ключи - строки или символы    любые типы ключей
не гарантирует порядок        гарантирует порядок вставки
легко сериализовать           удобен для сложных ключей
```

### применение

- кеширование данных
- подсчёт частоты слов
- словари и маппинги
- быстрый поиск по ключу

### пример

```js
// посчитать, сколько раз встречается каждое слово в массиве

const words = ['apple', 'banana', 'apple', 'orange', 'banana', 'apple'];
const count = {};

for (const word of words) {
  if (count[word]) {
    count[word] += 1;
  } else {
    count[word] = 1;
  }
}

console.log(count); // { apple: 3, banana: 2, orange: 1 }
```
