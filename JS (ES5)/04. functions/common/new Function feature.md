# new Function

- это конструктор функций
- он позволяет создавать новую функцию динамически из строки кода, прямо во время выполнения, то есть можно создавать функции "на лету", передавая из тело, как текст

## синтаксис

```js
let func = new Function([arg1, arg2, ...], functionBody);
```

- результат - полноценная функция, которую можно вызвать, как обычную

```js
let sum = new Function('a', 'b', 'return a + b');
console.log(sum(2, 3)); // 5
```

- здесь мы создали функцию `sum`, которая принимает два аргумента `a` и `b` и возвращает из сумму

💡 важный момент: `new Function` не умеет видеть внешние локальные переменные, только глобальные или свои аргументы

```js
let x = 10;

function normalFunc() {
  return x; // видит x из внешнего scope
}

let dynamicFunc = new Function('return x'); // x не определена здесь
console.log(normalFunc()); // 10
console.log(dynamicFunc()); // ReferenceError: x is not defined
```

## использование

- динамическое создание функций из текстового кода (например, из пользовательского ввода или конфигурации)
- генераторы функций для шаблонного кода
- иногда в библиотечных конструкциях для "eval-подобного" поведения, но безопаснее, чем `eval`

```js
let operation = '+';
let func = new Function('a', 'b', `return a ${operation} b`);
console.log(func(5, 3)); // 8
```

- можно менять оператор динамически, создавая разные функции

## нюансы

1. **безопасность**: так же как `eval`, использование `new Function` с вводом пользователя может привести к уязвимостям
2. **Performance**: функции, созданные через `new Function`, компилируются во время выполнения - это медленнее обычных функций
3. **Scope**: нет доступа к локальным переменным, только к глобальным и своим аргументам

## сравнение с `eval`

```js
let code = '2 + 3';

console.log(eval(code)); // 5, использует текущий scope
console.log(new Function('return ' + code)()); // 5, но глобальный scope
```

- `eval` видит локальные переменные
- `new Function` - нет, но создаёт "чистую" функцию

✅ чистая функция - функция, которая не имеет доступа к локальным переменным того места, где она была создана, она работает только со своими аргументами и глобальными переменными и не может использовать внешние переменные из текущего scope

- но это НЕ то же самое, что "pure function" (одинаковый ввод -> одинаковый вывод, без побочных эффектов)
