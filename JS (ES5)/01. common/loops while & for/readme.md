# циклы while и for

## 1 цикл while

- выполняет инструкцию, пока условие истинно, затем завершает

```js
while (condition) {
  // body
}
```

- условие проверяется перед каждой итерацией
- если условие изначально ложно, тело цикла не выполнится ни разу

⚠️ если не обновлять переменную счётчика, запустится бесконечный цикл

✅ используется, когда число итераций заранее неизвестно

```js
let i = 0;
while (i < 3) {
  console.log(i);
  i++;
}
```

## 2 цикл do...while

- повторяется пока заданное условие истинно
- в конце каждой итерации условие проверяется
- если условие ложно, цикл остановится

⚠️ тело выполняется хотя бы один раз, условие проверяется после выполнения

✅ используется, когда нужно хотя бы 1 раз выполнить код, затем проверить условие

```js
let i = 5;
do {
  console.log(i); // 5
  i++;
} while (i < 5); // выполнится один раз, несмотря на условие
```

## 3 цикл for

- цикл for повторяет действия, пока не произойдёт какое-либо специальное событие завершения цикла

```js
for (initialization; condition; afterthought) {
  // body
}
```

1. выполняется выражение инициализации (let i = 0) (его можно пропустить), оно создаёт один или несколько счётчиков
2. выполняется условие: если условие истинно, то выполняется тело цикла

⚠️ если условие пропущено, оно является true

3. выполняется тело цикла 4. обновляется шаг (afterthought) (i++), затем переходит к пункту 2

```js
for (let i = 0; i < 5; i++) {
  console.log(i);
}

// Можно пропустить части
let j = 0;
for (; j < 3; ) {
  console.log(j);
  j++;
}
```

- оператор `break` заканчивает цикл (ближайший, не внешний, если есть вложенность)
- оператор `continue` пропускает итерацию и переходит дальше

```js
for (let i = 0; i < 5; i++) {
  if (i === 2) continue; // пропустить 2
  if (i === 4) break; // выйти на 4
  console.log(i);
}
```

✅ используется, когда известен диапазон/шаг итераций, удобно инициализировать

## различия

- while и for: условие проверяется до итерации, do...while: условие проверяется после (поэтому он выполняется всегда хотя бы один раз)

## var и let в циклах

- var имеет функциональную или глобальную область видимости, то есть переменная видна всюду в функции
- let имеет блочную область видимости, то есть переменная видна только внутри {}, где она объявлена

```js
for (var i = 0; i < 3; i++) {
  console.log(i); // 0,1,2
}
console.log(i); // 3 — переменная i доступна вне цикла

for (let j = 0; j < 3; j++) {
  console.log(j); // 0,1,2
}
console.log(j); // Ошибка: j is not defined — переменная видна только внутри цикла
```

```js
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Вывод: 3,3,3 — все замыкания используют одну переменную i
```

1️⃣ Как работает цикл

1. var i = 0 — объявляется переменная i в функциональной или глобальной области видимости (не в блоке цикла!).

2. Условие i < 3 проверяется. Пока оно true, выполняется тело цикла.

3. В теле цикла вызывается setTimeout, который планирует функцию на выполнение через 100 мс, но немедленно не выполняет её.

⚠️ То есть на этом этапе цикл не ждёт setTimeout — он просто создаёт три таймера.

4. После каждого шага i++ увеличивает i. Цикл проходит три итерации:

   ● i = 0 → планируем setTimeout

   ● i = 1 → планируем setTimeout

   ● i = 2 → планируем setTimeout

После третьей итерации i становится 3, условие i < 3 становится false, цикл завершён.

2️⃣ Когда выполняются функции setTimeout

    ● Таймеры срабатывают примерно через 100 мс после завершения цикла.

    ● На момент выполнения всех трёх функций i уже равно 3!

    ● Все три функции обращаются к одной и той же переменной i (потому что var имеет общую область видимости), поэтому они все видят текущее значение i = 3.

```
Вывод:
console.log(i) → 3
console.log(i) → 3
console.log(i) → 3
```

3️⃣ Почему с let работает иначе

```js
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
```

● let создаёт новую переменную i для каждой итерации цикла (блочная область видимости).

● Каждая функция setTimeout "замыкается" на свою переменную i.

● Поэтому вывод: 0, 1, 2.

💡 Ключевой момент для собеседований:

● var → одна переменная на всю функцию, замыкания видят её текущее значение.

● let → новая переменная на каждую итерацию, замыкания видят своё значение.
