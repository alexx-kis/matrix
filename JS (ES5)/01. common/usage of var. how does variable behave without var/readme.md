# usage of var. how does variable behave without var

## 1️⃣ переменные с `var`

1. область видимости функции (function scope)
   - переменные, объявленные через `var` видны всей функции, даже если объявление находится внутри блока `if`, `for`, `{}` итд
   - то есть нет блочной области видимости - только функция или глобальная область

2. hoisting (подъём)
   - **объявление** переменной поднимается наверх своей функции (или глобальной области), но **инициализация (присвоение значения)** остаётся на месте

### пример

```js
function testVar() {
  console.log(a); // undefined, потому что объявление поднялось, а присвоение значения нет
  var a = 5;
  console.log(a); //5
}

testVar();
```

под капотом JS видит это так:

```js
function testVar() {
  var a; // объявление поднялось
  console.log(a); // undefined
  a = 5; // инициализация осталась на месте
  console.log(a); //5
}
```

✅ итог: `var` создаёт переменную внутри функции и поднимается наверх функции

## 2️⃣ переменные без `var` (или `let/const`)

- если написать переменную без ключевого слова:

```js
function testNoVar() {
  b = 10;
  console.log(b);
}

testNoVar(); //10
console.log(b); //10 - переменная стала глобальной
```

- JS автоматически создаёт переменную в глобальном объекте (window в браузере или `global` в NodeJS)ж
- это называется `неявная глобальная переменная`
- так делать опасно, потому что легко перезаписать глобальные переменные и сломать код

```js
function example() {
  if (true) {
    var x = 1;
    y = 2;
  }
  console.log(x); // 1 - видно в функции
  console.log(y); // 2 - тоже видно, но глобально
}

example();

console.log(y); // 2 - видно глобально
```

⚠️ если включен "use strict", то без var/let/const присвоение y = 2 вызовет ошибку

- переменная `y` становится глобальной **после вызова функции**
- то есть такой код вызовет ошибку:

```js
console.log(y); // ❌ ReferenceError: y is not defined

function example() {
  y = 2;
  console.log(y);
}

example();
```

- то есть `console.log(y);` выполняется до того, как `y` появилась, потому что она появился после **вызова** функции `example()`
