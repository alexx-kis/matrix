# рекурсия

- это когда функция вызывает сама себя внутри своего тела

- другими словами: функция решает задачу, разбивая её на меньшие подзадачи того же типа, пока не достигнет самой простой версии задачи (базового случая)

```js
function recursiveFunction(params) {
  if (baseCondition) {
    // ✅ базовый случай
    return result; // прекращаем рекурсию
  }
  return recursiveFunction(smallerParams); // 🔁 вызов самой себя
}
```

- базовый случай - условие, при котором рекурсия останавливается
- рекурсивный случай - часть функции, где она вызывает себя с более простой/меньшей задачей

## зачем нужна рекурсия

- рекурсия полезна, когда:
  1. задачу легко разбить на подзадачи того же типа
  2. мы хотим обойти сложные структуры данных (деревья, графы, вложенные массивы/объекты)

- без рекурсии такие задачи обычно решаются сложными циклами и дополнительными структурами (стеком, очередью)

## примеры рекурсии

### 1. факториал числа

- `n! = 1 * 2 * 3 * ... * n`

```js
function factorial(n) {
  if (n === 0) return 1; // базовый случай
  return n * factorial(n - 1); // рекурсивный случай
}

console.log(factorial(5)); // 120
```

- `factorial(5)` вызывает `factorial(4)`, потом `factorial(3)`, …

- рекурсия останавливается, когда `n === 0`

### 2. сумма вложенного массива

```js
function sumArray(arr) {
  if (arr.length === 0) return 0; // базовый случай
  return arr[0] + sumArray(arr.slice(1)); // рекурсивный случай
}

console.log(sumArray([1, 2, 3, 4])); // 10
```

- каждый вызов уменьшает массив на первый элемент, пока он не станет пустым

### 3. числа фибоначчи

```js
function fibonacci(n) {
  if (n === 0) return 0; // базовый случай
  if (n === 1) return 1; // базовый случай
  return fibonacci(n - 1) + fibonacci(n - 2); // рекурсивный случай
}

console.log(fibonacci(6)); // 8
```

- здесь рекурсия вызывает функцию дважды для каждой итерации
- важно правильно прописать базовые случаи, иначе будет бесконечная рекурсия

### 4. рекурсия для вложенных объектов

```js
const data = {
  value: 1,
  children: [
    { value: 2, children: [] },
    { value: 3, children: [{ value: 4, children: [] }] },
  ],
};

function sumValues(node) {
  let sum = node.value;
  for (let child of node.children) {
    sum += sumValues(child); // рекурсивный вызов
  }
  return sum;
}

console.log(sumValues(data)); // 10
```

- очень часто используется в деревьях и вложенных структурах

## Нюансы

1. базовый случай обязателен, иначе получится бесконечная рекурсия, и всё зависнет
2. рекурсия использует стек вызовов: каждый вызов функции добавляется в стек, при глубокой рекурсии можно столкнуться с переполнением стека
3. иногда рекурсия менее эффективна, чем цикл, особенно если много повторных вычислений (например, простая реализация Фибоначчи)
4. хвостовая рекурсия (tail recursion): если последний вызов функции - это рекурсивный вызов, многие движки JS могут оптимизировать её и не расходовать стек. пример:

```js
function factorialTail(n, acc = 1) {
  if (n === 0) return acc;
  return factorialTail(n - 1, n * acc);
}

console.log(factorialTail(5)); // 120
```
