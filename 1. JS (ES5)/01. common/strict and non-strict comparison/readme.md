# строгое и нестрогое сравнения

## нестрогое сравнение (==)

- сравнивает значения с приведение типов
- JS пытается привести оба значения к одному типу, а потом сравнить

```js
2 == '2'; // true, '2' приводится к числу
0 == false; // true, false приводится к 0
null == undefined; // true, особое правило
```

## строгое сравнение (===)

- сравнивает значение и тип
- приведение типов не выполняется

```js
2 === '2'; // false, разный тип
0 === false; // false, разный тип
null === undefined; // false, разные типы
```

## ловушки

1. `[] == false` // true

- пустой массив приводится к строке, потом к 0
- false приводится к 0
- 0 == 0

2. `[] === false` // false

- разные типы

3. `'' == 0` // true

- пустая строка приводится к 0

4. `'' === 0` // false

- разные типы

5.  любое сравнение с `NaN` -> false

6. null == undefined -> true

7. массивы приводятся к строкам

✅ всегда лучше === потому что нет неожиданных приведений типов -> меньше багов
