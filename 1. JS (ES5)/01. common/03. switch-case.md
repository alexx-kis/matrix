# switch-case конструкция

```js
switch (выражение) {
  case значение1:
    // выполняется, если выражение === значение1
    break;
  case значение2:
    // выполняется, если выражение === значение2
    break;
  default:
  // выполняется, если ни один case не подошёл
}
```

- аналог if-else, но когда условие - это сравнение выражения со значениями

```js
if (выражение === значение1) {
  // выполняется, если выражение === значение1
} else if (выражение === значение2) {
  // выполняется, если выражение === значение1
} else {
  // выполняется, если ни один case не подошёл
}
```

- сравнение проверяется через строгое (===) равенство
- break останавливает выполнение, иначе будет fall-through (или использовать return, если нужно что-то вернуть)
- default - аналог else (необязательный)

## когда использовать

- когда есть много вариантов одного выражения, и нужно выбрать только одно действие
- не использовать, если условие сложное

## плюсы

- читабельность при множестве вариантов
- лего добавить новый case
- можно объединять несколько значений на один блок:

```js
switch (fruit) {
  case 'apple':
  case 'pear':
    console.log('Это фрукт'); // apple и pear идут в один блок
    break;
  case 'carrot':
    console.log('Это овощ');
    break;
}
```

- то есть не надо писать

```js
fruit === 'apple' || fruit === 'pear' || //...
```


## минусы

- легко забыть break
- не подходит для сложных условий/диапазонов
- иногда читаемость хуже, если кейсов не много

## использование в функциях

- switch часто используют внутри функций, чтобы возвращать значение:

```js
function getDayName(dayNum) {
  switch(dayNum) {
    case 1: return 'Понедельник';
    case 2: return 'Вторник';
    case 3: return 'Среда';
    case 4: return 'Четверг';
    case 5: return 'Пятница';
    case 6: return 'Суббота';
    case 7: return 'Воскресенье';
    default: return 'Неизвестный день';
  }
}
```
