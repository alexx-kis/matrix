# конвертация объекта в примитив

## Object

- это тип данных, который хранит набор свойств
- это самый сложный тип данных

```js
const user = {
  name: 'Alex',
  age: 25,
};
```

## Primitive

- это простое значение, которое не является объектом
- в JS есть 7 примитивных типов:
  - string
  - number
  - boolean
  - null
  - undefined
  - symbol
  - bigint

- примитивы не имеют свойств и структуры, это просто одно значение

## преобразование

- JS иногда вынужден делать это автоматически

```js
const obj = {
  value: 10,
};

console.log(obj + 5); // [object Object]5
```

- преобразование происходит при математических операциях, операциях сравнения, конкатенации строк

## как JS превращает объект в примитив

- он пытается вызвать специальные методы:
  1. `Symbol.toPrimitive`
  2. `valueOf()`
  3. `toString()`

- если метода `Symbol.toPrimitive` существует, JS использует его

```js
const obj = {
  [Symbol.toPrimitive]() {
    return 10;
  },
};

console.log(obj + 5); // 15
```

- если метода `Symbol.toPrimitive` нет, JS пытается использовать `valueOf`

```js
const obj = {
  valueOf() {
    return 20;
  },
};

console.log(obj + 5); // 25
```

- если метода `valueOf()` нет, используется `toString()`

```js
const obj = {
  toString() {
    return 'hello';
  },
};

console.log('Say ' + obj);
```

- если ни одного метода нет (если они не описаны), у объекта всё равно есть методы `valueOf()` и `toString()`, потому что они приходят из `Object.prototype`

- в таком случае объект преобразуется в строку `"[object Object]"`

- метод `Object.prototype.toString()` возвращает строку `"[object Object]"`

## приколдэсы

- массив превращается в пустую строку `[] -> ''`

`console.log([] == ![])` // true

- потому что у оператора `!` приоритет, и он выполняется первым
- массив изначально truthly, поэтому `!` превращает его в false
- затем идёт сравнение и первый пустой массив превращается в пустую строку, которая falsy
- получается, что сравниваются два falsy значения, и в результате false == false -> true

- если оба операнда объекты, то сравнение идёт по ссылке, а не по значению
- поэтому `console.log([0] == [0]) // false`
