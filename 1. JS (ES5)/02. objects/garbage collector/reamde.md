# Garbage Collector

- это механизм в языке программирования, который автоматически освобождает память, когда она больше не используется программой

- проще говоря - это система, которая находит ненужные объекты в памяти и удаляет их

## почему нужна очистка памяти

- когда программа работает, она постоянно создаёт данные (объекты, массивы, строки, функции, DOM-элементы), они занимают память (RAM)

- если объект больше не нужен, но, если не удалить его из памяти, возникает проблема - Memory Leak (утечка памяти)

## как решали эту проблему раньше

- в старых языках (C / C++) программист сам управляет памятью
- он должен выделить память, а потом обязательно её освободить, иначе будет утечка памяти

## JS решает это автоматически

- он использует Garbage Collector, который
  1. следит за всеми объектами
  2. определяет - используются ли они
  3. удаляет ненужные

- поэтому разработчик не управляет памятью напрямую

## как JS понимает, что объект больше не нужен

- основная идея - `Reachability` (достижимость) - объект считается живым, если к нему можно добраться
- если добраться нельзя, GC читает его мусором

### пример

```js
let car = {
  brand: 'Tesla',
};
```

- объект `{ brand: 'Tesla' }` доступен через переменную `car`
- он достижим, значит не удаляется

- если убрать ссылку:

```js
car = null;
```

- объект `{ brand: 'Tesla' }` не имеет ссылок, он недостижим -> CG его удаляет из памяти

## root объекты

- GC начинает поиск от корневых объектов (roots)
- в JS это:
  - глобальные переменные
  - текущая функция
  - переменные в стеке вызовов
  - DOM

```
Roots
  ↓
Objects
  ↓
Other objects
```

- если к объекту есть путь от root - он жив

### пример

```js
let userAccount = {
  name: 'Alex',
};

let adminAccount = userAccount;
```

```
userAccount ──┐
              ↓
           { name: "Alex" }
              ↑
adminAccount ─┘
```

- объект имеет две ссылки, поэтому он жив

- удаляем одну ссылку:

```js
userAccount = null;
```

- теперь:

```
adminAccount → { name: "Alex" }
```

- объект всё ещё достижим -> GC его не удалит

- удаляем вторую ссылку

```js
adminAccount = null;
```

- теперь на объект `{ name: "Alex" }` нет ссылок, GC -> удаляет объект

## Mark and Sweep алгоритм GC

- это основной алгоритм работы Garbage Collector
- Garbage Collector делает 2 шага

### шаг 1 - mark (пометить)

- garbage collector:
  1. начинает с root-объектов
  2. проходит по всем ссылкам
  3. помечает объекты как "достижимые"

### шаг 2 - sweep (очистить)

- Garbage Collector проходит по памяти и удаляет всё, что не было помечено\

```
Object A  (marked)
Object B  (marked)
Object C  (unmarked)
Object D  (unmarked)
```

- после sweep:

```
Object C → delete
Object D → delete
```

### object reachability (достижимость объектов)

- объект считается живым, если к нему можно добраться от root-объекта через цепочку ссылок

### object pointers (references)

- в JS объекты передаются по ссылке, то есть переменная хранит указатель (reference) на объект в памяти

```js
let productInfo = { name: 'Phone' };

let productAlias = productInfo;
```

```
productInfo ──┐
              ↓
           { name: "Phone" }
              ↑
productAlias ─┘
```

- если удалить все ссылки на объект `productInfo = null`, то объект становится garbage

## Как GC понимает, что объект достижим?

- он проходит по ссылкам между объектами

```js
const companyData = {
  manager: {
    name: 'Alex',
  },
};
```

```
Root
 ↓
companyData
 ↓
manager
 ↓
{name: "Alex"}
```

### важный момент

- GC не смотрит на переменные, он смотрит на граф объектов
- то есть фактически память выглядит так:

```
Root
 |
 ↓
Object
 │
 ├── reference → Object
 └── reference → Object
```

### частый tricky-вопрос

```js
let nodeA = {};
let nodeB = {};

nodeA.link = nodeB;
nodeB.link = nodeA;

nodeA = null;
nodeB = null;
```

```
nodeA ↔ nodeB
```

- после:

```js
nodeA = null;
nodeB = null;
```

- но объекты всё ещё ссылаются друг на друга.

- удалит ли их GC?

- да, потому что нет пути от root

- GC удаляет даже циклические ссылки
