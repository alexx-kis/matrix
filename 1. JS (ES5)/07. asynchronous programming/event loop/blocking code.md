# blocking code (блокирующий код)

- это код, который останавливает выполнение всей программы, пока сам полностью не завершится

- пока этот код выполняется - ничего другое происходить не может

## почему это важно в JS

- JS - однопоточный (single-threaded) язык
- это значит:
  - у него один call stack (стек вызовов)
  - он может выполнять только одну операцию за раз

- поэтому если что-то зависло - зависает всё

## простой пример блокирующего кода

```js
console.log('Start');

for (let counterA = 0; counterA < 1000000000; counterA++) {
  // просто тяжелый цикл
}

console.log('End');
```

- что произойдёт:
  1. "Start" - выведется сразу
  2. потом начнётся огромный цикл
  3. пока он не закончится:
     - браузер зависает
     - интерфейс не реагирует
     - ничего не происходит
  4. только потом "End"

## почему это плохо

1. замораживается UI (кнопки не нажимаются, страница зависает)
2. плохой UX (пользователь думает, что сайт сломался)
3. Event Loop не может работать

## ещё пример

```js
function slowFunction() {
  while (true) {
    // бесконечный цикл
  }
}

slowFunction();

console.log('Hello');
```

- "Hello" никогда не выведется
- программа полностью заблокирована

## как это связано с Event Loop

- Event Loop - это механизм, который:
  - управляет очередями задач
  - решает, что выполнять дальше

- но, если стек занят blocking-кодом:
  - Event Loop не может вставить новые задачи
  - async код не выполнится

## визуально

```vb
Call Stack:
[ long loop running... ]

Event Loop:
❌ не может ничего сделать

Task Queue:
[ callback ] ← ждёт, но не выполнится
```

## важное правило

- пока стек (Call Stack) не освободится - ничего другого не выполнится

## антипример с асинхронностью

```js
setTimeout(() => {
  console.log('Timeout');
}, 0);

for (let counterB = 0; counterB < 1_000_000_000; counterB++) {}

console.log('Done');
```

- порядок:
  1. тяжёлый цикл (блокирует всё)
  2. "Done"
  3. только потом "Timeout"

- даже `setTimeout(..., 0)` не спасает от blocking code

## как с этим бороться

- разбивать задачи:
  - через setTimeout
  - через requestAnimationFrame

- использовать
  - Web Workers (вынести в другой поток)
