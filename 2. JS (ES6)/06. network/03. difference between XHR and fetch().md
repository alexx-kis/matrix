# difference between XHR and fetch

## что такое XHR

- XHR (XMLHttpRequest) – это встроенной объект браузера для выполнения HTTP-запросов к серверу из JS

- до появления fetch() практически все AJX-запросы выполнялись через XHR

```js
const userRequest = new XMLHttpRequest(); // 1. создаем объект

userRequest.open('GET', '/api/profile'); // 2. настраиваем запрос

userRequest.onload = () => {
  // 3. подписываемся на событие завершения
  console.log(userRequest.responseText);
};

userRequest.send(); // 4. отправляем запрос
```

## почему появился fetch?

- XHR имеет несколько проблем:

1. код получается многословным

2. не работает с Promise

- XHR появился задолго до Promise, поэтому используются события и коллбэки

3. менее удобная работа с данными

- например JSON приходится парсить вручную

```js
const profileData = JSON.parse(userRequest.responseText);
```

## Fetch

- в современных приложениях обычно используют:

```js
fetch('/api/profile')
  .then((response) => response.json())
  .then((profileData) => {
    console.log(profileData);
  });

// или

const response = await = fetch('/api/profile');
const profileData = await response.json();
```

## основные отличия XHR и Fetch

1. API

- XHR - события:

```js
request.onload = () => {};
request.onerror = () => {};
```

- Fetch - Promise:

```js
fetch('/api/profile')
  .then(...)
  .catch(...)

// или

await fetch('/api/profile')
```

2. читаемость

- XHR - код больше и сложнее
- Fetch - код меньше и проще читается

3. Работа с JSON

```js
// XHR:
const profileData = JSON.parse(userRequest.responseText);

// Fetch:
const profileData = response.json();
```

4. работа с async/await

- XHR напрямую не поддерживает `await some XHR()`
- чтобы использовать `await`, придется самостоятельно оборачивать XHR в Promise
- Fetch изначально создан для Promise

5. отмена запроса

- в XHR есть встроенный метод `request.abort()`
- в fetch используется специальный объект:

```js
const abortController = new AbortController();
fetch('/api/profile', {
  signal: abortController.signal,
});

abortController.abort();
```

- код сложнее, но более гибкий

6. отслеживание прогресса загрузки

- в XHR есть специальные события:

```js
request.onprogress = (event) => {
  console.log(event.loaded);
};
```

- удобно для загрузки файлов
- прогресс-баров

- в fetch долгое время встроенного удобного аналога не было
- для отслеживания прогресса приходится работать со Stream API, что заметно сложнее
- поэтому для некоторых сценариев загрузки файлов XHR до сих пор встречается

## Коротко для собеседования

- XMLHttpRequest (XHR) — старый браузерный API для выполнения HTTP-запросов

- Fetch — современный API для выполнения HTTP-запросов на основе Promise

- главные преимущества fetch:

1. поддержка Promise
2. поддержка async/await
3. более простой и читаемый код
4. удобная работа с JSON

- главное преимущество XHR:

1. встроенное отслеживание прогресса загрузки через события (onprogress).
