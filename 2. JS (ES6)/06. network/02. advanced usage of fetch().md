# advanced usage of fetch() (how to get binary data from body, how to get/set headers)

## что такое Headers

- заголовки – это дополнительная информация о запросе или ответе
- например:

```http
Content-Type: application/json
Authorization: Bearer token123
Accept-Language: en
```

- они не являются телом запроса или ответа
- можно представить HTTP-сообщение так:

```
Headers
↓
Body
```

## получение заголовков ответа

- допустим сервер вернул:

```http
Content-Type: application/json
Content-Length: 2048
```

- после запроса:

```js
fetch('/api/users').then((responseObject) => {
  console.log(responseObject.headers);
});
```

## объект Headers

- `response.headers` содержит специальный объект `Headers`
- получение конкретного заголовка:

```js
fetch('/api/users').then((responseObject) => {
  const contentTypeValue = responseObject.headers.get('Content-Type');

  console.log(contentTypeValue);
});
```

- результат

```
application/json
```

## проверка существования заголовка

```js
const hasContentTypeHeader = responseObject.headers.has('Content-Type'); // true/false
```

## отправка собственных заголовков

- для этого используется второй аргумент `fetch()`

```js
fetch('/api/users', {
  headers: {
    Authorization: 'Bearer token123',
  },
});

// несколько заголовков:

fetch('/api/users', {
  headers: {
    Authorization: 'Bearer token123',
    'Content-Type': 'application/json',
    'X-App-Version': '1.0.0',
  },
});
```

## самый популярный заголовок - Content-Type

- он говорит серверу, какой тип данных мы отправляем
- например, JSON:

```js
fetch('/api/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'Alex',
  }),
});
```

- без этого сервер может не понять, что пришел JSON

## заголовок Authorization

- очень часто используется для авторизации пользователя

```js
fetch('/api/profile', {
  headers: {
    Authorization: 'Bearer eyJhbGciOiJIUzI1NiIs...',
  },
});
```

- практически любой frontend-разработчик сталкивается с этим ежедневно

## получение бинарных данных

- до этого мы использовали `response.json()` или `response.text()`
- но что делать, если сервер прислал файл?
- например:
  - картинку
  - PDF
  - ZIP архив
  - Excel файл
- для этого существуют другие методы

## response.blob()

- самый популярный способ получить файл

```js
fetch('/images/logo.png')
  .then((responseObject) => {
    return responseObject.blob();
  })
  .then((imageBlob) => {
    console.log(imageBlob);
  });
```

- `Blob` – специальный объект, предоставляющий бинарные данные
- пример:

```
Blob {
  size: 52341,
  type: "image/png"
}
```

## отображение полученной картинки

- получили Blob:

```js
const imageBlob = await responseObject.blob();
```

- создаем временный URL:

```js
const imageUrl = URL.createObjectURL(imageBlob);
```

- теперь его можно использовать как обычную ссылку:

```js
imageElement.src = imageUrl;
```

### полный пример:

```js
async function loadAvatar() {
  const responseObject = await fetch('/images/avatar.png');

  const avatarBlob = await responseObject.blob();

  const avatarUrl = URL.createObjectURL(avatarBlob);

  document.querySelector('#avatar').src = avatarUrl;
}
```

## response.arrayBuffer()

- еще более низкоуровневый способ

```js
const binaryBuffer = await responseObject.arrayBuffer();
```

- получаем объект ArrayBuffer
- это просто набор байтов в памяти (условно)

```
[137, 80, 78, 71, ...]
```

- используется, когда нужно:
  - самостоятельно обрабатывать бинарные данные
  - работать с аудио
  - работать с видео
  - работать с криптографией
  - работать с файлами нестандартных форматов

### пример

```js
async function loadFileBytes() {
  const responseObject = await fetch('/report.pdf');

  const pdfBuffer = await responseObject.arrayBuffer();

  console.log(pdfBuffer);
}
```

## Blob vs ArrayBuffer

### Blob

- высокоуровневый объект

```js
const documentBlob = await responseObject.blob();
```

- используется, когда:
  - нужно скачать файл
  - показать картинку
  - сохранить PDF

### ArrayBuffer

- низкоуровневый объект

```js
const audioBuffer = await responseObject.arrayBuffer();
```

- используется, когда:
  - нужно читать байты
  - менять содержимое файла
  - выполнять вычисление над бинарными данными

- на практике чаще используется `response.blob()`, чем `response.arrayBuffer()`

## важный нюанс

- тело ответа можно прочитать только один раз

```js
// так нельзя:

const jsonData = await responseObject.json();
const textData = await responseObject.text();

// потому что поток уже был прочитан

// то же самое:

await responseObject.blob();
await responseObject.arrayBuffer();

// та же ошибка
```

- если нужно читать тело несколько раз, используют `responseObject.clone()`

```js
const responseCopy = responseObject.clone();

const jsonData = await responseObject.json();

const responseText = await responseCopy.text();
```
