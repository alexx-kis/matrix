# basic usage of fetch()

- `fetch()` - встроенная функция браузера для выполнения HTTP-запросов к серверу
- например:
  - получить список пользователей
  - отправить форму
  - загрузить товары интернет-магазина
  - получить настройки приложения

- до появления fetch часто использовали `XMLHttpRequest`

## самый простой запрос

```js
fetch('https://api.example.com/users');
```

- после выполнения браузер отправит HTTP GET-запрос по указанному адресу
- но результат мы пока никак не обрабатываем

## что возвращает fetch()

- `fetch()` всегда возвращает Promise
- поэтому результат нужно получать через `.then()` или `async/await`

```js
fetch('https://api.example.com/users').then((response) => {
  console.log(response);
});
```

## что такое `response`

- в callback приходи объект ответа от сервера

- примерно такой:

```js
Response {
  status: 200,
  ok: true,
  headers: ...
}
```

## почему нельзя сразу получить данные

- многие ожидают такое:

```js
fetch('https://api.example.com/users').then((response) => {
  console.log(response.data);
});
```

- но так не работает
- response содержит не сами данные, а поток данных (stream)
- поэтому нужно отдельно прочитать тело ответа

## получение JSON

```js
fetch('https://api.example.com/users')
  .then((response) => {
    return response.json();
  })
  .then((dataObject) => {
    console.log(dataObject);
  });
```

- `json()` тоже возвращает Promise
- поэтому появляется второй `.then()`

## проверка успешности запроса

```js
fetch('https://api.example.com/users')
  .then((response) => {
    if (!response.ok) {
      throw new Error('Request failed');
    }

    return response.json();
  })
  .then((usersList) => {
    console.log(usersList);
  })
  .catch((errorObject) => {
    console.log(errorObject);
  });
```

## получение обычного текста

- не все ответы приходят в JSON
- если сервер вернул обычный текст:

```js
fetch('https://api.example.com/message')
  .then((response) => response.text())
  .then((messageText) => {
    console.log(messageText);
  });
```

### самые популярные методы чтения ответа:

- `.json()`: JSON -> JS объект
- `.text()`: текст
- `.blob()`: файлы, картинки
- `.arrayBuffer()`: бинарные данные

## то же самое через `async/await`

```js
async function loadUsers() {
  const response = await fetch('https://api.example.com/users');

  const usersList = await response.json();

  console.log(usersList);
}
```

- с проверкой ошибок:

```js
async function loadProducts() {
  try {
    const response = await fetch('https://api.example.com/products');

    if (!response.ok) {
      throw new Error('Request failed');
    }

    const productsList = await response.json();

    console.log(productsList);
  } catch (errorObject) {
    console.error(errorObject);
  }
}
```

## что обычно спрашивают на собеседовании

1. что возвращает `fetch()`? - Promise, который зарезолвился в объект `Response`
2. что возвращает `response.json()` - тоже Promise
3. попадет ли HTTP 404 в `catch()`? - нет, при ответе "404 Not Found" Promise все равно будет успешным, нужно самостоятельно проверять
4. когда `fetch()` попадет в `catch()`? - когда произошла именно сетевая ошибка (нет интернета, ошибка DNS, соединение оборвалось, запрос был отменен)
