# default export

- есть 2 основных типа экспорта:

1. named export

```js
// mathUtils.js
export function sumNumbers(firstValue, secondValue) {
  return firstValue + secondValue;
}
```

- импорт:

```js
import { sumNumbers } from './mathUtils.js';
```

2. default export

```js
// mathUtils.js
export default function sumNumbers(firstValue, secondValue) {
  return firstValue + secondValue;
}
```

- импорт

```js
import sumNumbers from './mathUtils.js';
```

### главная проблема при default export

- имя при импорте можно менять как угодно

### почему это плохо

1. нет однообразия имён

- в большом проекте один и тот же модуль могут импортировать под разными именами, получается хаос

2. труднее искать по проекту

3. IDE хуже помогает

- когда есть named export IDE знает точное имя:
  - автодополнение
  - rename refactor
  - поиск usages
  - навигация

- работают стабильнее

4. легче случайно сломать импорт

- named export:

```js
import { generateToken } from './auth';
```

- если такого экспорта нет -> ошибка сразу

- при default export имя вообще не проверяется

5. named export лучше масштабируются

```js
export function openModalWindow() {}
export function closeModalWindow() {}
export function toggleModalWindow() {}

// импорт:

import { openModalWindow, closeModalWindow, toggleModalWindow } from './modalUtils.js';
```

- очень явно видно:
  - что используется
  - какие функции существуют
  - откуда они пришли

## почему default export вообще существует

- он удобен, когда модуль представляет:
  - одну главную сущность
  - один основной объект
  - один компонент

## почему многие избегают default export

- во многих командах правило:
  - использовать только named exports

- причины:
  - легко поддерживать
  - легче искать
  - легче рефакторить
  - меньше путаницы
  - лучше читается кодовая база
