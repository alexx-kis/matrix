# module namespace exports

- это связано с таким импортом

```js
import * as mathHelpers from './mathHelpers.js';
```

- здесь `mathHelpers` – это module namespace objects

## что такое module namespace object

- когда ты пишешь

```js
import * as mathHelpers from './mathHelpers.js';
```

- JS
  1. загружает модуль
  2. берёт все его exports
  3. складывает их в специальный объект

- например:

```js
// mathHelpers.js

export const sumValues = (leftNumber, rightNumber) => {
  return leftNumber + rightNumber;
};

export const multiplyValues = (leftNumber, rightNumber) => {
  return leftNumber * rightNumber;
};
```

- импорт:

```js
import * as mathHelpers from './mathHelpers.js';

console.log(mathHelpers);
```

- получим примерно:

```js
{
  sumValues: [Function],
  multiplyValues: [Function]
}
```

## почему это называется namespace

- потому что все exports лежат внутри одного пространства имён

- без namespace:

```js
import { sumValues, multiplyValues } from './mathHelpers.js';
```

- функции попадают прямо в текущую область видимости

- с namespace:

```js
import * as mathHelpers from './mathHelpers.js';
```

- теперь доступ идёт через объект:

```js
mathHelpers.sumValues(2, 3);
mathHelpers.multiplyValues(4, 5);
```

## зачем это нужно

### 1. избежать конфликтов имён

- например:

```js
import * as userMathUtils from './userMathUtils.js';
import * as adminMathUtils from './adminMathUtils.js';
```

- теперь одинаковые имена не конфликтуют:

```js
userMathUtils.calculateTotal();
adminMathUtils.calculateTotal();
```

### 2. удобная группировка

- иногда модуль содержит много функций

- вместо:

```js
import { createSlider, destroySlider, updateSlider, resetSlider } from './slider.js';
```

- можно:

```js
import * as sliderUtils from './slider.js';

sliderUtils.createSlider();
sliderUtils.resetSlider();
```

- код становится визуально понятнее

### 3. похоже на обычные объекты

- это удобно психологически:

```js
api.getUsers();
api.createUser();
api.deleteUser();
```

- выглядит, как работа с API-объектом

## важная особенность

- module namespace object - не обычный объект

- он:
  - readonly
  - автоматически синхронизируется с exports модуля
  - имеет специальные внутренние механизмы JS

## нельзя изменять namespace

```js
import * as mathHelpers from './mathHelpers.js';

mathHelpers.sumValues = 123;
```

- ошибка TypeError, потому что exports защищены

## live bindings

- module namespace хранит не копии значений, а ссылки на exports

- пример:

```js
// counterState.js

export let counterValue = 0;

export const incrementCounter = () => {
  counterValue++;
};
```

- импорт:

```js
import * as counterState from './counterState.js';

console.log(counterState.counterValue); // 0

counterState.incrementCounter();

console.log(counterState.counterValue); // 1
```

- хотя объект `counterState` не пересоздавался

## default export тоже попадает в namespace

```js
// logger.js

export default function logMessage() {
  console.log('log');
}

export const loggerVersion = '1.0';
```

- импорт:

```js
import * as loggerModule from './logger.js';
```

- получим:

```js
loggerModule.default;
loggerModule.loggerVersion;

// использование:

loggerModule.default();
```

## почему это редко используют с default export

- потому что выглядит странно:

```js
loggerModule.default();

// обычно делают:

import logMessage from './logger.js';
```

## частая ошибка

- пытаются делать так:

```js
import mathHelpers from './mathHelpers.js';
```

- но в модуле нет default export, а есть только:

```js
export const sumValues = ...
```

- тогда будет ошибка
- для namespace нужен именно:

```js
import * as mathHelpers
```

## главное различие

### named import

```js
import { sumValues } from './mathHelpers.js';
```

- берём конкретные exports

### namespace import

```js
import * as mathHelpers from './mathHelpers.js';
```

- берём весь модуль целиком, как объект

## когда использовать

- обычно namespace import используют, когда:
  - модуль большой
  - много функций
  - нужен красивый grouping
  - возможны конфликты имён

## когда не использовать

- если нужны 1–2 функции

## кратко

```js
import * as moduleName from './file.js';
```

- создаёт специальный объект:

```
{
  export1,
  export2,
  export3
}
```

- который:
  - содержит все exports модуля
  - readonly
  - использует live bindings
  - помогает группировать код
  - избегает конфликтов имён
