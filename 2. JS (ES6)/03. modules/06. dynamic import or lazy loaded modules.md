# dynamic import or lazy loaded modules

- обычный import загружается сразу при запуске файла:

```js
import { formatPrice } from './helpers.js';
```

- это называется static import (статический импорт)

## проблема обычного import

- представь сайт:
  - главная страница
  - настройки
  - админка
  - тяжёлый график
  - редактор изображений

- если всё импортировать сразу:

```js
import './chart.js';
import './adminPanel.js';
import './imageEditor.js';
```

- то браузер загрузит всё сразу, даже если пользователь никогда не откроет эти страницы

- это:
  - увеличивает размер initial bundle
  - замедляет загрузку
  - ухудшает performance

## dynamic import

- позволяет загрузить модуль только тогда, когда он реально нужен

### синтаксис

```js
import('./module.js');
```

- это не обычный import
- это специальная функция
- она возвращает Promise

### пример

```js
// calculator.js

export function sum(a, b) {
  return a + b;
}
```

```js
// dynamic import

buttonElement.addEventListener('click', async () => {
  const calculatorModule = await import('./calculator.js');

  console.log(calculatorModule.sum(2, 3));
});
```

### что здесь происходит

- после клика

```js
await import('./calculator.js');
```

- браузер
  1. загружает файл
  2. выполняет его
  3. возвращает exports

### что возвращает import()

- возвращается объект модуля:

```js
const mathModule = await import('./math.js');

console.log(mathModule);
```

- например:

```js
{
  divide: f,
  multiply: f
}
```

## dynamic import + default export

```js
// logger.js

export default function logMessage(textValue) {
  console.log(textValue);
}
```

```js
// import
const loggerModule = await import('./logger.js');

loggerModule.default('hello');
```

### почему `.default`

- потому что

```js
export default ...
```

- хранится внутри объекта модуля как свойство `default`

## lazy loading

- ленивая загрузка

### смысл

- не загружать код сразу
- загружать позже при необходимости

- dynamic import - основной механизм lazy loading в JS

## где это особенно важно

1. react

```js
const ProfilePage = lazy(() => import('./ProfilePage.js'));
```

- тогда компонент загрузится только при переходе на страницу

2. большие библиотеки

- chart libraries
- editors
- 3D engines
- admin tools

3. редко используемый код

- модальное окно
- onboarding
- tutorial
- debug panel

## static import vs dynamic import

```
Static import              Dynamic import

Загружается сразу          Загружается позже
Синтаксис сверху файла     Можно вызывать где угодно
Не Promise                 Promise
Используется постоянно     Используется по необходимости
```

## можно ли вызвать import() внутри if

- да

```js
if (userRole === 'admin') {
  const adminModule = await import('./adminTools.js');

  adminModule.openAdminPanel();
}
```

- это одна из главных причин существования dynamic import
- обычный `import` так делать не может

## code splitting

- bundlers:
  - webpack
  - vite
  - rollup

- видят:

```js
import('./analytics.js');
```

- и создают отдельный chunk

- например:

```
main.js
analytics.chunk.js
admin.chunk.js
```

- то есть приложение разбивается на части
- это называется code splitting

## очень важный момент

- dynamic import:
  - асинхронный
  - может упасть

- поэтому нужен `try-catch`

```js
try {
  const apiModule = await import('./api.js');

  apiModule.loadData();
} catch (errorValue) {
  console.log('loading failed');
}
```

## когда не нужен dynamic import

- если модуль:
  - маленький
  - нужен всегда
  - критичен для initial render

## вопросы

- почему `import()` возвращает Promise?

- потому что загрузка модуля:
  - асинхронная
  - требует network request
  - занимает время

## частая ошибка

```js
const userModule = import('./user.js');

userModule.getUser(); // ошибка, потому что import() возвращает Promise
```

- правильно так:

```js
const userModule = await import('./user.js'); // с `await`

userModule.getUser();

// или

import('./user.js').then((profileModule) => {
  profileModule.getProfile();
});
```
