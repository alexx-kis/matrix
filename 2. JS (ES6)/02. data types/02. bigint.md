# bigint

- это специальный тип данных в JS для работы с очень большими целыми числами, которые не помещаются в обычный тип `number`

## почему появился bigint

- внутри JS все числа хранятся в формате:
  - IEEE 754
  - 64-bit floating point

- из-за этого у number есть ограничение по точности

## максимально безопасное число в number

- в JS существует:

```js
Number.MAX_SAFE_INTEGER;
```

- это 9007199254740991

## что значит sage integer

- это число, которое JS может бранить без потери точности

- например:

```js
console.log(9007199254740991); // ok
```

- но:

```js
console.log(9007199254740992);
console.log(9007199254740993);
```

- результат будет неожиданным
- JS начинает терять точность

## пример проблемы

```js
const brokenValueA = 9007199254740992;
const brokenValueB = 9007199254740993;

console.log(brokenValueA === brokenValueB); // true
```

- хотя числа разные

## зачем нужен `BigInt`

- он позволяет работать с целыми числами
  - без ограничения `MAX_SAFE_INTEGER`
  - без потери точности

## как создать `BigInt`

### способ 1 - добавить `n`

```js
const hugeNumber = 999999999999999999999999999999999n;
```

### способ 2 - через `BigInt()`

```js
const anotherHugeNumber = BigInt(999999999999999);

// или

const hugeFromString = BigInt('999999999999999999999999999999');
```

## тип данных

```js
const bigintValue = 123n;

console.log(typeof bigintValue); // bigint
```

## важнейшая особенность `BigInt`

- `BigInt` работает только с целыми числами, дробные числа запрещены

## арифметика с `BigInt`

```js
// сложение

const firstBigValue = 10000000000000000000n;
const secondBigValue = 20000000000000000000n;

console.log(firstBigValue + secondBigValue);

// вычитание

const leftBigValue = 50n;
const rightBigValue = 20n;

console.log(leftBigValue - rightBigValue);

// умножение

const multiplyBigValueA = 12n;
const multiplyBigValueB = 3n;

console.log(multiplyBigValueA * multiplyBigValueB);

// деление

const divideBigValueA = 10n;
const divideBigValueB = 3n;

console.log(divideBigValueA / divideBigValueB);

// `BigInt` не хранит дробную часть
// деление всегда отбрасывает остаток и возвращает целое число

// остаток от деления

const moduloBigValueA = 10n;
const moduloBigValueB = 3n;

console.log(moduloBigValueA % moduloBigValueB); // 1n
```

## нельзя смешивать `number` и `bigint`

- это одна из самых важных тем

- ошибка:

```js
const regularNumericValue = 10;
const bigintNumericValue = 20n;

console.log(regularNumericValue + bigintNumericValue); // TypeError
```

- потому что `number` - floating point, `bigint` - arbitrary precision integer

- JS не хочет автоматически конвертировать типы, чтобы не потерять точность

## как исправить

- преобразовать `number` -> `bigint`

```js
const regularToBigIntValue = 10;
const bigintTargetValue = 20n;

console.log(BigInt(regularToBigIntValue) + bigintTargetValue);
```

- или `bigint` -> `number`

```js
const bigintToNumberValue = 20n;

console.log(Number(bigintToNumberValue) + 10);
```

- но тут уже возможна потеря точности

## сравнение `BigInt`

- обычные сравнения работают

```js
console.log(10n > 5n);

console.log(10n === 10n);
```

### интересный момент с `==`

```js
console.log(10n == 10); // true

console.log(10n === 10); // false
```

- потому что типы разные `bigint` и `number`

## математические методы и `BigInt`

- `Math` работает только с `number`

## Boolean и `BigInt`

- поведение похоже на обычные числа

- `0n` -> false
- любое другое значение -> true

## `BigInt` и JSON

```js
const serverPayload = {
  id: 123n,
};

JSON.stringify(serverPayload); // TypeError: Do not know how to serialize a BigInt
```

- JSON официально не поддерживает `BigInt`

- решение - перевести `BigInt` в строку

## где используется `BigInt`

1. криптография

- очень большие числа:
  - RSA
  - шифрование
  - цифровые подписи

2. Blockchain / Web3

- там огромные числа постоянно
  - wei
  - balances
  - gas

3. базы данных

- некоторые ID могут быть больше, чем safe integer

4. финансовые системы

- когда нельзя терять точность

## почему всегда нельзя использовать `BigInt` ?

- медленнее
- несовместимы с Math
- не поддерживает дроби
- не совместим с JSON
- нельзя смешивать с `number`
- большинство API работают с `number`
