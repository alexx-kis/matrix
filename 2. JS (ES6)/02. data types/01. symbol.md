# symbol

- это примитивный тип данных в JS, который используется для создания уникальных идентификаторов

## зачем он нужен

- пусть есть объект

```js
const userProfile = {
  id: 101,
  name: 'Alex',
};
```

- позже другой код может случайно добавить поле с тем же именем:

```js
userProfile.id = 'new-id';
```

- или две библиотеки могут использовать одинаковые ключи:

```js
objectData.status;
```

- возникают конфликты имён

## `Symbol` решает эту проблему

- каждый созданный `Symbol` - уникален
- даже если описания одинаковые:

```js
const firstToken = Symbol('id');
const secondToken = Symbol('id');

console.log(firstToken === secondToken); // false
```

- это главное свойство Symbol

## как создать Symbol

```js
const uniqueKey = Symbol();
```

- можно передать описание:

```js
const userIdKey = Symbol('userId');
```

- описание нужно только для отладки

## главное применение

- уникальные ключи объектов

```js
// обычные ключи конфликтуют

const settingsData = {};
settingsData.mode = 'dark';
settingsData.mode = 'light';

console.log(settingsData.mode); // light
```

- с Symbol ключ уникален

```js
const darkModeKey = Symbol('mode');
const lightModeKey = Symbol('mode');

const settingsObject = {
  [darkModeKey]: 'dark',
  [lightModeKey]: 'light',
};

console.log(settingsObject[darkModeKey]); // dark
console.log(settingsObject[lightModeKey]); // light
```

## Symbol-ключи скрыты от обычного перебора

```js
const secretField = Symbol('secret');

const accountData = {
  username: 'alex',
  [secretField]: 'hidden',
};

console.log(Object.keys(accountData)); // ['username']
```

### for...in тоже не видит Symbol

```js
for (const propertyName in accountData) {
  console.log(propertyName);
}
```

- вывод

```
username
```

## как получить Symbol-ключи

- есть специальный метод

```js
Object.getOwnPropertySymbols();
```

### пример

```js
const passwordKey = Symbol('password');

const authData = {
  login: 'alex',
  [passwordKey]: '123456',
};

const symbolKeys = Object.getOwnPropertySymbols(authData);

console.log(symbolKeys); // [Symbol(password)]
```

## глобальный реестр Symbol

- обычно каждый Symbol уникален:

```js
const firstValue = Symbol('token');
const secondValue = Symbol('token');

console.log(firstValue === secondValue); // false
```

- но иногда нужен один общий Symbol
- для этого есть `Symbol.for()`

### как работает `Symbol.for()`

```js
const globalKeyOne = Symbol.for('session');
const globalKeyTwo = Symbol.for('session');

console.log(globalKeyOne === globalKeyTwo); // true
```

### получение имени Symbol

```js
const apiToken = Symbol.for('api');

console.log(Symbol.keyFor(apiToken)); // api
```

### обычный Symbol не работает с keyFor

```js
const localToken = Symbol('local');

console.log(Symbol.keyFor(localToken)); // undefined
```

## Symbol и JSON

- Symbol игнорируется в JSON

```js
const internalId = Symbol('id');

const productData = {
  title: 'Phone',
  [internalId]: 555,
};

console.log(JSON.stringify(productData)); // {"title": "Phone"}
```

- так сделали, потому что Symbol считается служебным/внутренним ключом

## Symbol нельзя автоматически преобразовать в строку

```js
const colorKey = Symbol('color');

console.log('Value ' + colorKey); // TypeError
```

- потому что это могло бы случайно уничтожить уникальность

- чтобы преобразовать, нужно использовать `String()`

## получение описания символа

```js
const eventKey = Symbol('clickEvent');

console.log(eventKey.description); // 'clickEvent'
```

## известные встроенные Symbol

- JS содержит специальные системные Symbol:
  - `Symbol.iterator`
  - `Symbol.toPrimitive`
  - `Symbol.toStringTag`
  - `Symbol.asyncIterator`

- это уже механизм внутренней настройки поведения объектов
