# decorators

- это способ изменять или расширять класс, метод, свойство или accessor без изменения исходного кода

### самая важная проблема

- decorators в JS – сложная тема, потому что:
  - долго были experimental
  - существовало несколько версий proposal
  - TypeScript decorators и modern JS decorators отличаются
  - поддержка зависит от окружения

- но концепция очень важна: decorator получает что-то и возвращает улучшенную версию

## идея decorator

- есть класс:

```js
class UserService {
  //
}
```

- decorator может:
  - добавить логирование
  - добавить кеширование
  - заморозить объект
  - добавить метаданные
  - изменить поведение метода
  - добавить validation

- это похоже на wrapper

### простейшая аналогия

- было:

```js
function greetUser() {
  console.log('Hello');
}
```

- decorator:

```js
function loggerDecorator(originalFunction) {
  return function () {
    console.log('Before');

    originalFunction();

    console.log('After');
  };
}

const wrappedGreet = loggerDecorator(greetUser);

wrappedGreet(); // Before Hello After
```

- это decorator pattern

### decorators для классов

- старый TypeScript/Babel syntax

```js
@loggerDecorator
class UserService {
  //
}
```

- `@decorator` примерно означает:

```js
UserService = loggerDecorator(UserService);
```

- то есть декоратор получает класс

### пример decorator function

```js
function sealedDecorator(targetClass) {
  Object.seal(targetClass);
  Object.seal(targetClass.prototype);
}

@sealedDecorator
class ApiManager {
  //
}
```

- `Object.seal` запрещает добавлять и удалять свойства

- синтаксис `@decorator` работает не везде
- в обычном JS чаще нужен transpiler (Babel, TypeScript, framework compiler)
- поэтому важно понимать идею decorators, а не только синтаксис

### пример идеи

```js
// был метод
saveData();

// decorator добавляет:
console.log before/after
```

- пример:

```js
function logDecorator(originalMethod) {
  return function (...methodArgs) {
    console.log('Method started');
    const result = originalMethod.apply(this, methodArgs);

    console.log('Method finished');

    return result;
  };
}
```

```js
//  обычный метод:
class StorageManager {
  saveData() {
    console.log('Saving...');
  }
}

// оборачиваем вручную:
StorageManager.prototype.saveData = logDecorator(StorageManager.prototype.saveData);

// использование:
const localStorageManager = new StorageManager();

localStorageManager.saveData();

// результат:
Method started
Saving...
Method finished
```

## где decorators особенно популярны

- angular очень активно использует decorators, например:

```ts
@Component()
@Injectable()
@Input()
@Output()
```

- NestJS

```ts
@Controller()
@Get()
@Post()
```

- MobX

```ts
@observable
@action
```

## почему decorators удобны

- потому что позволяют писать дополнительную логику отдельно, а не внутри класса

## decorators и separation of concerns

- без decorators:

```js
class ApiService {
  fetchData() {
    console.log('Logging');

    console.log('Validation');

    console.log('Caching');

    console.log('Metrics');

    console.log('Request');
  }
}
```

- с decorators:

```js
@log
@validate
@cache
@metrics
fetchData()
```

- логика разделяется

- но есть проблема - decorators могут
  - усложнять код
  - скрывать поведение
  - ломать читаемость
  - усложнять дебаг

- поэтому decorators – мощный, но опасный инструмент

## modern decorators proposal

- сейчас JS имеет новый decorators proposal
- синтаксис отличается от старого TypeScript decorators, но концепция остаётся той же - decorator получает сущность и изменяет её

## важно понять

- decorators не являются inheritance
- decorator:
  - не создаёт prototype chain
  - не использует extends
  - не создаёт parent class

- decorator модифицирует существующий код

## Decorators vs Mixins

```
| Mixins               | Decorators             |
| -------------------- | ---------------------- |
| добавляют методы     | изменяют поведение     |
| composition          | wrapping/modification  |
| работают с prototype | работают с definitions |
| расширение API       | метапрограммирование   |
```

## decorators and metaprogramming

- decorators – часть metaprogramming
- то есть код изменяет другой код
