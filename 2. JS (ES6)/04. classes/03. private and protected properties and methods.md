# private and protected properties and methods

- public - нет ограничений
- protected - только соглашение
- private - настоящая приватность

## public свойства

- по умолчанию всё public

```js
class UserSettings {
  constructor(theme) {
    this.theme = theme;
  }
}
```

- можно читать и менять снаружи:

```js
const dashboardSettings = new UserSettings('dark');

console.log(dashboardSettings.theme);

dashboardSettings.theme = 'light';
```

## protected свойства

- в JS нет настоящего protected, но есть соглашение – именование через подчёркивание

```js
class BankProfile {
  constructor(balance) {
    this._balance = balance;
  }
}
```

- подчёркивание означает "не трогай это напрямую", но технически доступ всё равно есть

```js
const mainBankAccount = new BankProfile(1000);

console.log(mainBankAccount._balance); // 1000
```

- подчёркивание используют, чтобы показать другим разработчикам, что это внутреннее свойство

## private свойства

- современный JS поддерживаем настоящие private fields
- для этого используется `#`

```js
class PaymentCard {
  #pinCode;

  constructor(pinCode) {
    this.#pinCode = pinCode;
  }
}
```

- теперь доступ снаружи запрещён

```js
const userCard = new PaymentCard(1234);

console.log(userCard.#pinCode); // ошибка
```

- потому что `#pinCode` – это настоящее private field

### важно понять

- private поля:
  - не лежат, как обычные свойства объекта
  - доступны только внутри класса
  - проверяются самим JS

## использование private fields внутри класса

```js
class MusicPlayer {
  #volumeLevel = 50;

  increaseVolume() {
    this.#volumeLevel++;
  }

  showVolume() {
    console.log(this.#volumeLevel);
  }
}

const audioPlayer = new MusicPlayer();

audioPlayer.increaseVolume();

audioPlayer.showVolume(); // 51
```

- но так нельзя:

```js
console.log(audioPlayer.#volumeLevel); // ошибка синтаксиса
```

- это не `undefined`, это именно syntax error, JS даже не даст выполнить код

## private methods

- методы тоже могут быть приватными

```js
class AuthManager {
  #validateToken() {
    console.log('Token validated');
  }

  login() {
    this.#validateToken();
  }
}

const securityAuth = new AuthManager();

securityAuth.login();

securityAuth.#validateToken(); // так нельзя
```

### private methods нужны для:

- внутренней логики
- скрытых helper methods
- защиты API класса

## важный нюанс

- private fields доступны только внутри того же класса

```js
class ParentDevice {
  #secretCode = '123';
}

class ChildDevice extends ParentDevice {
  showSecret() {
    console.log(this.#secretCode); // ошибка
  }
}
```

- это отличается от protected в других языках, например, Java, C#, TypeScript, там protected доступен наследникам
- в JS нет настоящего protected

## когда что использовать

### `#private`

- чувствительных данных
- внутреннего состояния
- скрытых helper methods
- защиты API классов

### `_protected`

- доступ наследникам
- просто показать "не трогать"
- сохранять гибкость
