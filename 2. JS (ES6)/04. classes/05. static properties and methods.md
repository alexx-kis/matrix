# static properties and methods

- обычно свойства и методы принадлежат объектам
- но иногда нужно, чтобы что-то принадлежало самому классу
- для этого существует `static`

## обычные свойства и методы

```js
class UserProfile {
  constructor(name) {
    this.name = name;
  }

  sayHello() {
    console.log(`Hello ${this.name}`);
  }
}

const firstProfile = new UserProfile('Alex');

firstProfile.sayHello(); // метод вызывается у объекта
```

- но иногда метод не относится к конкретному объекту

- например:
  - helpers methods
  - utility functions
  - factory methods
  - configuration
  - counters
  - constants

- тогда используют static

## static method

```js
class MathHelper {
  static double(numberValue) {
    return numberValue * 2;
  }
}

console.log(MathHelper.double(5)); // 10
```

## важный момент

- метод вызывается у класса, а не у объекта

- так не работает:

```js
const mathInstance = new MathHelper();
mathInstance.double(5); // ошибка double is not a functions
```

- почему? – потому что static methods не попадают в объект

- где хранятся static methods?
- обычные методы:

```js
ClassName.prototype;
```

- static methods:

```js
ClassName;
```

## static properties

```js
class ApiConfig {
  static baseUrl = 'https://api.test.com';
}

console.log(ApiConfig.baseUrl);
```

- объекту property не доступен

```js
const apiInstance = new ApiConfig();

console.log(.apiInstance.baseUrl) // undefined
```

## зачем нужны static properties

1. константы

```js
class KeyboardCodes {
  static ENTER = 'Enter';
  static ESCAPE = 'Escape';
}

// использование:
if (pressedKey === KeyboardCodes.ENTER) {
  //
}
```

2. конфигурация

```js
class ApplicationSettings {
  static apiVersion = 'v2';
}
```

3. счётчики

```js
class UserCounter {
  static totalUsers = 0;

  constructor() {
    UserCounter.totalUsers++;
  }
}

new UserCounter();
new UserCounter();

console.log(UserCounter.totalUsers); // 2
```

## очень важный момент

- внутри объекта `this` не указывает на static properties

```js
class ThemeManager {
  static currentTheme = 'dark';

  showTheme() {
    console.log(this.currentTheme);
  }
}

const pageTheme = new ThemeManager();
pageTheme.shoeTheme(); // undefined
```

- потому что `this` – это объект, а static property лежит в классе

## static methods и inheritance

- static methods тоже наследуются

```js
class BaseFormatter {
  static formatText(valueText) {
    return valueText.trim();
  }
}

class HtmlFormatter extends BaseFormatter {
  //
}

console.log(HtmlFormatter.formatText('hello ')); // 'hello'
```

## static methods и super

- можно использовать `super`

```js
class BaseApiService {
  static getBaseUrl() {
    return '/api';
  }
}

class UserApiService extends BaseApiService {
  static getUsersUrl() {
    return `${super.getBaseUrl()}/users`;
  }
}

console.log(UserApiService.getUsersUrl()); // /api/users
```

## static blocks

- современный JS поддерживает

```js
static {
  //
}
```

```js
class CacheManager {
  static cacheVersion;

  static {
    this.cacheVersion = 'v1';
    console.log('Cache initialized');
  }
}
```

- static block выполняется 1 раз, при создании класса

### частая ошибка 1

- вызвать static methods у объекта

```js
const textHelper = new StringFormatter();

textHelper.capitalize(); // ошибка

StringFormatter.capitalize(); // правильно
```

### частая ошибка 2

- пытаться использовать instance properties внутри static method

```js
class ProductCard {
  constructor(title) {
    this.title = title;
  }

  static printTitle() {
    console.log(this.title);
  }
}

ProductCard.printTitle(); // undefined
```

- потому что в static method this – это сам класс ProductCard, а не объект

## полезный паттерн – factory methods

```js
class UserFactory {
  constructor(name, role) {
    this.name = name;
    this.role = role;
  }

  static createAdmin(name) {
    return new UserFactory(name, 'admin');
  }
}

const adminUser = UserFactory.createAdmin('Alex');

console.log(adminUser);
```

### зачем это удобно?

- можно создавать:
  - готовые конфигурации
  - специальные типы объектов
  - helper constructors

## мини-таблица

```
| Тип              | Где лежит |
|------------------|-----------|
| обычный метод    | prototype |
| static method    | сам класс |
| обычное свойство | объект    |
| static property  | класс     |
```

## самое главно понять

- `static` означает "принадлежит самому классу", а не объектам этого класса
