# reverting of class inheritance to functional inheritance and vice versa

- самая главная идея – классы в JS – это не отдельная система
- внутри класса всё равно работает через:
  - функции
  - prototype
  - prototype chain

- то есть class inheritance можно переписать через functions и наоборот

## functional inheritance

- до появления классов в JS использовали:
  - функции-конструкторы
  - prototype

### пример function constructor

```js
function AnimalEntity(type) {
  this.type = type;
}
```

### методы добавления в prototype

```js
AnimalEntity.prototype.showType = function () {
  console.log(this.type);
};
```

### создание объекта

```js
const wildAnimal = new AnimalEntity('Tiger');

wildAnimal.showType();
```

- это старый стиль классов, хотя слова `class` здесь нет

## inheritance

- родитель:

```js
function VehicleEntity(brand) {
  this.brand = brand;
}

VehicleEntity.prototype.showBrand = function () {
  console.log(this.brand);
};
```

- дочерний constructor

```js
function CarEntity(brand, model) {
  VehicleEntity.call(this, brand);

  this.model = model;
}
```

- что делает `call()`

```js
VehicleEntity.call(this, brand);
```

- означает "вызови VehicleEntity с текущим `this`"

- это аналог `super(brand)` в классах

## inheritance prototype

- вот ключевая строка:

```js
CarEntity.prototype = Object.create(VehicleEntity.prototype);
```

- что делает `Object.create()` ? – создаёт объект с нужным prototype

- внутри получается:

```
CarEntity.prototype
  ↓
VehicleEntity.prototype
```

- теперь методы родителя доступны

- добавим собственный метод:

```js
CarEntity.prototype.showModel = function () {
  console.log(this.model);
};

const sportCar = new CarEntity('BMW', 'M5');

sportCar.showBrand();
sportCar.showModel();
```

- но есть проблема: после

```js
CarEntity.prototype = Object.create(...)
```

- ломается `constructor`

```js
console.log(sportCar.constructor); // VehicleEntity
```

- потому что prototype был заменён

- поэтому обычно пишут

```js
CarEntity.prototype.constructor = CarEntity;
```

### полная старая схема inheritance

```js
function VehicleEntity(brand) {
  this.brand = brand;
}

VehicleEntity.prototype.showBrand = function () {
  console.log(this.brand);
};

function CarEntity(brand, model) {
  VehicleEntity.call(this, brand);

  this.model = model;
}

CarEntity.prototype = Object.create(VehicleEntity.prototype);

CarEntity.prototype.constructor = CarEntity;

CarEntity.prototype.showModel = function () {
  console.log(this.model);
};
```

## теперь то же самое через class

```js
class VehicleEntity {
  constructor(brand) {
    this.brand = brand;
  }

  showBrand() {
    console.log(this.brand);
  }
}

class CarEntity extends VehicleEntity {
  constructor(brand, model) {
    super(brand);

    this.model = model;
  }

  showModel() {
    console.log(this.model);
  }
}
```

- это один и тот же механизм

## class version внутри делает примерно это:

- constructor: `super(brand)` ≈ `Vehicle.call(this, brand)`

- extends: `extends VehicleEntity` ≈ `Object.create(VehicleEntity.prototype)`

## важно понять

- классы не заменили prototype inheritance, они просто сделали запись удобнее
