# classes. basic syntax

- классы - это более удобный и понятный способ создавать объекты по шаблону
- раньше для этого использовали функции-конструкторы и `prototype`, а классы – это более современная оболочка над системой

## зачем нужны классы

- допустим нужно создавать много похожих объектов
- например:
  - пользователей
  - карточки товаров
  - врагов в игре
  - кнопки
  - компоненты

- без классов пришлось бы каждый раз вручную писать объект:

```js
const userOne = {
  name: 'Alex',
  age: 25,
};

const userTwo = {
  name: 'Kate',
  age: 30,
};
```

- это не удобно
- класс позволяет описать шаблон один раз

## базовый синтаксис

```js
class UserProfile {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
}
```

1. `class` создаёт класс

- имя класса обычно с большой буквы – это соглашение

2. `constructor` – специальный метод

- он вызывается автоматически при создании объекта
- он получает данные для нового объекта

3. `this` - это создаваемый объект

```js
this.name = name;

// означает:

создаваемыйОбъект.name = name;
```

## создание объекта

- для классов используется `new`

```js
const firstUser = new UserProfile('Alex', 30);
```

- происходит следующее

1. создаётся пустой объект `{}`
2. этот объект становится `this`
3. вызывается конструктор
4. в объект записываются свойства
5. объект возвращается автоматически

## методы класса

- внутри класса можно создавать методы

```js
class ProductCard {
  constructor(title, price) {
    this.title = title;
    this.price = price;
  }

  showInfo() {
    console.log(`${this.title}: ${this.price}`);
  }
}
```

## использование

```js
const keyboardProduct = new ProductCard('Keyboard', 120);

keyboardProduct.showInfo(); // Keyboard: 120
```

## важный момент

- в классе методы пишутся
  - без `function`
  - без запятых

## что хранится в объекте

```js
const keyboardProduct = new ProductCard('Keyboard', 120);
```

- в самом объекте:

```js
{
  title: 'Keyboard',
  price: 120
}
```

- а методы не копируются в каждый объект, они хранятся в `prototype`
- это очень важно для памяти

### проверим

```js
console.log(keyboardProduct.hasOwnProperty('title')); // true
console.log(keyboardProduct.hasOwnProperty('showInfo')); // false
```

- потому что:
  - `title` лежит в объекте
  - `showInfo` лежит в `prototype`

## class expression

- класс можно записать, как выражение

```js
const VehicleFactory = class {
  startEngine() {
    console.log('Engine started');
  }
};
```

## named class expression

```js
const AnimalCreator = class InnerAnimalClass {
  move() {
    console.log('Moving');
  }
};
```

## но обычно используют обычный синтаксис

```js
class UserProfile {
  //
}
```

## важное отличие от function declaration

- function declaration можно вызвать до объявления:

```js
runApp();

function runApp() {
  console.log('run');
}
```

- с классами так нельзя:

```js
const newAdmin = new AdminUser();

class AdminUser {
  //
}

// Cannot access before initialization
```

- классы не поднимаются так же, как функции
- точнее:
  - имя создаётся
  - но использовать класс до объявления нельзя

- это называется Temporal Dead Zone (TDZ), как у `let` и `const`

## класс – это специальный тип функции

```js
class MessageBuilder {
  //
}

console.log(typeof MessageBuilder); // function
```

## отличия от обычной функции

- класс нельзя вызвать без `new`
