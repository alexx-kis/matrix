# inheritance

- это механизм, когда один класс может использовать свойства и методы другого класса

## зачем нужно наследование

- пусть есть общий класс:

```js
class UserAccount {
  //
}
```

- и есть разные типы пользователей:
  - admin
  - moderator
  - customer

- у всех есть что-то общее:
  - name
  - email
  - login()

- но у admin есть дополнительные возможности
- чтобы не копировать одинаковый код, используют inheritance

## основной синтаксис

- для наследования используется `extends`

### пример

```js
class PersonProfile {
  constructor(name) {
    this.name = name;
  }

  sayHello() {
    console.log(`Hello, ${this.name}`);
  }
}
```

- теперь создадим дочерний класс:

```js
class EmployeeProfile extends PersonProfile {
  //
}
```

- теперь EmployeeProfile получает:
  - свойства
  - методы
  - prototype chain
- от PersonProfile

## добавление собственных методов

- дочерний класс может иметь свои методы

```js
class EmployeeProfile extends PersonProfile {
  work() {
    console.log(`${this.name} is working`);
  }
}
```

## использование

```js
const developerEmployee = new EmployeeProfile('Kate');

developerEmployee.sayHello();
developerEmployee.work();
```

## конструктор в дочернем классе

- теперь важнейшая часть про inheritance

- если constructor не писать, JS автоматически делает примерно это:

```js
constructor(...args) {
  super(...args);
}
```

- поэтому этот код работает:

```js
class AnimalProfile {
  constructor(type) {
    this.type = type;
  }
}

class DogProfile extends AnimalProfile {
  //
}

const dogOne = new DogProfile('Labrador');

console.log(dogOne.type);
```

- но если constructor есть – появляется super()

- пример:

```js
class AnimalProfile {
  constructor(type) {
    this.type = type;
  }
}

class CatProfile extends AnimalProfile {
  constructor(type, color) {
    super(type);

    this.color = color;
  }
}
```

## что такое `super()` ?

```js
super(type);
```

- вызывает constructor родительского класса

## почему super() обязателен?

- потому что в наследуемом классе `this` не создаётся автоматически
- сначала JS должен вызвать родительский constructor

## очень важное правило

- в дочернем классе `super()` должен быть вызван до использования `this`

### полный пример:

```js
class VehicleUnit {
  constructor(brand) {
    this.brand = brand;
  }

  showBrand() {
    console.log(this.brand);
  }
}

class CarUnit extends VehicleUnit {
  constructor(brand, model) {
    super(brand);

    this.model = model;
  }

  showModel() {
    console.log(this.model);
  }
}

const bmwCar = new CarUnit('BMW', 'M5');

bmwCar.showBrand();
bmwCar.showModel();
```

## переопределение методов

- дочерний класс может заменять методы родителя

### пример:

```js
class ShapeFigure {
  draw() {
    console.log('Basic shape');
  }
}

class CircleFigure extends ShapeFigure {
  draw() {
    console.log('Drawing circle');
  }
}
```

- при этом родительский метод не удаляется, он просто перекрывается
- можно вызвать метод родителя через super

### пример:

```js
class NotificationBase {
  send() {
    console.log('Sending notification');
  }
}

class EmailNotification extends NotificationBase {
  send() {
    super.send();

    console.log('Sending email');
  }
}

const welcomeEmail = new EmailNotification();

welcomeEmail.send();
```

```
Sending notification
Sending email
```

## instanceof

- проверка принадлежности к классу

### пример:

```js
console.log(welcomeEmail instanceof EmailNotification); // true

console.log(welcomeEmail instanceof NotificationBase); // true
```

- потому что объект связан prototype chain с обоими классами

### важный момент

- inheritance работает через prototype

- вот, что делает extends внутри примерно:

```js
EmailNotification.prototype.__proto__ = NotificationBase.prototype;
```

- мини-схема:

```
welcomeEmail
  ↓
EmailNotification.prototype
  ↓
NotificationBase.prototype
  ↓
Object.prototype
```
