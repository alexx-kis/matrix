# instance of

- это оператор для проверки "создан ли объект этим классом?"

- или точнее "есть ли prototype этого класса в prototype chain объекта?"

## базовый синтаксис

```js
object instanceof ClassName;
```

## пример

```js
class UserAccount {
  //
}

const mainUser = new UserAccount();

console.log(mainUser instanceof UserAccount); // true
```

## как работает внутри

- когда выполняется

```js
mainUser instanceof UserAccount;
```

- JS проверяет: есть ли UserAccount.prototype в prototype chain объекта

## prototype chain

```
mainUser
  ↓
UserAccount.prototype
  ↓
Object.prototype
```

## пример с наследованием

```js
class AnimalBase {
  //
}

class DogAnimal extends AnimalBase {
  //
}

const petDog = new DogAnimal();

console.log(petDog instanceof DogAnimal); // true
```

- `true` потому что:

```
DogAnimal.prototype
  ↓
AnimalBase.prototype
```

- полная цепочка:

```
petDog
  ↓
DogAnimal.prototype
  ↓
AnimalBase.prototype
  ↓
Object.prototype
```

## instanceof работает через prototype chain

## проверка встроенных объектов

### Array

```js
const colorList = ['red', 'blue'];

console.log(colorList instanceof Array); // true
```

### Date

```js
const currentDate = new Date();

console.log(currentDate instanceof Date); // true
```

### Error

```js
const apiError = new Error('Failed');

console.log(apiError instanceof Error); // true
```

## важный момент

- `typeof` и `instanceof` - это разные вещи

- typeof проверяет тип значения
- instanceof проверяет prototype chain

## сравнение

```js
const userList = [];

console.log(typeof userList); // object

console.log(userList instanceof Array); // true
```

- это важно, потому что массив – это объект
- typeof у массивов – всегда `object`

- поэтому для массивов обычно используют `Array.isArray()` или иногда `instanceof Array`

## `instanceof` с обычными объектами

```js
const settingsObject = {};

console.log(settingsObject instanceof Object); // true
```

- цепочка такая:

```
settingsObject
 ↓
Object.prototype
```

## Функции – тоже объекты

```js
function openMOdal() {
  //
}

console.log(openModal instanceof Function); // true

// и даже так:

console.log(openModal instanceof Object); // true
```

- потому что цепочка такая:

```
openModal
  ↓
Function.prototype
  ↓
Object.prototype
```

## частая ошибка

### проверка примитивов

```js
console.log('hello' instance of String) // false
```

- потому что 'hello' – это primitive, а не объект

- а вот так:

```js
const wrappedText = new String('hello');

console.log(wrappedText instanceof String); // true
```

- но `new String` почти не используют, потому что это object-wrapper

## самое главное понять

- `instanceof` не проверяет "из какого класса создан объект" напрямую
- он проверяет "есть ли prototype класса в prototype chain объекта"

- и это фундаментально связано с prototype system JS
