# цикл for...of

- это цикл, который перебирает значения элементов в итерируемых структурах
- именно значения, а не индексы

## синтаксис

```js
for (const element of iterable) {
  // код
}
```

- `element` - текущий элемент
- `iterable` - структура, по которой можно итерироваться

### пример (массив):

```js
const numbersArray = [10, 20, 30];

for (const value of numbersArray) {
  console.log(value);
}
```

- вывод: 10 20 30

- здесь `value` - это сам элемент массива, не индекс

## важно: iterable (итерируемые объекты)

- `for...of` работает только с итерируемыми объектами:
  - массивы
  - строки
  - `Set`
  - `Map`
  - `NodeList`
  - `arguments`

- не работает с обычными объектами `{}`

### пример со строкой

```js
const textValue = 'hello';

for (const char of textValue) {
  console.log(char);
}
```

- вывод:

```
h
e
l
l
o
```

## отличие от for...in

- в переменную попадают индексы (ключи):

```js
const numbersList = [10, 20, 30];

for (const index in numbersList) {
  console.log(index);
}
```

- вывод:

```
0
1
2
```

## когда использовать `for...of`

- нужны значения
- не нужен индекс
- более читаемый код

## можно ли прервать цикл?

- да

```js
const numbersBreak = [10, 20, 30, 40];

for (const value of numbersBreak) {
  if (value === 30) break;

  console.log(value);
}
```

- `break` работает
- `continue` тоже работает

## важное отличие от forEach

```js
const numbersLoop = [10, 20, 30];

numbersLoop.forEach(function (value) {
  if (value === 20) return;
  console.log(value);
});
```

- нельзя сделать `break`
- `return` не останавливает внешний цикл

- а вот в `for...of` цикл остановится

### частый вопрос

- почему `for...of` не работает с объектами? - потому что объект не является iterable

```js
const userObject = {
  name: 'Alex',
  age: 25,
};

// ❌ Ошибка
for (const value of userObject) {
}
```

### как обойти это

```js
const userObjectTwo = {
  name: 'Alex',
  age: 25,
};

// перебирать значения свойств:
for (const value of Object.values(userObjectTwo)) {
  console.log(value);
}
```
