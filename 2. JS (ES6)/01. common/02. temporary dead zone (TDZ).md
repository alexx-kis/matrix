# temporary dead zone

- это промежуток времени, когда:
  - переменная уже создана
  - но ещё не инициализирована
  - и к ней нельзя обращаться

- если обратиться, будет `ReferenceError`

## 1. самый простой пример

```js
console.log(userAgeValue); // ❌ ReferenceError

let userAgeValue = 25;
```

- если есть hoisting, почему ошибка, а не undefined, как с var?

## 2. как это работает под капотом

- JS делает 2 этапа:

1. Creation phase (фаза создания)

- движок:
  - находит переменные
  - регистрирует их в памяти
  - но значение не присваивается
  - переменная в состоянии uninitialized

```js
let productPrice;
```

2. Execution phase (фаза выполнения)

```js
productPrice = 100;
```

- только здесь она становится доступной

- вот этот промежуток и есть TDZ

```
начало области видимости
↓
TDZ (нельзя использовать)
↓
строка с let/const
↓
переменная доступна
```

## 3. визуально

```js
{
  // ❗ TDZ начинается
  console.log(totalSum); // ❌

  let totalSum = 50; // ← инициализация

  console.log(totalSum); // ✅ 50
}
```

## 4. сравнение с var

- `var`

```js
console.log(orderCount); // undefined

var orderCount = 10;
```

- потому что var сразу получает значение `undefined`

## 6. TDZ и блоки {}

```js
{
  // TDZ уже начался

  // console.log(cityName); ❌

  let cityName = 'Amsterdam';
}
```

## 7 tricky- вопрос

```js
let amountValue = 5;

function testAmount() {
  console.log(amountValue); // ❌ ReferenceError
  let amountValue = 10;
}

testAmount();
```

## 8. TDZ и параметры по умолчанию

```js
function calculateTotal(valueA = valueB, valueB = 10) {
  return valueA + valueB;
}

calculateTotal(); // ❌ ReferenceError
```

- потому что `valueA` пытается использовать `valueB`
- но `valueB` ещё в TDZ

## 9. зачем вообще сделали TDZ?

- чтобы убрать опасное поведение var

```js
// старый JS (var)
if (true) {
  priceValue = 100;
  var priceValue;
}
```

- это работало, но было:
  - не очевидно
  - сложно дебажить

- TDZ заставляет:
  - писать код последовательно
  - избегать магии hoisting
  - ловить ошибки раньше

## итог

- TDZ — это период между началом области видимости переменной и её объявлением, в течение которого переменная уже существует, но не инициализирована, и при обращении к ней возникает ReferenceError.
