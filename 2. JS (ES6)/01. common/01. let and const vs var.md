# `let/const` vs var

- до ES6 (2015) в JS был только один способ объявлять переменные

```js
var value = 10;
```

- это приводило к проблемам, из-за которых код становился:
  1. непредсказуемым
  2. сложным для отладки
  3. опасным в больших проектах

- поэтому в ES6 добавили `let` и `const`

## 1. основное различие (самое важное)

- `var` - function scoped
- `let / const` - block scoped

### область видимости - это где переменная доступна в коде

#### пример с `var`

```js
function testVarScope() {
  if (true) {
    var numberVar = 42;
  }

  console.log(numberVar); // 42 😱
}
```

- var игнорирует `{}` блоки
- он живёт во всей функции

#### пример с `let` (с const то же самое)

```js
function testLetScope() {
  if (true) {
    let numberLet = 42;
  }

  console.log(numberLet); // ❌ ReferenceError
}
```

## 2. повторное объявление

- `var` - можно (и это плохо)

```js
var userName = 'Alex';
var userName = 'John'; // без ошибки
```

- это может ломать код, особенно в больших файлах

- `let / const` - нельзя

```js
let userAge = 25;
let userAge = 30; // ❌ SyntaxError

const apiUrl = 'url1';
const apiUrl = 'url2'; // ❌ SyntaxError
```

## 3. переопределение значения

- `var / let` - можно

```js
var score = 10;
score = 20; // ✅ можно

let counter = 1;
counter = 2; // ✅ можно
```

- `const` - нельзя

```js
const maxLimit = 100;
maxLimit = 200; // ❌ TypeError
```

- ⚠️ `const` не делает объект неизменяемым

```js
const userObject = {
  name: 'Alex',
};

userObject.name = 'John'; // ✅ можно
```

- потому что нельзя менять ссылку, но можно менять содержимое объекта

## 4. hoisting

- `var`

```js
console.log(testVar); // undefined 😐

var testVar = 5;
```

- ошибки нет, потому что переменная поднимается наверх и инициализируется со значением `undefined`

- `let / const`

```js
console.log(testLet); // ❌ ReferenceError

let testLet = 5;
```

- здесь уже ошибка

## 5. Глобальный объект

- в браузере есть глобальный объект `window`

```js
var globalVar = 10;

console.log(window.globalVar); // 10

let globalLet = 20;

console.log(window.globalLet); // undefined ❌
```

- `let / const` не добавляются в `window`

## 6. поведение в циклах

- `var`

```js
for (var indexVar = 0; indexVar < 3; indexVar++) {
  setTimeout(() => {
    console.log(indexVar);
  }, 100);
}
```

- выведет 3 3 3

- `let`

```js
for (let indexLet = 0; indexLet < 3; indexLet++) {
  setTimeout(() => {
    console.log(indexLet);
  }, 100);
}
```

- выведет 0 1 2

- потому что `let` создаёт новую переменную на каждой итерации

## 7. когда что использовать

- `const` по умолчанию

- `let` если значение будет меняться

- `var` не использовать

## итог

```
Характеристика	      var	          let	            const

Scope         	      function	    block	          block
Hoisting      	      undefined     TDZ	            TDZ
Переобъявление	      ✅ можно      ❌ нельзя       ❌ нельзя
Изменение значения	  ✅ можно      ✅ можно       ❌ нельзя
Добавляется в window	✅ да         ❌ нет          ❌ нет
```
