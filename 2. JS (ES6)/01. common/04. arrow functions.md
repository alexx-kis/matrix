# arrow functions

## 1. обычная функция

- это классический способ объявления функции в JS

```js
function sayHello(name) {
  return 'Hello ' + name;
}

// или

const sayHi = function (name) {
  return 'Hi ' + name;
};
```

- это полноценные функции с собственным контекстом `this`, `arguments`, возможностью быть конструктором итд

## 2. стрелочная функция

- появилась в ES6.
- более короткий синтаксис

```js
const sayHelloArrow = (name) => {
  return 'Hello ' + name;
};

// или

const sayHelloShort = (name) => 'Hello ' + name;
```

- это не просто короткая запись, у них другое поведение

# ключевые отличия

## 1. разное поведение `this`

- в обычной функции `this` определяется в момент вызова:

```js
const userOne = {
  name: 'Alex',
  sayName: function () {
    console.log(this.name);
  },
};

userOne.sayName(); // "Alex"
```

- в стрелочной функции `this` не свой, а берётся из внешней области (lexical this)

```js
const userTwo = {
  name: 'Alex',
  sayName: () => {
    console.log(this.name);
  },
};

userTwo.sayName(); // undefined ❌
```

- потому что `this` берётся снаружи объекта, а там обычно window/undefined

### вывод:

- стрелочные функции не подходят для методов объекта

## 2. `this` нельзя изменить

- в обычной функции:

```js
function showName() {
  console.log(this.name);
}

const personOne = { name: 'John' };

showName.call(personOne); // "John"
```

- в стрелочной функции:

```js
const showNameArrow = () => {
  console.log(this.name);
};

showNameArrow.call({ name: 'John' }); // undefined ❌
```

- `call`, `apply`, `bind` не работают со стрелками

## 3. нет arguments

- обычная функция:

```js
function sumNumbers() {
  console.log(arguments);
}

sumNumbers(1, 2, 3); // есть arguments
```

- стрелочная функция:

```js
const sumNumbersArrow = () => {
  console.log(arguments);
};

sumNumbersArrow(1, 2, 3); // ❌ ошибка
```

- у стрелочных функций нет `arguments`
- использовать rest оператор:

```js
const sumNumbersArrowFixed = (...args) => {
  console.log(args);
};
```

## 4. нельзя использовать, как конструктор

- обычная функция:

```js
function User(name) {
  this.name = name;
}

const newUser = new User('Alex');
```

- стрелочная функция:

```js
const UserArrow = (name) => {
  this.name = name;
};

const newUser = new UserArrow('Alex'); // ❌ ошибка
```

- стрелочные функции нельзя использовать с `new`

## 5. нет `prototype`

```js
function TestFunc() {}
console.log(TestFunc.prototype); // есть

const TestArrow = () => {};
console.log(TestArrow.prototype); // undefined
```

## 6. более короткий синтаксис

```js
// один аргумент:
const doubleValue = (x) => x * 2;

// несколько аргументов:
const sumValues = (a, b) => a + b;

// возврат объекта:
const createUser = () => ({ name: 'Alex' });
```

## 7. неявный return

```js
const multiplyValues = (a, b) => a * b;
```

## 8. нет hoisting, как у function declaration

```js
sayHelloFunc();

function sayHelloFunc() {
  console.log('Hello');
}

//

sayHelloArrow(); // ❌ ошибка

const sayHelloArrow = () => {
  console.log('Hello');
};
```

## когда что использовать

- стрелочную функцию:
  - callback
  - короткие функции
  - работа с массивами (`map`, `filter`, `reduce`)
  - когда нужен внешний `this`

- не стрелочную функцию
  - методы объектов
  - конструкторы
  - когда нужен свой `this`
  - обработчики, когда важен контекст

### 🧩 Мини-сравнение

```
Возможность       Function	      Arrow
this              динамический	  лексический
arguments         есть	          нет
new               можно	          нельзя
prototype         есть	          нет
call/apply/bind   работают	      не работают
hoisting          есть (function)	нет
```
