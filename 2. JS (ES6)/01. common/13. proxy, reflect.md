# proxy, reflect

## 1. proxy

- это обёртка над объектом, которая позволяет перехватывать любые операции с этим объектом

```js
const originalObject = { value: 10 };

const proxyObject = new Proxy(originalObject, {
  get(target, property) {
    console.log(`Чтение свойства: ${property}`);
    return target[property];
  },
});

console.log(proxyObject.value);
```

- вывод:

```
Чтение свойства: value
10
```

## 2. что можно перехватывать

- proxy может перехватить почти всё

```
Операция              Trap (перехватчик)

чтение                свойства	get
запись                свойства	set
проверка              "prop" in obj	has
удаление свойства     deleteProperty
вызов функции	        apply
new (конструктор)	    construct
```

## 3. базовая структур proxy

```js
const proxyInstance = new Proxy(targetObject, handlerObject);
```

- `targetObject` - оригинальный объект
- `handlerObject` - объект с перехватчиками (trap-ами)

## 4. пример с set

```js
const userProfile = {
  age: 25,
};

const userProxy = new Proxy(userProfile, {
  set(target, property, value) {
    if (property === 'age' && value < 0) {
      throw new Error('Возраст не может быть отрицательным');
    }

    target[property] = value;
    return true;
  },
});

userProxy.age = -10; // ❌ Ошибка
```

- это называется валидация через proxy

## 5. важный момент

- trap обязан вернуть значение:
  - set -> должен вернуть true или false
  - get -> должен вернуть значение

- если не вернуть, может быть ошибка

## 6. проблема proxy без reflect

```js
const product = {
  price: 100,
};

const productProxy = new Proxy(product, {
  get(target, property) {
    return target[property];
  },
});
```

- а что если:
  - есть getter
  - есть prototype
  - сложная логика доступа

- тогда можно сломать поведение JS

## 7. reflect

- это встроенный объект, который содержит методы для работы с объектами

```js
Reflect.get();
Reflect.set();
Reflect.deleteProperty();
Reflect.has();
```

## 8. зачем нужен reflect

- он делает то же самое, что JS делает "по умолчанию"

```js
// то есть:

target[property];

// лучше заменить на:

Reflect.get(target, property);
```

## 9. правильный proxy с reflect

```js
const order = {
  total: 200,
};

const orderProxy = new Proxy(order, {
  get(target, property, receiver) {
    console.log(`Доступ к ${property}`);
    return Reflect.get(target, property, receiver);
  },
});
```

## 10. почему reflect лучше

1. учитывает prototype
2. работает с getter/setter
3. поведение как у JS

## 11. пример с getter

```js
const account = {
  balance: 1000,
  get formattedBalance() {
    return `$${this.balance}`;
  },
};

const accountProxy = new Proxy(account, {
  get(target, property, receiver) {
    return Reflect.get(target, property, receiver);
  },
});
```

- receiver очень важен
- если не передать, `this` может сломаться

## 12. Proxy + Reflect (ключевая идея)

- Proxy = "перехватить"
- Reflect = "сделать, как было"

## 13. реальный кейс

- логирование

```js
const settings = {
  theme: 'dark',
};

const settingsProxy = new Proxy(settings, {
  get(target, property, receiver) {
    console.log(`GET ${property}`);
    return Reflect.get(target, property, receiver);
  },

  set(target, property, value, receiver) {
    console.log(`SET ${property} = ${value}`);
    return Reflect.set(target, property, value, receiver);
  },
});
```

## 14. когда используют Proxy

1. Реактивность (Vue 3 использует Proxy)
2. Валидация данных
3. Логирование
4. Защита объектов
5. Lazy вычисления
