# tag function

- это функция, которая вызывается перед обработкой шаблонной строки

```js
function printTemplate(templateParts, value) {
  console.log(templateParts);
  console.log(value);
}

const userName = 'Alex';

printTemplate`Hello ${userName}`;
```

- это не обычный вызов функции, функция ставится прямо перед шаблонной строкой

## что происходит внутри

- JS автоматически разбивает шаблонную строку на части

```js
//  шаблон:
`Hello ${userName}`;

// разделяется на:
// 1. текстовые куски
// 2. значение внутри ${}
```

## что получает функция

- внутри:

```js
function printTemplate(templateParts, value) {
  console.log(templateParts);
  console.log(value);
}
```

- мы получаем:

```js
templateParts = ['Hello ', ''];
value = 'Alex';
```

## почему массив содержит две строки

- потому что

```js
`Hello ${userName}`;
```

- это:

```
[текст] + [значение] + [текст]
```

- то есть:

```
'Hello ' + userName + ''
```

## более сложный пример

```js
function inspectTemplate(stringParts, firstValue, secondValue) {
  console.log(stringParts);
  console.log(firstValue);
  console.log(secondValue);
}

const productTitle = 'MacBook';
const productPrice = 2500;

inspectTemplate`
  Product: ${productTitle}
  Price: ${productPrice}
`;
```

- получим:

```js
stringParts = ['\n  Product: ', '\n  Price: ', '\n'];

firstValue = 'MacBook';
secondValue = 2500;
```

## важнейшая идея

- tag function получает:

```js
function tag(strings, ...values)
```

- `strings` - массив всех текстовых частей
- `...values` - все значения из `${}`

## стандартный шаблон tag function

- вот так пишут почти всегда:

```js
function templateTag(stringParts, ...dynamicValues) {
  console.log(stringParts);
  console.log(dynamicValues);
}
```
