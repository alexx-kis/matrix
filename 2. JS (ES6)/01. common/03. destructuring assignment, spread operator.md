# деструктуризация и spread-оператор

## 1. деструктуризация

- это способ достать значения из массивов или объектов и сразу разложить их по переменным

- без деструктуризации:

```js
const userData = { username: 'Alex', userAge: 25 };

const nameValue = userData.username;
const ageValue = userData.userAge;
```

- с деструктуризацией:

```js
const userData = { username: 'Alex', userAge: 25 };

const { username, userAge } = userData;
```

- то есть мы распаковываем структуру и сразу создаём переменные

## 2. деструктуризация объектов

### базовый синтаксис:

```js
const profile = {
  firstName: 'John',
  years: 30,
};

const { firstName, years } = profile;
```

#### важно:

- имена переменных должны совпадать с ключами

### переименование переменных

```js
const profile = {
  firstName: 'John',
};

const { firstName: userFirstName } = profile;
```

- теперь переменная называется `userFirstName`

### значения по умолчанию

```js
const settings = {
  theme: 'dark',
};

const { theme, language = 'en' } = settings;
```

- если свойства нет → берётся дефолт

### частая ошибка

```js
const { missingValue } = {};
console.log(missingValue); // undefined
```

- ошибки нет, просто `undefined`

### вложенные объекты

```js
const responseData = {
  user: {
    id: 1,
    email: 'test@mail.com',
  },
};

const {
  user: { email },
} = responseData;
```

## 3. деструктуризация массивов

### базовый синтаксис

```js
const colorsList = ['red', 'green', 'blue'];

const [firstColor, secondColor] = colorsList;
```

### пропуск значений

```js
const numbersArray = [10, 20, 30];

const [, , thirdNumber] = numbersArray;
```

### значения по умолчанию

```js
const numbersArray = [5];

const [firstNum, secondNum = 100] = numbersArray;
```

### обмен переменных

```js
let aValue = 1;
let bValue = 2;

[aValue, bValue] = [bValue, aValue];
```

## 4. Spread оператор (`...`)

- это оператор распаковки, он разворачивает массив или объект

### в массивах

```js
const arr1 = [1, 2];
const arr2 = [3, 4];

const combinedArr = [...arr1, ...arr2];
```

- результат `[1, 2, 3, 4]`

### добавление элементов

```js
const baseArray = [1, 2];

const newArray = [...baseArray, 3];
```

### копирование массива

```js
const originalArr = [1, 2, 3];

const copiedArr = [...originalArr];
```

- но это поверхностная копия

### в объектах

```js
const baseUser = { name: 'Alex' };

const extendedUser = {
  ...baseUser,
  age: 25,
};
```

### перезапись свойств

```js
const baseUser = { name: 'Alex', age: 20 };

const updatedUser = {
  ...baseUser,
  age: 30,
};
```

- последнее значение побеждает

### копирование объекта

```js
const obj1 = {
  nested: { value: 10 },
};

const obj2 = { ...obj1 };

obj2.nested.value = 999;

console.log(obj1.nested.value); // 999 😱
```

- вложенный объект не копируется глубоко - это shallow copy

## 5. Rest оператор (`...`)

- он выглядит так же, но используется в деструктуризации

### в объектах:

```js
const userData = {
  name: 'Alex',
  age: 25,
  city: 'Amsterdam',
};

const { name, ...restData } = userData;
```

- `restData`:

```js
{
  age: 25,
  city: "Amsterdam"
}
```

### в массивах

```js
const numbersList = [1, 2, 3, 4];

const [firstItem, ...restItems] = numbersList;
```

### как не путаться

```
Оператор      Где используется      Что делает

...spread     справа                разворачивает
...rest       слева                 собирает
```

## где реально используется

### в React очень часто

```js
const { title, description } = props;
```

### в функциях

```js
function printUser({ name, age }) {
  console.log(name, age);
}
```

### в копировании state

```js
const newState = { ...prevState, count: 10 };
```
