# optional chaining

## 1. проблема, с которой всё началось

- пусть есть объект:

```js
const userProfile = {
  name: 'Alex',
  address: {
    city: 'Amsterdam',
  },
};

// попробуем получить:

console.log(userProfile.address.city); // всё ок
```

- а если свойства нет?

```js
console.log(userProfile.contact.phone); // Cannot read properties of undefined
```

- потому что `contact` -> `undefined`
- JS пытается взять `.phone` у `undefined`
- это runtime error

## 2. как раньше решали проблему

- писали много проверок:

```js
if (userProfile.contact && userProfile.contact.phone) {
  console.log(userProfile.contact.phone);
}

// или

const phoneValue = userProfile.contact && userProfile.contact.phone;
```

- это:
  - громоздко
  - плохо читается
  - неудобно при глубокой вложенности

## 3. optional chaining (`?.`)

- ES6+ (точнее ES2020) дал решение:

```js
userProfile.contact?.phone;
```

### как это работает

```js
const phoneValueSafe = userProfile.contact?.phone;
```

- если `contact` существует -> берём `.phone`
- если `contact === undefined/null` -> возвращает `undefined`
- ошибка не выбрасывается

### важно запомнить

`?.` - если слева есть значение - продолжай, иначе остановись

## 4. цепочки (глубокие объекты)

```js
const cityName = userProfile.address?.city;
```

- а если глубже:

```js
const zipCodeValue = userProfile.address?.zip?.code;
```

- если что-то в цепочке `undefined` -> результат `undefined`

## 5. optional chaining с массивами

```js
const usersList = [{ name: 'John' }, { name: 'Jane' }];

console.log(usersList[0]?.name);
```

- если индекс не существует

```js
console.log(usersList[5]?.name);
```

- вернёт `undefined`, без ошибки

## 6. optional chaining с функциями

```js
const apiObject = {
  getData: () => 'data',
};

apiObject.getData?.();
```

- если функции нет, ничего не произойдёт, вернётся `undefined`

## 7. optional properties (optional mark)

- это чаще используется в typescript, но концепцию нужно понимать

### что значит "optional property?"

- свойство может быть:
  - либо значением
  - либо отсутствовать

```js
const productItem = {
  name: 'Laptop',
  discount: undefined,
};
```

#### в typescript это пишется так:

```ts
type ProductType = {
  name: string;
  discount?: number;
};
```

- `?` означает:
  - поле не обязательно
  - может отсутствовать

## 8. разница `?.` и `?`

```
Синтаксис	  Где используется    	Что делает
`?.`        JS	                  безопасный доступ
`?`         TypeScript	          делает поле необязательным
```
