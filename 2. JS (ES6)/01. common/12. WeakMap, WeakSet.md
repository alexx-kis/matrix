# WeakMap, WeakSet

## 1. почему появились WeakMap и WeakSet

- в структурах Map и Set данные хранятся жёстко (strong reference)
- то есть пока объект лежит в Map или Set, он не будет удалён из памяти

```js
let userProfile = { name: 'Alex' };

const cacheStorage = new Map();
cacheStorage.set(userProfile, 'cached data');

// убрали ссылку
userProfile = null;
```

- но объект всё равно остаётся в памяти, потому что Map всё ещё на него ссылается
- это называется утечка памяти

## 2. WeakMap

- решает эту проблему
- в WeakMap ключи хранятся слабо (weak reference)
- если объект больше нигде не используется -> он автоматически удаляется из памяти вместе с записью в WeakMap

## 3. WeakMap - основные ограничения:

1. ключи - только объекты

```js
const weakMapStorage = new WeakMap();

const personObject = {};
weakMapStorage.set(personObject, 'data'); // ✅

weakMapStorage.set('test', 123); // ❌ ошибка
```

- потому что только объекты можно собирать с помощью garbage collector

2. нет перебора (iteration)

- нельзя:

```js
weakMapStorage.keys(); // ❌
weakMapStorage.values(); // ❌
weakMapStorage.entries(); // ❌
```

- потому что JS не знает, когда именно GC удалит объект
- если бы перебор был, то он был бы нестабильным

3. нет size

```js
weakMapStorage.size; // ❌ нет такого
```

## 4. методы WeakMap:

```js
weakMapStorage.set(objectKey, value);
weakMapStorage.get(objectKey);
weakMapStorage.has(objectKey);
weakMapStorage.delete(objectKey);
```

## 5. пример использования

- приватные данные объекта

```js
const privateStorage = new WeakMap();

function createAccount(accountName) {
  const accountObject = { name: accountName };

  privateStorage.set(accountObject, {
    balance: 0,
  });

  return accountObject;
}

const myAccount = createAccount('Alex');

console.log(privateStorage.get(myAccount).balance); // 0
```

- данные не лежат в самом объекте
- нельзя случайно получить их извне
- при удалении объекта данные тоже исчезнут

## 6. WeakSet - аналогично

- это как Set, но:
  - хранит только объекты
  - хранит их слабо

### методы WeakSet

```js
weakSetCollection.add(objectValue);
weakSetCollection.has(objectValue);
weakSetCollection.delete(objectValue);
```

## 7. пример WeakSet

```js
const processedItems = new WeakSet();

function processItem(itemObject) {
  if (processedItems.has(itemObject)) {
    return;
  }

  processedItems.add(itemObject);

  console.log('Processing:', itemObject);
}

const orderItem = {};

processItem(orderItem); // обработка
processItem(orderItem); // пропуск
```

- если `orderItem` исчезнет -> он автоматически исчезнет из WeakSet

## 8. главное отличие (Map vs WeakMap)

```
.	              Map	      WeakMap
Ключи	          любые	    только объекты
Ссылки	        сильные	  слабые
Перебор	        ✅	       ❌
size	          ✅        ❌
GC очистка	    ❌        ✅
```

## 9. когда использовать

- WeakMap если:
  - нужно привязать данные к объекту
  - не хочешь мешать garbage collector
  - делаешь кэш
  - делаешь приватные поля

- WeakSet если:
  - нужно хранить "флаг" для объекта, например:
    - уже обработан
    - уже посещён
    - уже закеширован

## 10. важная мысль

- WeakMap / WeakSet - это не просто коллекции, а инструменты управления памятью
