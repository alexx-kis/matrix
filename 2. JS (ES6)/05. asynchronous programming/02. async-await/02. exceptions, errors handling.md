# exceptions/errors handling

- когда Promise завершается с ошибкой - await выбрасывает exception
- поэтому ошибки в async-await обычно обрабатываются через try-catch

```js
function loadProfileData() {
  return new Promise((resolve, reject) => {
    reject('Profile loading failed');
  });
}

async function showProfileData() {
  try {
    const profileData = await loadProfileData();

    console.log(profileData);
  } catch (profileError) {
    console.log(profileError);
  }
}

showProfileData(); // Profile loading failed
```

- без try-catch:

```js
function getOrderData() {
  return Promise.reject('Order error');
}

async function loadOrderData() {
  const orderData = await getOrderData();

  console.log(orderData);
}

loadOrderData(); // Uncaught (in promise) Order error
```

- `Uncaught (in promise)` появляется потому что:
  1. Promise завершился ошибкой
  2. никто ее не обработал
  3. ошибка "вылетела наружу"

- try-catch ловит и обычные ошибки тоже

```js
async function parseUserData() {
  try {
    const userInfo = {
      username: 'Alex',
    };

    console.log(userInfo.address.city);
  } catch (runtimeError) {
    console.log(runtimeError);
  }
}

parseUserData();
```

- здесь ошибка не связана с Promise, но try-catch все равно ее поймает

## try-catch вокруг нескольких await

```js
function getTokenData() {
  return Promise.resolve('token_123');
}

function getPrivateData() {
  return Promise.reject('Access denied');
}

async function loadPrivatePage() {
  try {
    const tokenData = await getTokenData();

    console.log(tokenData);

    const privateData = await getPrivateData();

    console.log(privateData);
  } catch (privateError) {
    console.log(privateError);
  }
}

loadPrivatePage();
```

- результат:

```
token_123
Access denied
```

- после ошибки код дальше не идет

```js
async function runCalculation() {
  try {
    await Promise.reject('Calculation failed');

    console.log('Этот код не выполнится');
  } catch (calculationError) {
    console.log(calculationError);
  }
}

runCalculation();
```

- после ошибки выполнение сразу переходит в `catch`

## можно выбрасывать свои ошибки

```js
async function validateAgeValue(ageValue) {
  if (ageValue < 18) {
    throw new Error('Age is too low');
  }

  return 'Access granted';
}

async function checkAccessPermission() {
  try {
    const accessMessage = await validateAgeValue(15);

    console.log(accessMessage);
  } catch (accessError) {
    console.log(accessError.message);
  }
}

checkAccessPermission(); // Age is too low
```

## практический пример

```js
async function loadDashboardInfo() {
  try {
    console.log('Показать loader');

    const dashboardData = await Promise.resolve('Dashboard data');

    console.log(dashboardData);
  } catch (dashboardError) {
    console.log(dashboardError);
  } finally {
    console.log('Скрыть loader');
  }
}

loadDashboardInfo();
```

## обработка ошибок fetch

```js
async function loadProductsData() {
  try {
    const responseObject = await fetch('/api/products');

    if (!responseObject.ok) {
      throw new Error('HTTP error');
    }

    const productsData = await responseObject.json();

    console.log(productsData);
  } catch (productsError) {
    console.log(productsError.message);
  }
}
```

- fetch() не делает reject при 404, 500, 403
- он делает reject только при сетевых ошибках, поэтому проверяют `responseObject.ok` или `responseObject.status`

- try-catch не ловит ошибки асинхронно вне await:

```js
async function runTimerTask() {
  try {
    setTimeout(() => {
      throw new Error('Timer error');
    }, 1000);
  } catch (timerError) {
    console.log(timerError);
  }
}

runTimerTask();
```

- catch не поймает ошибку
- потому что `setTimeout` выполняется позже, в другом call stack
- когда ошибка возникает - try-catch уже завершился

### как правильно

- вариант 1 - try-catch внутри setTimeout

```js
setTimeout(() => {
  try {
    throw new Error('Timer error');
  } catch (timerError) {
    console.log(timerError);
  }
}, 1000);
```

- вариант 2 - Promise

```js
function createDelay() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      reject(new Error('Delay failed'));
    }, 1000);
  });
}

async function runDelayTask() {
  try {
    await createDelay();
  } catch (delayError) {
    console.log(delayError.message);
  }
}

runDelayTask();
```
