# async/await class methods

- в классах методы могут быть асинхронными точно так же, как обычные функции
- для этого перед именем метода ставится `async`

## базовый пример

```js
class UserService {
  async getUserName() {
    return 'Alex';
  }
}

const userService = new UserService();

userService.getUserName().then((userName) => {
  console.log(userName);
});
```

- почему работает `.then()`? - потому что любой async-метод всегда возвращает promise

```js
// этот код:

class UserService {
  async getUserName() {
    return 'Alex';
  }
}

// примерно эквивалентен:

class UserService {
  getUserName() {
    return Promise.resolve('Alex');
  }
}
```

## использование await внутри метода

- обычно async-методы нужны именно для этого

```js
class ProductService {
  async getProductData() {
    const productData = await Promise.resolve({
      title: 'Laptop',
    });

    return productData;
  }
}
```

## вызов async-метода через await

- часто один async-метод вызывает другой

```js
class SettingsService {
  async getThemeName() {
    return 'dark';
  }

  async printThemeName() {
    const themeName = await this.getThemeName();

    console.log(themeName);
  }
}

const settingsService = new SettingsService();

settingsService.printThemeName();
```

## реальный пример с fetch

```js
class ApiClient {
  async getUsers() {
    const responseObject = await fetch('/api/users');
    const usersList = await responseObject.json();
    return usersList;
  }
}

const apiClient = new ApiClient();

const usersList = await apiClient.getUsers();
```

## обработка ошибок внутри метода

```js
class OrdersService {
  async getOrders() {
    try {
      const responseObject = await fetch('/api/orders');
      return await responseObject.json();
    } catch (requestError) {
      console.log(requestError);
      return [];
    }
  }
}
```

- либо обрабатывать ошибку снаружи

```js
class OrdersService {
  async getOrders() {
    const responseObject = await fetch('/api/orders');
    return responseObject.json();
  }
}

const ordersService = new OrdersService();

try {
  const ordersList = await ordersService.getOrders();
  console.log(ordersList);
} catch (requestError) {
  console.log(requestError);
}
```

## какой подход лучше?

- часто сервисы не ловят ошибки сами, а обработка происходит там, где метод вызвали
- так код становится более гибким

## async static methods

- асинхронными могут быть и статические методы

```js
class Configuration {
  static async loadConfig() {
    return {
      apiUrl: '/api',
    };
  }
}

const configData = await Configuration.loadConfig();

console.log(configData);
```

## наследование и async-методы

```js
class BaseRepository {
  async getData() {
    return 'Base data';
  }
}

class UserRepository extends BaseRepository {
  async getData() {
    const baseData = await super.getData();

    return `${baseData} + users`;
  }
}

const userRepository = new UserRepository();

console.log(await userRepository.getData()); // 'Base data + users'
```

## async getter сделать нельзя

```js
class UserProfile {
  async get profileData() {
    return {}
  }
}
```

- ошибка синтаксиса

- поэтому вместо getter обычно делают обычный метод:

```js
class UserProfile {
  async getProfileData() {
    return {};
  }
}
```

## конструктор не может быть async

```js
class NotificationService {
  async constructor() {
//
  }
}
```

- это ошибка, потому что конструктор должен сразу вернуть созданный объект, а async всегда возвращает promise
- эти механизмы не совместимы

## как делают на практике

- создают обычный объект

```js
class NotificationService {
  constructor() {
    this.settingsData = null;
  }

  async initialize() {
    this.settingsData = await Promise.resolve({
      sound: true,
    });
  }
}

const notificationService = new NotificationService();

await notificationService.initialize();
```

## частый паттерн: initialize()

- очень часто встречается такой код:

```js
class DatabaseConnection {
  async initialize() {
    // подключение к БД
  }
}

// или

class AuthManager {
  async initialize() {
    // получение токена
  }
}
```

- потому что конструктор не может быть асинхронным

## главное, что нужно запомнить

- async-метод

```js
class ExampleService {
  async loadData() {
    //
  }
}
```

- всегда возвращает promise

- внутри методов можно использовать `await` точно так же, как в обычных функциях

- async могут быть:
  - обычные методы
  - статические методы
  - методы в наследниках

- async не могут быть:
  - конструктор
  - getters/setters

- поэтому для асинхронной инициализации обычно используют отдельный метод:

```js
await serviceInstance.initialize();
```
