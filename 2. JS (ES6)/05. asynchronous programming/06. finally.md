# finally

- это метод Promise, который выполняется всегда
- не важно success, error, fulfilled, rejected

```js
promiseValue.finally(() => {
  //
});
```

## зачем нужен

- обычно для:
  - cleanup
  - скрытия loader
  - остановки spinner
  - очистки ресурсов
  - закрытия соединений

### пример

```js
// успех
Promise.resolve('success').finally(() => {
  console.log('cleanup');
});

// ошибка
Promise.reject('failed')
  .finally(() => {
    console.log('cleanup');
  })
  .catch((errorValue) => {
    console.log(errorValue);
  });
```

## типичная структура

```js
showLoader();

fetchUserData()
  .then((userData) => {
    console.log(userData);
  })
  .catch((errorValue) => {
    console.log(errorValue);
  })
  .finally(() => {
    hideLoader();
  });
```

## почему finally удобен

- без него пришлось бы дублировать код

```js
fetchUserData()
  .then((userData) => {
    hideLoader();

    console.log(userData);
  })
  .catch((errorValue) => {
    hideLoader();

    console.log(errorValue);
  });
```

## важная особенность

- finally не получает результат Promise

```js
Promise.resolve('hello').finally((resultValue) => {
  console.log(resultValue); // undefined
});
```

- потому что finally не предназначен для работы с данными
- он предназначен для cleanup

## finally может сломать chain

```js
Promise.resolve('success')
  .finally(() => {
    throw new Error('finally failed');
  })
  .catch((errorObject) => {
    console.log(errorObject.message);
  });
```

- потому что ошибка внутри finally создала rejected Promise

### finally тоже возвращает Promise

- как then и catch

```js
const finalPromise = Promise.resolve(1).finally(() => {});

console.log(finalPromise);
```

## асинхронный finally

- можно вернуть Promise

```js
Promise.resolve('done')
  .finally(() => {
    return new Promise((resolveFunction) => {
      setTimeout(() => {
        console.log('async cleanup');

        resolveFunction();
      }, 1000);
    });
  })
  .then((resultValue) => {
    console.log(resultValue);
  });
```

- chain подождет завершения finally

1. основной Promise завершился
2. запустился async cleanup
3. только потом продолжится `.then()`
