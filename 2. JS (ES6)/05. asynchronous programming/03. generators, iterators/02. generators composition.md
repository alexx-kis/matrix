# generators composition

- композиция генераторов – это ситуация, когда один генератор использует другой генератор
- главная цель – переиспользовать существующие генераторы и собирать сложные последовательности из более простых

- для этого существует специальный синтаксис: `yield*`

## без композиции

- допустим, есть 2 набора данных:

```js
function* generateNumbers() {
  yield 1;
  yield 2;
  yield 3;
}

function* generateLetters() {
  yield 'A';
  yield 'B';
  yield 'C';
}
```

- хотим получить:

```
1
2
3
A
B
C
```

- без композиции пришлось бы писать:

```js
function* generateAllValues() {
  yield 1;
  yield 2;
  yield 3;

  yield 'A';
  yield 'B';
  yield 'C';
}
```

- это работает, но код дублируется

## `yield*`

- для подключения другого генератора используется `yield anotherGenerator()`

```js
function* generateNumbers() {
  yield 1;
  yield 2;
  yield 3;
}

function* generateLetters() {
  yield 'A';
  yield 'B';
  yield 'C';
}

function* generateAllValues() {
  yield* generateNumbers();
  yield* generateLetters();
}

for (const value of generateAllValues()) {
  console.log(value);
}
```

- результат:

```
1
2
3
A
B
C
```

## что делает `yield*`

- когда интерпретатор встречает `yield generateNumbers()`, он фактически делает что-то похожее на:

```js
for (const item of generateNumbers()) {
  yield item
}
```

- то есть:
  - запускает другой генератор
  - получает все его значения
  - передает их наружу

## вложенная композиция

- генераторы можно объединять цепочками

```js
function* generateDigits() {
  yield 1;
  yield 2;
}

function* generateCharacters() {
  yield 'A';
  yield 'B';
}

function* generateCombined() {
  yield* generateDigits();
  yield* generateCharacters();
}

function* generateEverything() {
  yield 'START';

  yield* generateCombined();

  yield 'END';
}
```

- результат:

```
START
1
2
A
B
END
```

## `yield*` работает не только с генераторами

- оператор ожидает любой итерируемый объект
- например, массив:

```js
function generateValues() {
  yield * [10, 20, 30];
}
```

- результат:

```
10
20
30
```

- строка:

```js
function* generateCharacters() {
  yield* 'Hello';
}
```

- результат:

```
H
e
l
l
o
```

- Set:

```js
function* generateSetValues() {
  yield* new Set([1, 2, 3]);
}
```

- результат:

```
1
2
3
```

## обычный yield и yield\* – разница

### yield

```js
function* generateExample() {
  yield [1, 2, 3];
}
```

- результат:

```
[1, 2, 3]
```

- будет выдан один элемент – массив целиком

### yield\*

```js
function* generateExample() {
  yield* [1, 2, 3];
}
```

- результат:

```
1
2
3
```

- будет выдан каждый элемент массива отдельно

## практический пример

- допустим, есть генераторы для разных отделов меню:

```js
function* generateMainMenu() {
  yield 'Home';
  yield 'Catalog';
}

function* generateUserMenu() {
  yield 'Profile';
  yield 'Settings';
}

function* generateFullMenu() {
  yield* generateMainMenu();
  yield* generateUserMenu();
}

for (const menuItem of generateFullMenu()) {
  console.log(menuItem);
}
```

- результат:

```
Home
Catalog
Profile
Settings
```

- каждый генератор отвечает только за свою часть данных

## что важно запомнить

- обычный yield выдает одно значение
- yield\* выдает все значения из другого итерируемого объекта

- композиция генераторов позволяет
  - разбивать большие генераторы на маленькие
  - переиспользовать существующие генераторы
  - строить цепочки генераторов
  - работать с любыми итерируемыми объектами (генераторы, массивы, строки, Set, итд)
