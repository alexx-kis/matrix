# simple generators syntax and how it works

- генераторы появились в JS для создания итераторов более удобным способом
- до генераторов, если объект нужно было сделать итерируемым, приходилось вручную писать `Symbol.iterator()` и объект-итератор с методом `next()`
- генератор делает это автоматически

## базовый синтаксис

- генератор – это функция со звездочкой `*`

```js
function* createCounter() {
  yield 1;
  yield 2;
  yield 3;
}
```

- ключевое слово `yield` означает "верни значение и останови выполнение до следующего вызова"

## что возвращает генератор

- обычная функция:

```js
function getNumber() {
  return 10;
}

const result = getNumber();

console.log(result); // 10
```

- генератор:

```js
function* createCounter() {
  yield 1;
  yield 2;
  yield 3;
}

const generatorObject = createCounter();

console.log(generatorObject); // Object [Generator] {}
```

- то есть генератор не запускается сразу, он возвращает специальный объект-итератор

## метод `next()`

- используется для получения значений

```js
function* createCounter() {
  yield 1;
  yield 2;
  yield 3;
}

const counterIterator = createCounter();

console.log(counterIterator.next()); // { value: 1, done: false }
console.log(counterIterator.next()); // { value: 2, done: false }
console.log(counterIterator.next()); // { value: 3, done: false }
console.log(counterIterator.next()); // { value: undefined, done: true }
```

## что такое done

- каждый вызов `next()` возвращает объект:

```js
{
  value: ...,
  done: ...,
}
```

- пока есть значения, done -> false
- когда генератор закончился, done -> true, а value -> undefined

## как работает yield

```js
function* generateValues() {
  console.log('Start');
  yield 'A';

  console.log('Middle');
  yield 'B';

  console.log('End');
}

const valuesIterator = generateValues();

// первый вызов:
valuesIterator.next(); // 'Start'
// результат { value: 'A', done: false }
// выполнение остановилось на yield 'A'

// второй вызов:
valuesIterator.next(); // 'Middle'
// результат { value: 'B', done: false }

// третий вызов
valuesIterator.next(); // 'End'
// результат { value: undefined, done: true }
```

### yield ≈ 'паузе'

- можно представить генератор так:

```js
yield value;

// как:

return value;
```

- но с важным отличием:
  - `return` завершает функцию навсегда
  - `yield` ставит выполнение на паузу

- поэтому генератор можно запускать много раз через `next()`

## генератор является итератором

```js
function* createLetters() {
  yield 'A';
  yield 'B';
  yield 'C';
}

const lettersIterator = createLetters();

lettersIterator.next(); // потому что генератор уже реализует интерфейс итератора
```

## генератор является итерируемым объектом

- это важная особенность

- генератор одновременно:
  - итератор
  - итерируемый объект

- поэтому работает в `for...of`

```js
function* createLetters() {
  yield 'A';
  yield 'B';
  yield 'C';
}

for (const letter of createLetters()) {
  console.log(letter);
}
```

```
A
B
C
```

## зачем нужны генераторы

- они позволяют:
  - создавать собственные итераторы
  - выдавать данные по одному элементу
  - не хранить сразу большой массив в памяти
  - строить бесконечные последовательности
  - упрощать работу с `Symbol.iterator`

```js

  let currentId = 1;

  while (true) {

}

const idIterator = createIdGenerator();

console.log(idIterator.next().value) // 1
```

- генератор создает значения только тогда, когда они понадобились

## что важно запомнить

```js
function* exampleGenerator() {
  yield 10;
  yield 20;
}
```

- `function*` – объявление генератора
- вызов генератора не запускает код внутри
- генератор возвращает объект-итератор
- `yield` возвращает значение и ставить выполнение на паузу
- `next()` продолжает выполнение с места остановки
- результат `next()` имеет вид

```js
{
  value: ...,
  done: ...,
}
```

- генератор одновременно является и итератором, и итерируемым объектом
- генераторы чаще всего используются для создания собственных последовательностей и итераторов
