# exceptions/errors handling

- одна из главных причин появления promise - нормальная обработка ошибок в асинхронном коде

## exception

- это ошибка, которая прерывает выполнение кода

## синхронная обработка ошибок

- обычный JS использует try-catch

```js
try {
  const resultNumber = JSON.parse('wrong json');
} catch (parseError) {
  console.log(parseError);
}
```

## но с асинхронностью возникают проблемы

- например callback

```js
setTimeout(() => {
  throw new Error('timer error');
}, 1000);
```

- `try-catch` снаружи не поймает такую ошибку
- потому что callback выполнится позже, когда основной стек вызовов уже закончится

## как promise решает проблему

- он автоматически перехватывает ошибки

### ошибка внутри promise

```js
const brokenPromise = new Promise(() => {
  throw new Error('something failed');
});
```

- promise автоматически станет rejected

### получение ошибки через .catch()

```js
brokenPromise.catch((errorObject) => {
  console.log(errorObject.message);
});
```

- вывод: something failed

### важный момент

- throw внутри promise === reject()

```js
// вот это:
throw new Error('failed');

// почти эквивалентно:
reject(new Error('failed'));
```

### пример reject()

```js
const apiPromise = new Promise((resolveFunction, rejectFunction) => {
  rejectFunction(new Error('server unavailable'));
});
```

## обработка ошибки

```js
fetchUserData()
  .then((userValue) => {
    return loadOrders(userValue.id);
  })
  .then((ordersValue) => {
    return processOrders(ordersValue);
  })
  .catch((errorObject) => {
    console.log(errorObject);
  });
```

- если ошибка произошла:
  - в любом `.then()`
  - внутри promise
  - внутри callback в `.then()`
- то управление перейдет в `.catch()`

```js
Promise.resolve('start')
  .then(() => {
    throw new Error('then failed');
  })
  .catch((errorObject) => {
    console.log(errorObject.message);
  });
```

- вывод: then failed

## почему это удобно

- не нужен try-catch на каждом шаге
- один catch может обработать ошибки всей цепочки

- promise chain работает примерно как

```js
try {
  // весь chain
} catch (errorObject) {
  // catch
}
```

- но для асинхронного кода

### catch тоже возвращает promise

```js
Promise.reject('error')
  .catch((errorValue) => {
    console.log(errorValue);

    return 'recovered';
  })
  .then((resultValue) => {
    console.log(resultValue);
  });
```

- вывод: error ; recovered

### что произошло

1. promise был rejected
2. `.catch()` обработал ошибку
3. `return 'recovered'` создал новый fulfilled promise
4. следующий `.then()` получил значение

## ошибка внутри `catch()`

```js
Promise.reject('initial error')
  .catch(() => {
    throw new Error('new error');
  })
  .catch((errorObject) => {
    console.log(errorObject.message);
  });
```

- вывод: new error

## finally

- это метод, который выполнится всегда

```js
Promise.resolve('done').finally(() => {
  console.log('cleanup');
});

Promise.reject('failed').finally(() => {
  console.log('cleanup');
});
```

- finally выполнится в обоих случаях

### для чего нужен finally

- обычно для cleanup
  - скрыть loader
  - закрыть соединение
  - остановить spinner
  - очистить ресурсы

### важная особенность finally

- finally не получает результат promise

```js
Promise.resolve('hello').finally((valueData) => {
  console.log(valueData);
});
```

- вывод: undefined

- потому что finally предназначен не для работы с данными, а для cleanup

## частая ошибка

- пытаться обработать async ошибки через внешний try-catch

```js
try {
  fetchData().then(() => {
    throw new Error('failed');
  });
} catch (errorObject) {
  console.log(errorObject);
}
```

- это не сработает, потому что ошибка происходит асинхронно

- правильно:

```js
fetchData()
  .then(() => {
    throw new Error('failed');
  })
  .catch((errorObject) => {
    console.log(errorObject);
  });
```
