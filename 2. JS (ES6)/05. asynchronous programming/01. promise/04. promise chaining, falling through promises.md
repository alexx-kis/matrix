# promise chaining, falling through promises

- асинхронные операции часто зависят друг от друга
- например:
  1. загрузить пользователя
  2. получить его заказы
  3. получить товары из заказов
  4. сохранить результат

- каждый следующий шаг требует результат предыдущего

### без chaining - callback hell

```js
loadUserData((loadedUser) => {
  loadOrdersData(loadedUser.id, (loadedOrders) => {
    loadProductsData(loadedOrders, (loadedProducts) => {
      console.log(loadedProducts);
    });
  });
});
```

### с chaining:

```js
loadUserData()
  .then((userValue) => {
    return loadOrdersData(userValue.id);
  })
  .then((ordersValue) => {
    return loadProductsData(ordersValue);
  })
  .then((productsValue) => {
    console.log(productsValue);
  });
```

- код становится плоским и читаемым

## как работает chaining

- `then()` всегда возвращает новый promise
- даже если ничего не возвращать

```js
const chainPromise = Promise.resolve(10);

const nextPromise = chainPromise.then((numberValue) => {
  return numberValue * 2;
});

console.log(nextPromise);
```

- `nextPromise` - это новый promise

### что происходит внутри chain

1. `Promise.resolve(5)` создает fulfilled promise

2. ```js
   .then((numberValue) => { return numberValue + 3; });
   ```

- результат: 8

3. `then()` автоматически оборачивает результат в promise

- фактически происходит что-то вроде `Promise.resolve(8)`

- поэтому следующий `then` получает 8

```js
Promise.resolve(5)
  .then((numberValue) => {
    return numberValue + 3;
  })
  .then((resultValue) => {
    console.log(resultValue);
  });
```

### возврат обычного значения

```js
.then(() => {
  return 100
})
```

- автоматически превращается в

```js
Promise.resolve(100);
```

## возврат Promise

```js
function fetchProfileData() {
  return Promise.resolve('profile loaded');
}

Promise.resolve()
  .then(() => {
    return fetchProfileData();
  })
  .then((profileValue) => {
    console.log(profileValue);
  });
```

- `then()` получил Promise

```js
return fetchProfileData();
```

- chain не идет дальше сразу, он ждет завершения этого Promise
- после завершения результат передается дальше
- это называется flattening или falling through promises

### что значит falling through?

- если вернуть Promise внутри `.then()`

```js
return anotherPromise;
```

- то внешний chain не получит Promise как значение, он провалится внутрь, дождется результата

```js
const delayedPromise = new Promise((resolveFunction) => {
  setTimeout(() => {
    resolveFunction(50);
  }, 1000);
});

Promise.resolve()
  .then(() => {
    return delayedPromise;
  })
  .then((resultValue) => {
    console.log(resultValue);
  });
```

- следующий `.then()` получает не Promise, а уже результат Promise - 50

## ошибка внутри chain

- любая ошибка ломает цепочку и идет в `.catch()`

```js
Promise.resolve(10)
  .then((numberValue) => {
    return numberValue * 2;
  })
  .then(() => {
    throw new Error('chain failed');
  })
  .then(() => {
    console.log('never runs');
  })
  .catch((errorObject) => {
    console.log(errorObject.message);
  });
```

- вывод: chain failed

- почему then после ошибки не выполнился? - потому что Promise chain перешел в rejected state

### старая форма

```js
promiseValue.then(successHandler, errorHandler);
```

- но обычно используют

```js
promiseValue.then(successHandler).catch(errorHandler);
```

- это читается лучше

## частая ошибка

- не вернуть Promise

```js
loadUserData()
  .then((userValue) => {
    loadOrdersData(userValue.id); // нет return
  })
  .then((ordersValue) => {
    console.log(ordersValue);
  });
```

- в итоге следующий then получит `undefined`, а не результат `loadOrdersData`

- правильно:

```js
loadUserData()
  .then((userValue) => {
    return loadOrdersData(userValue.id);
  })
  .then((ordersValue) => {
    console.log(ordersValue);
  });
```
