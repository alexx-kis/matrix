# custom promise

- это promise, который мы создаем сами через `new Promise()`

## зачем создавать promise вручную?

- потому что многие старые API работают через callbacks
- или потому что мы хотим обернуть какую-то асинхронную логику в promise

```js
const customPromise = new Promise((resolveFunction, rejectFunction) => {
  //
});
```

- resolveFunction - это функция успешного завершения, вызывается, когда promise -> fulfilled
- rejectFunction - это функция ошибки, вызывается, когда promise -> rejected

```js
const simplePromise = new Promise((resolveFunction) => {
  resolveFunction('success');
});

simplePromise.then((resultValue) => {
  console.log(resultValue); // success
});
```

## promise с таймером

```js
const timerPromise = new Promise((resolveFunction) => {
  setTimeout(() => {
    resolveFunction('timer finished');
  }, 2000);
});

timerPromise.then((resultValue) => {
  console.log(resultValue);
});
```

## promise с ошибкой

```js
const failedPromise = new Promise((resolveFunction, rejectFunction) => {
  const hasError = true;

  if (hasError) {
    rejectFunction('server error');
  }
});

failedPromise.catch((errorValue) => {
  console.log(errorValue);
});
```

- обычно promise возвращают из функции, чтобы вызывать функцию много раз

```js
function waitDelay(delayTime) {
  return new Promise((resolveFunction) => {
    setTimeout(() => {
      resolveFunction('finished');
    }, delayTime);
  });
}

waitDelay(1000).then((resultValue) => {
  console.log(resultValue);
});
```

### еще более полезный пример

- фейковый запрос к API

```js
function loadArticleData(articleId) {
  return new Promise((resolveFunction, rejectFunction) => {
    const isArticleFound = true;

    setTimeout(() => {
      if (isArticleFound) {
        resolveFunction({
          id: articleId,
          title: 'JavaScript Guide',
        });
      } else {
        rejectFunction('article not found');
      }
    }, 1500);
  });
}

loadArticleData(10)
  .then((articleData) => {
    console.log(articleData);
  })
  .catch((errorValue) => {
    console.log(errorValue);
  });
```

- что происходит внутри promise:

1. создается promise
2. он сразу получает state -> pending
3. асинхронная операция работает
4. когда операция закончилась:
   - либо `resolveFunction(value);`
   - либо `rejectFunction(error);`

## важный момент

- executor вызывается сразу

- вот эта функция называется executor:

```js
(resolveFunction, rejectFunction) => {
  //
};
```

- она запускается мгновенно

```js
console.log('start');

const instantPromise = new Promise(() => {
  console.log('executor');
});

console.log('end');
```

- вывод:

```
start
executor
end
```
