# states of promises

- у Promise есть внутреннее состояние
- всего их бывает 3: pending, fulfilled, rejected

## зачем нужны состояния

- Promise представляет асинхронную операцию
- например, запрос к серверу, таймер, чтение файла
- такая операция не завершается мгновенно
- поэтому promise должен хранить:
  - операция еще выполняется?
  - успешно завершилась?
  - произошла ошибка?

- для этого существуют states

## 1. pending

- операция еще не завершилась
- это начальное состояние любого promise

```js
const loadingPromise = new Promise(() => {});
```

- этот promise не завершился успешно и не завершился с ошибкой
- он навсегда останется pending

- более реальный пример:

```js
const timerPromise = new Promise((resolveFunction) => {
  setTimeout(() => {
    resolveFunction('done');
  }, 2000);
});
```

- сразу после создания promise находится в состоянии pending, потому что таймер еще не закончился
- через 2 секунды вызывается `resolveFunction('done')`, и состояние меняется

## 2. fulfilled

- означает, что операция успешно завершилась
- promise становится fulfilled, когда вызывается `resolve(value)`

```js
const successPromise = new Promise((resolveFunction) => {
  resolveFunction('success');
});
```

### значение promise

- когда promise fulfilled, он хранит результат
- например:

```js
resolveFunction('hello');
```

- тогда результат - 'hello'

### получение результата

- через `.then()`

```js
successPromise.then((resultValue) => {
  console.log(resultValue);
});
```

- вывод: success

## 3. rejected

- означает, что операция завершилась ошибкой
- promise становится rejected, когда вызывается `reject(error)`

```js
const failedPromise = new Promise((resolveFunction, rejectFunction) => {
  rejectFunction('server error');
});
```

- состояние: rejected

### ошибка promise

- promise хранит причину ошибки

### получение ошибки

- через `.catch()`

```js
failedPromise.catch((errorValue) => {
  console.log(errorValue);
});
```

- вывод: server error

## очень важный момент

- promise может изменить состояние только 1 раз
- pending -> стал fulfilled или rejected.

## еще один важный момент

- promise state != result
- состояние и значение - это разные вещи

- fulfilled имеет:
  - state: fulfilled
  - result: какое-то значение

- rejected имеет:
  - state: rejected
  - result: ошибка
