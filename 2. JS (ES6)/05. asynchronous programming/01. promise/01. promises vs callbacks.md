# promises vs callbacks

- почти весь асинхронный код строится либо на callbacks, либо на promise

## что такое асинхронность

- JS выполняет код сверху вниз по одной строке
- но некоторые операции занимают время:
  - запрос к API
  - чтение файла
  - таймер
  - работа с базой данных
  - загрузка картинки

- JS не хочет "замораживать" весь сайт на это время
- поэтому такие операции выполняются асинхронно

## callback

- это функция, которую мы передаём в другую функцию, чтобы она вызвалась позже

```js
function printMessage(messageText) {
  console.log(messageText);
}

function runTask(callbackFunction) {
  console.log('task started');

  callbackFunction();
}

runTask(printMessage);
```

## callback в асинхронности

```js
setTimeout(() => {
  console.log('done');
}, 2000);
```

- что происходит:
  1. `setTimeout` запускается
  2. JS НЕ ждёт 2 секунды
  3. код идёт дальше
  4. через 2 секунды вызывается callback

## реальный пример callback

- представим запрос к серверу:

```js
function loadUserData(callbackFunction) {
  setTimeout(() => {
    const userData = {
      name: 'Alex',
    };

    callbackFunction(userData);
  }, 1000);
}

loadUserData((receivedUser) => {
  console.log(receivedUser);
});
```

## проблема callback

- когда асинхронных операций много, код начинает превращаться в "лесенку"
- это называется callback hell:

```js
loadUserData((loadedUser) => {
  loadOrders(loadedUser.id, (loadedOrders) => {
    loadProducts(loadedOrders, (loadedProducts) => {
      saveProducts(loadedProducts, () => {
        console.log('finished');
      });
    });
  });
});
```

- проблемы:
  1. огромная вложенность
  2. трудно читать
  3. трудно дебажить
  4. трудно обрабатывать ошибки
  5. трудно поддерживать

## promise

- появился как решение этих проблем
- promise - это объект, который представляет результат асинхронной операции

### promise имеет состояния:

1. pending - операция ещё не выполнена
2. fulfilled - операция успешно завершилась, есть результат
3. rejected - произошла ошибка

- пример:

```js
const userPromise = new Promise((resolve, reject) => {
  const isSuccess = true;

  if (isSuccess) {
    resolve('success'); // операция завершилась успешно
  } else {
    reject('error'); // операция завершилась с ошибкой
  }
});
```

## получения результата Promise

```js
userPromise.then((resultValue) => {
  console.log(resultValue);
});
```

## обработка ошибок

```js
userPromise
  .then((resultValue) => {
    console.log(resultValue);
  })
  .catch((errorValue) => {
    console.log(errorValue);
  });
```

## почему promise лучше

1. нет callback hell - можно строить цепочки:

```js
loadUser()
  .then((userValue) => {
    return loadOrders(userValue.id);
  })
  .then((ordersValue) => {
    return loadProducts(ordersValue);
  })
  .then((productsValue) => {
    console.log(productsValue);
  });
```

- код становится плоским

2. удобная обработка ошибок

- в callback:

```js
functionValue(errorValue, resultValue);
```

- в promise

```js
.catch()
```

3. лучше читаемость - promise-chain выглядит линейно
4. основа async/await - работает поверх Promise

- без понимания Promise async/await понять невозможно

## очень важный момент

- Promise не делает код синхронным

```js
console.log('start');

fetchData().then(() => {
  console.log('done');
});

console.log('end');
```

- вывод:

```
start
end
done
```

- потому что promise - это всё ещё асинхронность

## главное, что нужно понять

- callback - это "вызови эту функцию позже"
- promise - это "в будущем появится результат"
